## 摘要
检索增强生成（RAG）智能体越来越多地在持久内存中运行，这些内存在用户会话间不断累积。这创造了一个新的攻击面：仅通过正常渠道交互的对手可以注入精心设计的内存，一旦被检索，就会引导智能体为未来用户生成的响应，而无需触及模型权重或代码。我们称这种攻击为多会话内存中毒（MSMP），并表明现有防御无法对其提供认证保护；静态语料库防御（RobustRAG、ReliabilityRAG）假设知识库是固定的，而启发式过滤器可以被流畅的企业风格文本绕过。我们提出带签名内存与平滑检索（SMSR），这是首个为此设置提供认证鲁棒性边界的防御。组件1在写入时添加HMAC-SHA256来源证明，阻止未签名注入。组件2在查询时应用随机内存消融和基于判决的多数投票，限制已认证对手的影响力。我们证明了无来源证明的检索时过滤器无法对自适应注入进行认证，为组件2推导出超几何证书，并形式化了“一致少数效应”，即一致的恶意答案在基于字符串的投票中以数值上的少数胜出，而基于判决的投票则消除了它。在15个企业场景（3150次重复试验）中，组件1将所有未签名变种的攻击成功率从93-100%降至0%。对于具有单次注入的认证对手，组件2将成功率保持在8.0%（95% CI [5.8, 10.9]，n=450），低于认证的最坏情况。在一个端到端的仅查询攻击中，智能体本身写入毒化内容而非预先植入，SMSR在实时智能体堆栈上将成功率从65.3%降至5.3%（n=150，置信区间不重叠）。干净查询的效用为90%（组件1）和85%（组合）。

## Abstract
Retrieval-augmented generation (RAG) agents increasingly run with persistent memory that accumulates across user sessions. This creates a new attack surface: an adversary interacting only through normal channels can inject crafted memories that, once retrieved, steer the agent's responses for future users, without touching model weights or code. We call this Multi-Session Memory Poisoning (MSMP) and show that no existing defence certifies against it; static-corpus defences (RobustRAG, ReliabilityRAG) assume a fixed knowledge base, and heuristic filters are bypassed by fluent enterprise-style text. We present Signed Memory with Smoothed Retrieval (SMSR), the first defence with a certified robustness bound for this setting. Component 1 adds HMAC-SHA256 provenance at write time, blocking unsigned injection. Component 2 applies randomised memory ablation with verdict-based majority voting at query time, bounding the influence of authenticated adversaries. We prove that no provenance-free retrieval-time filter can certify against adaptive injection, derive a hypergeometric certificate for Component 2, and formalise the Consistent Minority Effect, whereby a consistent adversarial answer wins string-based voting as a numerical minority while verdict-based voting removes it. Across 15 enterprise scenarios (3,150 repeated trials), Component 1 cuts attack success from 93-100% to 0% for all unsigned variants. For an authenticated adversary with a single injection, Component 2 holds success to 8.0% (95% CI [5.8, 10.9], n=450), below the certified worst case. In an end-to-end query-only attack where the agent itself writes the poison rather than it being pre-seeded, SMSR reduces success from 65.3% to 5.3% (n=150, non-overlapping CIs) on a live agent stack. Clean-query utility is 90% (Component 1) and 85% (combined).

- **核心威胁**：论文定义了“多会话记忆投毒”（Multi‑Session Memory Poisoning，MSMP）。攻击者无需修改模型权重或应用代码，仅通过正常的用户交互（如客户、员工或受感染的工具）即可将恶意记忆注入数据库。这些毒记忆在未来被其他用户查询触发时，会引导智能体产生特定恶意响应。
- **现有防御的不足**：已有的认证鲁棒防御（如 RobustRAG、ReliabilityRAG）均假设知识库是静态、预先索引的，无法应对运行时的动态写入。启发式过滤器（关键词黑名单、困惑度检测、语义异常检测）又容易被流畅的企业文本绕过。其他诸如 MINJA、AgentPoison 等攻击工作虽然揭示了威胁，但所提反制措施均缺乏形式化安全保证。
- **论文目标**：提出首个针对持久化 LLM 智能体系统中运行时记忆投毒的、具备形式化认证边界的防御方案，并为组件提供可证明的鲁棒性证书。

### 2. 方法论：SMSR 构造

SMSR 包含两个协同的组件，分别应对未签名攻击和已签名攻击。

- **组件 1：HMAC 来源证明（Provenance Tagging）**
  - **思想**：在可信写入通道上，为每一条合法记忆附加 HMAC‑SHA256 签名标签。签名输入包括记忆内容、会话 ID 和时间戳，基于服务器密钥 K。
  - **工作方式**：查询时仅放行带有有效 HMAC 标签的条目进入候选池；未获授权的直接数据库注入因缺少有效签名而被彻底滤除。
  - **安全保证**：在 HMAC 伪随机性假设下，未签名攻击者成功伪造标签的概率上限为 \(2^{-256}\)，从而对未签名对手达到 \((0, 2^{-256})\)‑SMSR 安全。

- **组件 2：随机记忆消融 + 判决投票**
  - **针对场景**：已认证对手可通过正常交互写入签名记忆，组件 1 无法拒止。组件 2 提供对这种“合法用户注入”的认证防护。
  - **核心机制**：
    1. **过取 + 随机消融**：从验证后的记忆池中取 top‑m 个候选（m > k），然后重复 nruns 次独立实验，每次从中无放回随机抽样 k 条记忆构建上下文 C(j)。
    2. **判决投票（Verdict‑Based Aggregation）**：每个上下文 C(j) 经 LLM 生成回答 r(j)，再由一个独立的 LLM 裁判（judge）给出判决 \(v^{(j)} \in \{\text{correct}, \text{malicious}, \text{neither}\}\)。最终以多数判决决定系统的最终响应（若“恶意”判决占多数则视为攻击成功）。
  - **一致少数效应（Consistent Minority Effect, CME）**：
    - **现象**：如果直接对生成的文本进行字符串多数投票，攻击者只需生成多个文本一致的回答（而良性回答措辞多样），即可在总票数未过半的情况下因字符串唯一最高频而胜出。
    - **消除方法**：判决投票要求裁判模型将语义相同的恶意回答均映射为同一标签 `malicious`，从而消除措辞差异导致的票数分散。
  - **形式化证书（Theorem 2）**：  
    假设 N 条记忆中有至多 t 条恶意，检索池 C 大小为 m，每次消融取样 k 条。定义无污染概率 \(p_{\text{clean}} = \binom{m-t'}{k} / \binom{m}{k}\)（\(t' \leq \min(t, m)\)）。则恶意判决获多数票的概率上界为：
    \[
    \delta = \sum_{i=\lceil n/2 \rceil}^{n} \binom{n}{i} (1 - p_{\text{clean}})^i p_{\text{clean}}^{n-i}
    \]
    该证书保证系统是 \((t, \delta)\)‑SMSR 安全的。

- **两者协同**：组件 1 构成写入时的强边界，组件 2 所依赖的池 `M_verified` 正是在此边界内构建，从而将组分纳入一种前后贯通的认证保卫体系。

### 3. 实验设计

- **威胁场景与数据**：
  - 构造一个名为 Nexora Corp 的虚拟企业 RAG 智能体，涵盖 15 个企业知识场景（财务政策、合规、IT 安全、HR、采购、事件响应等），每个场景定义一组合法答案和一个攻击者设定的恶意答案。
  - 记忆库预填充 10 条已签名的合法企业政策条目。攻击的投毒条目以自然的企业政策文本形式注入，而非明显的越狱字符串。

- **攻击类别**：
  1. **未签名注入**（MINJA 式）：直接数据库写入，无 HMAC 标签。含三类变种：直接注入（完全复制）、洪水（不同表述但语义等价）、隐蔽（融入自然研究笔记的表述）。
  2. **已认证注入**：攻击者作为合法用户通过正常智能体交互写入，条目获得有效签名。
  3. **启发式绕过**：专门构造能绕过关键词黑名单、困惑度过滤和语义异常检测的未签名注入。

- **防御配置**：
  - `none`：无防御基线。
  - `heuristic`：关键词黑名单 + 困惑度 + 语义异常（无来源证明）。
  - `c1`：仅 HMAC 来源证明。
  - `c1c2`：HMAC + 随机消融 + 判决投票（即完整的 SMSR）。
  - 参数：m=20, k=5, nruns=5（少数 7 仅作计算参考）。注入数 t∈{1,2,3}。

- **对比方法**：
  - 与 A‑MemGuard 风格的共识防御做对比（全量检索记忆增强响应与仅参数响应比对，不一致则回退到仅参数响应）。
  - 用两种不同模型族（Claude Haiku 4.5 和 claude‑sonnet‑4‑6）分别做智能体和裁判，验证泛化性。

- **评价指标**：
  - **攻击成功率（ASR）**：LLM 裁判认为最终输出为 `malicious` 的比例。
  - **效用**：纯净查询回答正确的比例。
  - **裁判可靠性**：通过 Haiku 与 Sonnet 裁判的 Cohen’s κ 衡量（κ=0.955）。

### 6. 主要结论与发现

- **组件 1 完全阻断未签名注入**：在所有未签名攻击变种（包括试图绕过启发式过滤的文本）上，攻击成功率从 93–100% 降至 0%。
- **组件 2 可认证地限制已认证攻击**：
  - 在小规模评估池（m′=11, t=1）中，30 次重复的池化 ASR 为 37.8%（CI [33.4, 42.3]），低于证书上界 \(\delta=41.5\%\)，边界紧密。
  - 在生产规模存储（20 种子记忆，m=20）下，t=1 的 ASR 降至 8.0%（CI [5.8, 10.9]），显著低于证书上界 \(\delta=10.4\%\)（NRuns=5；若 NRuns=7 可进一步提升至 7.1%）。
  - 随着 t 增大，防护效果下降，但实测 ASR 始终不高于相应的证书边界。
- **端到端真实攻击**：当攻击完全由智能体自身通过查询交互写入毒记忆时，SMSR 将 ASR 从 65.3% 降至 5.3%（n=150，置信区间不重叠），印证了防御在真实堆栈上的有效性。
- **一致少数效应**：基于字符串的多数投票在 12/15 场景中以少胜多，恶意回答因文本一致而获胜；判决投票消除了该效应。
- **效用代价**：组件 1 不影响效用（仍为 90%）；组件 2 因随机消融导致效用下降到 85%，为 5% 的折损。
- **跨模型泛化**：Sonnet 与 Haiku 在组件 2 下获得完全相同的 8.0% ASR，说明防御机制是模型无关的。
- **与 A‑MemGuard 对比**：在同等 20‑seed 存储下，A‑MemGuard 的 ASR 为 3.8% (CI [2.4, 6.0])，SMSR 为 8.0% (CI [5.8, 10.9])，置信区间重叠，表明两者经验表现可比，但 **SMSR 提供了形式化认证边界**，这是关键区分点。

### 7. 优点

- **首个认证防御**：为动态、运行时记忆投毒提供数学上证明的 (t, δ) 安全保证，填补了领域空白。
- **威胁模型与现实贴合**：考虑攻击者完全通过正常交互注入，且注入的文本为逼真企业文本，非明显的对抗性字符串。
- **理论贡献丰富**：
  - 证明了无来源证明的检索时过滤器不可能认证（Theorem 1），有力论证了来源机制的必要性。
  - 推导了基于超几何分布的随机消融证书，并可推广到不同的 t、m、k、n 设置。
  - 形式化了“一致少数效应”，解释了简单字符串投票的脆弱性，并给出解决方案。
- **两组件解耦清晰**：组件 1 针对未授权注入提供强安全保证；组件 2 则用随机化和投票限制已认证攻击者的影响，二者可组合且对 LLM 模型透明。
- **实验充分**：重复次数多、统计报告完备（置信区间）、涵盖多种攻击类别和场景、跨模型验证、端到端测试，增强了结果的可信度。
- **实际可部署性**：以包装器形式运行，不修改底层 LLM 或嵌入模型，引入的计算开销（约 10 次 API 调用/查询）在大多数企业应用中可接受。

### 8. 不足与局限

- **证书在 t 较大时松弛**：当 t 接近或超过 m/2 时，δ≈1，防御无法提供有效保证。论文已指明需要运营商根据预期对手能力合理设置 m（经验法则 m≈19t 以获得 δ≤0.10），但这在实际中存在预估 t 的困难。
- **对自适应对手的考虑有限**：论文讨论了裁判模型可能被针对性地优化以通过判定，但目前仅通过轮换裁判或提示词作为缓解手段，未提供正式的抗自适应证明。
- **评估环境的抽象性**：所有实验基于合成企业场景（Nexora Corp）和 LLM 裁判，虽然裁判间一致度极高，但缺少真实人工标注，且复杂企业记忆条目的边界可能更模糊，裁判的鲁棒性仍需进一步验证。
- **存储和签名基础设施**：部署需确保 HMAC 密钥安全存储于 HSM 或密钥管理服务中，所有写入必须经过签名预言机，这增加了系统复杂度以及密钥轮换时的历史记忆失效问题。
