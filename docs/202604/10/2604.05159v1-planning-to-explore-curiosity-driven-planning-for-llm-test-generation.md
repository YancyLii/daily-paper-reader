## 摘要
大语言模型（LLM）在代码生成方面的应用已自然地扩展到代码测试与评估。随着代码库规模和复杂性的增加，对自动化测试生成的需求也随之增长。目前基于 LLM 的测试生成方法依赖于最大化即时覆盖率增益的策略，这种贪婪方法在处理需要多个设置步骤（且单个步骤不产生新覆盖率）才能到达深层分支的代码时会陷入瓶颈。借鉴贝叶斯探索的原理，我们将程序的分支结构视为未知环境，并将不断演化的覆盖率图谱视为代表 LLM 迄今为止所发现内容的代理概率后验。我们的方法 CovQValue 将覆盖率图谱反馈给 LLM，并行生成多样的候选计划，并通过 LLM 估计的 Q 值选择信息量最大的计划，旨在寻求平衡即时分支发现与未来可达性的行动。我们的方法在 TestGenEval Lite 上的表现优于贪婪选择，在三种主流 LLM 中实现了 51-77% 的分支覆盖率提升，并在 77-84% 的目标上胜出。此外，我们构建了一个用于迭代测试生成的基准测试集 RepoExploreBench，在该基准上达到了 40-74% 的覆盖率。这些结果展示了好奇心驱动的规划方法在基于 LLM 的探索中的潜力，能够通过顺序交互更有效地发现程序行为。

## Abstract
The use of LLMs for code generation has naturally extended to code testing and evaluation. As codebases grow in size and complexity, so does the need for automated test generation. Current approaches for LLM-based test generation rely on strategies that maximize immediate coverage gain, a greedy approach that plateaus on code where reaching deep branches requires setup steps that individually yield zero new coverage. Drawing on principles of Bayesian exploration, we treat the program's branch structure as an unknown environment, and an evolving coverage map as a proxy probabilistic posterior representing what the LLM has discovered so far. Our method, CovQValue, feeds the coverage map back to the LLM, generates diverse candidate plans in parallel, and selects the most informative plan by LLM-estimated Q-values, seeking actions that balance immediate branch discovery with future reachability. Our method outperforms greedy selection on TestGenEval Lite, achieving 51-77% higher branch coverage across three popular LLMs and winning on 77-84% of targets. In addition, we build a benchmark for iterative test generation, RepoExploreBench, where they achieve 40-74%. These results show the potential of curiosity-driven planning methods for LLM-based exploration, enabling more effective discovery of program behavior through sequential interaction

### 1. 核心问题与整体含义
*   **研究动机**：现有的基于大语言模型（LLM）的自动化测试生成方法大多采用“贪婪策略”，即只追求下一步能立即增加的覆盖率。
*   **核心挑战**：在复杂程序中，到达深层代码分支往往需要一系列前置设置步骤（如类初始化、复杂的输入验证等）。这些中间步骤本身可能不产生任何新覆盖率，导致贪婪算法陷入局部最优（覆盖率平台期）。
*   **研究意义**：本文将测试生成视为对未知环境的“贝叶斯探索”，引入“好奇心”驱动的规划机制，使 LLM 能够为了长远的覆盖率增益而执行暂时没有覆盖率回报的行动。

### 2. 方法论（CovQValue）
*   **核心思想**：借鉴强化学习中的好奇心理论，将程序分支结构视为未知环境，将已有的覆盖率地图（Coverage Map）作为探索状态的后验概率代理。
*   **关键技术细节**：
    *   **覆盖率地图反馈**：将已发现的分支、最近的发现率、停滞警告等结构化信息反馈给 LLM，作为其决策的依据。
    *   **多样化计划生成**：LLM 并行生成 $K$ 个候选测试计划（每个计划包含 $S$ 个连续脚本），并使用不同的“多样性提示词”（如侧重主功能、错误处理或交互逻辑）。
    *   **好奇心 Q 值评分**：利用 LLM 作为启发式评估器，为每个计划估算 Q 值：$Q = \hat{g} + \gamma \cdot \hat{v}$。
        *   $\hat{g}$（即时增益）：预计该计划能发现的新分支数量。
        *   $\hat{v}$（未来价值）：该计划执行后，能为后续测试开辟的潜在可达路径（如通过了某个验证门槛）。
        *   $\gamma$：折扣因子，平衡即时回报与长期潜力。
    *   **循环迭代**：选择 Q 值最高的计划执行，更新覆盖率地图，进入下一轮。

*   **新基准贡献**：提出的 RepoExploreBench 填补了现有基准在“迭代探索”和“复杂依赖”评估上的空白。
*   **即插即用**：作为一种推理侧的规划策略，不需要对 LLM 进行微调，具有很强的通用性。

### 8. 不足与局限
*   **通过率权衡（Pass Rate Trade-off）**：为了探索未知的深层代码，CovQValue 生成的测试脚本往往更复杂、更具挑战性，导致其测试通过率（Pass Rate）低于贪婪方法（约低 10-20%）。
*   **评估器偏差**：Q 值的估算完全依赖于 LLM 的内部知识，如果模型对特定库的逻辑理解有误，可能会导致评分失效。
*   **推理开销**：相比单步生成，并行生成多个计划并评分会消耗更多的 Token 和推理时间。
