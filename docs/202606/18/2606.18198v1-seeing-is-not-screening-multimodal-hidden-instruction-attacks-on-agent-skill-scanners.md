## 摘要
智能体技能正成为基于大语言模型系统中的一个重要攻击面。通过对现有技能扫描器的实证研究，我们发现当前防御主要依赖文本描述、清单和源代码作为安全分析的主要信号，这可能导致视觉传达的恶意意图未被充分检测。这形成了一个实际盲点：隐藏在图像中的有害操作指令可能在扫描时被绕过，但在部署时仍可被多模态智能体恢复。为系统研究这一威胁，我们提出 SkillCamo，一种文档介导的多模态指令攻击，将恶意指令隐藏在随技能打包的图像中，同时重写周边文档，使这些图像被自然引用为正常工作流的一部分。因此，该攻击并非仅依赖图像，而是依赖执行时文本指引与视觉载荷的联合解读。为防御此类攻击，我们进一步提出 ExecScan，一个基于执行的多模态扫描模块，对技能制品进行意图提取、行为重构、滥用评估和审慎执行模拟。ExecScan 联合分析文档、代码、引用资源和视觉内容，以恢复隐藏指令、重构可执行行为链，并识别数据窃取、破坏、持久化、欺骗和权限提升等下游风险。大量实验表明，图像隐藏的恶意指令对现有技能扫描器构成挑战，而 ExecScan 能够提升技能扫描性能。

## Abstract
Agent skills are emerging as an important attack surface in LLM-based systems. Through an empirical study of existing skill scanners, we find that current defenses primarily rely on textual descriptions, manifests, and source code as the main signals for security analysis, which can leave visually conveyed malicious intent insufficiently examined. This creates a practical blind spot: harmful operational instructions hidden in images may bypass scanning while still being recoverable by multimodal agents during deployment. To systematically investigate this threat, we propose SkillCamo, a document-mediated multimodal instruction attack that conceals malicious instructions within images bundled with a skill while rewriting the surrounding documentation to naturally reference those images as part of the normal workflow. Thus, the attack does not rely on the image alone, but on the joint interpretation of textual guidance and visual payload at execution time. To defend against such attacks, we further propose ExecScan, an execution-grounded multimodal scanning module that performs intent extraction, behavior reconstruction, abuse assessment, and deliberative execution simulation over skill artifacts. ExecScan jointly analyzes documentation, code, referenced resources, and visual content to recover hidden instructions, reconstruct executable behavior chains, and identify downstream risks such as exfiltration, destruction, persistence, deception, and privilege escalation. Extensive experiments show that image-hidden malicious instructions challenge existing skill scanners, while ExecScan can improve the skill scanning performance.

- **研究背景**：大语言模型（LLM）智能体越来越多地通过外部“技能（skills）”扩展能力，技能通常包含文档、脚本、图片等资源。然而，技能自身的安全性正成为一个关键攻击面。
- **核心盲点**：现有技能扫描器（如 Cisco Skill Scanner、Snyk Agent Scan 等）主要依赖文本描述、元数据和源代码进行安全分析，对图片等视觉资源中的恶意指令缺乏有效检查。在实际部署中，多模态智能体能够联合理解文本和图像，从而还原并执行隐藏的恶意操作，形成一条被忽视的攻击通道。
- **整体含义**：该研究揭示了“所见非所检”的安全鸿沟——技能在静态文本层面看似良性，但在多模态执行时可能触发数据窃取、持久化等恶意行为。作者通过设计攻击与防御框架，系统论证了这一威胁并提出了针对性缓解方案。

### 2. 方法论：核心思想与关键技术

#### 攻击方法：SkillCamo
- **核心思想**：将恶意操作指令（如窃取凭证、建立后门）隐藏在技能包内的图像资源中，同时重写技能文档（SKILL.md），使图像被自然地描述为正常工作流的一部分（如设置图、使用示例），从而在不引起文本扫描器警觉的前提下，利用多模态智能体的图文联合理解能力实施攻击。
- **技术流程（四阶段迭代）**：
  1. **技能上下文分析**：解析原始 SKILL.md，提取技能目的、章节层次和布局线索，指导后续重写以保持风格一致性。
  2. **图像载体构造**：从恶意脚本中提取目标命令 `c`，根据所选策略（全部隐藏 FULL、填空隐藏 CLOZE、分割隐藏 SPLIT）将其渲染为视觉载体 `˜I`，以看似正常的图表形式嵌入技能资源中。
  3. **文档语义重写**：使用大模型重写整个 SKILL.md，在保持原始技能有用性的前提下，将 `˜I` 的图像引用以一种自然、无害的方式融入文档（如“配置参考图”），且避免在文本中暴露真实指令。
  4. **扫描器引导的迭代精化**：将生成的技能提交给目标扫描器，若被检出则将其反馈压缩为改写指导，循环优化直到通过扫描，从而逐步消除表面可疑信号。
- **隐藏策略变体**：SkillCamo（全部隐藏在图像）、SkillCamo-Cloze（图像仅存放关键词，文档中以填空形式引导智能体补全）、SkillCamo-Split（指令片段分别放在图像和文档中，需跨模态拼接恢复）。

#### 防御方法：ExecScan
- **核心思想**：跳出静态制品检查，模拟多模态智能体在实际任务中对技能的完整理解与执行过程，从而暴露仅依赖文本分析无法发现的隐藏指令和风险行为链。
- **两阶段框架**：
  1. **多模态制品分析**：同时分析文档、代码、配置、图片等所有资源，利用多模态大模型 `LA` 提取 **意图概貌**（声明用途、访问范围）、**行为概貌**（文件读写、进程调用、网络访问）以及 **滥用概貌**（评估数据窃取、破坏、持久化等初步风险）。
  2. **审慎执行模拟**：
     - **上下文构建**：基于意图概貌，生成 `K` 个贴近真实使用的任务场景。
     - **指令恢复**：在每个场景下，联合文本与视觉信息，恢复 **表面指令**（文档、代码中直接可见）和 **潜在指令**（仅能通过图像或跨模态推理获得），得到完整指令集 `ˆI_k`。
     - **执行链投影**：根据恢复的指令和场景，推演智能体可能执行的完整操作链 `E_k`（如读取敏感文件→通过子进程发送至外部服务器）。
     - **风险判断**：分别计算仅基于表面指令的风险 `r_surface` 与基于完整指令（含视觉）的风险 `r_exec`，通过两者的差异识别由隐藏指令引入的额外威胁。
- **最终评分**：取所有模拟执行路径中的最高风险 `max r_exec_k` 作为技能的恶意分数，与阈值比较后给出判定。

### 6. 主要结论与发现

- **攻击有效性**：SkillCamo 及其变体对现有扫描器取得了高达 78–100% 的 ASR，而基准文本攻击 SkillJect 的 ASR 为 0% 至 90%，说明图像隐藏是一种高度有效的绕过手法。
- **防御有效性**：ExecScan 将 SkillCamo 的 ASR 压低至 8%（Cloze 变体 31%，Split 变体 17%），同时保持了较低的误报率（27.4%）和最高的 F1 分数（83.8%），显著优于其他所有扫描器。
- **盲点根因**：显式添加图像警告提示虽能少量改善某些扫描器，但无法根本解决问题，甚至可能因注意力转移而增加误判，证实了必须从架构上引入多模态、执行驱动的分析。
- **可转移性**：SkillCamo 生成的对抗性技能在不同扫描器间具有较强的迁移性，但 ExecScan 对其几乎完全免疫，表明 ExecScan 捕捉到了与现有扫描器本质不同的安全信号。

### 7. 优点

- **新颖的攻击范式**：首次系统提出并实现了在技能图片中隐藏指令以绕过文本扫描的攻击方法，并通过多模态文档重写和迭代优化使其高度隐蔽。
- **系统化的防御设计**：ExecScan 将静态分析、执行模拟与跨模态推理有机整合，从“技能包含什么”转向“技能会做什么”，填补了现有工具的盲区。
- **全面的实验验证**：覆盖多种扫描器和攻击变体，包含消融、转移性、统计检验和鲁棒性分析，结论扎实。
- **透明度高**：在附录中提供了完整的提示词集，有利于复现和后续研究。

### 8. 不足与局限

- **计算开销**：ExecScan 需要进行多次 LLM 调用（多模态分析 + K 次执行模拟），成本显著高于传统文本扫描器，更适合作为二级深度审查工具，而非轻量级过滤。
- **模型依赖性**：攻击和防御的性能受限于底层大模型的能力，当多模态模型演化时，ASR 和检测率可能发生变化。
- **威胁模型限定于预部署扫描**：ExecScan 未包含运行时沙箱、权限限制或来源校验等机制，实际生产应用仍需与其他安全措施配合。
- **攻击模板和技能来源有限**：恶意指令池和技能数据源较为固定，未来研究需在更广泛多样的生态系统中验证通用性。
- **对抗适应性未充分探索**：攻击者可能通过在图像中进一步混淆指令或混合更复杂的文档操纵来对抗 ExecScan，论文对此尚未深入分析。
