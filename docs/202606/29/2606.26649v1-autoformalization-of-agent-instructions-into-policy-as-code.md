## 摘要
在高风险领域，代理安全需要正式的策略执行，但现有大多数方法要么依赖于概率性护栏（微调分类器、基于提示的引导），无法提供正式保证，要么依赖于手工编码的符号执行，无法扩展到实际策略规范的广度。我们提出一个自动形式化流水线，利用基于大型语言模型的生成-批评循环，将代理提示、MCP工具描述和自然语言策略文档转换为经过形式验证的策略。生成的策略采用Cedar策略语言编写。在MedAgentBench基准测试中，我们的自动形式化策略比先前工作中手工编码的符号执行覆盖了更多的源自然语言规范。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

## 2. 方法论：验证三明治架构与生成‑批评循环
整个流水线称为 **验证三明治**（Verification Sandwich），由三层组成，并采用基于大语言模型的**生成‑批评循环**迭代精化策略。

*   **接地层（底层）**：
    *   从系统提示、MCP工具定义中提取实体、工具模式，构建主体‑资源‑动作本体。
    *   自动生成 Cedar 策略语言所需的 **Schema**，为后续策略生成提供类型约束。

*   **模型层（中层）**：
    *   利用 LLM（实验中为 Gemini 3 Pro）作为**策略生成器**，输入代理的系统指令、工具定义和自然语言政策文档，输出候选的 Cedar 策略集合。

*   **安全层（顶层）**：
    *   **硬批评器**：基于 Cedar 内置的静态分析工具，执行确定性检查，包括语法正确性、Schema 合规性、逻辑矛盾冲突、空策略检测。
    *   **软批评器**：以 LLM‑as‑Judge（实验中为 Gemini 2.5 Flash）评估候选策略对原始自然语言的语义对齐度，并按评分标准定性质检。
    *   **迭代回馈**：硬批评器和软批评器的反馈被送回生成器，循环改进直到策略集通过双重检查，输出最终的 Cedar 策略集。

*   **运行时执行**：
    *   生成的 Cedar 策略被部署到外部确定性策略引擎中，在代理执行工具调用时进行实时授权决策，实现安全策略与 LLM 推理的**解耦**。

*   **解耦提供更强安全**：将策略实施从 LLM 上下文窗口中移出，交由确定性引擎处理，使得攻击者无法通过越狱或间接提示注入“说服”安全层忽略规则。
*   **形式验证的价值**：Cedar 语言的可读性和可审计性，以及硬批评器提供的类型安全和逻辑无矛盾性，是纯神经方法所不具备的。

## 8. 不足与局限
*   **领域单一**：仅在 MedAgentBench（医疗电子病历）领域进行了评估，方法在其他高风险领域（如金融、法律、自主驾驶）的泛化能力未经验证。
*   **语言与引擎限制**：
    *   Cedar 是**无状态**策略语言，难以原生表达依赖操作顺序或持久上下文的代理策略。
    *   实验重放依赖于一个额外的“SessionContext 提供器”来模拟状态，真实部署中状态同步是挑战。
*   **策略覆盖与正确性**：
    *   自动生成的策略虽然覆盖率更高，但仍**不能保证 100% 捕捉或完全正确**，软批评器只能给出评估分数，无法逻辑证明语义完备。
    *   部分策略条款（如关于代理输出风格的条款）被明确标记为“分类器形状”，无法用 Cedar 表达，说明方法仍有**覆盖盲区**。
*   **生成质量依赖**：策略生成完全依赖底层 LLM 的能力，论文未深入讨论生成失败、幻觉实体或策略过于严苛导致代理完全瘫痪的风险。
