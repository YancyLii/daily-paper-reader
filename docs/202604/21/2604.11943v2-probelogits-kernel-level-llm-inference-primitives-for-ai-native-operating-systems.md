## 摘要
内部运行大语言模型（LLM）推理的操作系统内核可以在生成任何文本之前读取 logit 分布，并将其作为治理原语进行操作。本文提出了 ProbeLogits，这是一种内核级操作，通过执行单次前向传递并读取特定 token 的 logit，将智能体行为分类为安全或危险，且无需任何学习参数。我在三个外部基准测试（HarmBench、XSTest 和 ToxicChat）上，针对三种基础模型（Qwen 2.5-7B、Llama 3 8B、Mistral 7B）对 ProbeLogits 进行了评估。在 HarmBench 非版权数据集（n=300）上，配合适当的 verbalizer，所有三种模型均达到了 97-99% 的拦截率。在 ToxicChat（n=1,000）上，ProbeLogits 在相同托管环境下的 F1 分数达到或优于 Llama Guard 3 (LG3)：最强配置（Qwen 2.5-7B Safe/Dangerous, alpha=0.0）的 F1 分数达到 0.812，其自助法 95% 置信区间（CI）与 LG3 不重叠（显著提升 13.7 个百分点）；Llama 3 S/D 在置信区间内与 LG3 持平（+0.4 个百分点）；Mistral Y/N 则高出 4.4 个百分点。由于该原语读取单个 logit 位置而非生成 token，其在相同托管环境下的延迟比 LG3 快约 2.5 倍；在裸机原生运行时中，ProbeLogits 的延迟降至 65 毫秒。一个关键的设计贡献是校准强度 alpha，它作为部署时的策略调节旋钮，而非学习到的超参数。上下文校准修正了 verbalizer 的先验不对称性，偏置大小随（模型, verbalizer）对的不同而变化。我在 Anima OS 中实现了 ProbeLogits，这是一个由约 86,000 行 Rust 代码编写的裸机 x86_64 操作系统。由于智能体行为必须通过 15 个内核介导的宿主函数，ProbeLogits 的强制执行运行在 WASM 沙箱边界之下，使其比应用层分类器更难被规避。

## Abstract
An OS kernel that runs LLM inference internally can read logit distributions before any text is generated and act on them as a governance primitive. This paper presents ProbeLogits, a kernel-level operation that performs a single forward pass and reads specific token logits to classify agent actions as safe or dangerous, with zero learned parameters.   I evaluate ProbeLogits across three base models (Qwen 2.5-7B, Llama 3 8B, Mistral 7B) on three external benchmarks: HarmBench, XSTest, and ToxicChat. On HarmBench non-copyright (n=300), all three models reach 97-99% block rate with the right verbalizer. On ToxicChat (n=1,000), ProbeLogits achieves F1 parity-or-better against Llama Guard 3 in the same hosted environment: the strongest configuration (Qwen 2.5-7B Safe/Dangerous, alpha=0.0) reaches F1=0.812 with bootstrap 95% CIs disjoint from LG3 (+13.7pp significant); Llama 3 S/D matches LG3 within CI (+0.4pp, parity); Mistral Y/N exceeds by +4.4pp. Latency is approximately 2.5x faster than LG3 in the same hosted environment because the primitive reads a single logit position instead of generating tokens; in the bare-metal native runtime ProbeLogits drops to 65 ms.   A key design contribution is the calibration strength alpha, which serves as a deployment-time policy knob rather than a learned hyperparameter. Contextual calibration corrects verbalizer prior asymmetry, with bias magnitude varying by (model, verbalizer) pair.   I implement ProbeLogits within Anima OS, a bare-metal x86_64 OS written in approximately 86,000 lines of Rust. Because agent actions must pass through 15 kernel-mediated host functions, ProbeLogits enforcement operates below the WASM sandbox boundary, making it significantly harder to circumvent than application-layer classifiers.

### 1. 核心问题与研究动机
*   **核心问题**：传统的操作系统（OS）将进程视为不透明的实体，仅管理资源而不干预其内部逻辑。但在 AI 原生时代，大语言模型（LLM）的推理过程产生了丰富的语义信号（如 Logit 分布），现有的系统架构将安全治理放在应用层，导致了**高延迟**（需生成文本再解析）、**易被绕过**（治理边界高于智能体权限）以及**语义不透明**（内核无法感知模型意图）三大问题。
*   **研究背景**：作者提出在 OS 内核中直接运行 LLM 推理，利用模型在生成文本前的 Logit 分布作为治理原语，从而实现更深层次、更高效的 AI 治理。

### 2. 方法论：核心思想与技术细节
*   **ProbeLogits 原语**：
    *   **核心思想**：不进行完整的文本生成，而是执行单次前向传播（Forward Pass），直接读取特定 Token（如“Yes/No”或“Safe/Dangerous”）位置的 Logit 值。
    *   **分类机制**：通过对目标 Token 集合进行受限的 Softmax 计算，得出分类概率。这种方法将分类成本从“多步生成”降低为“单步探测”。
    *   **上下文校准（Contextual Calibration）**：引入校准强度参数 $\alpha$。通过在启动时探测空输入下的模型偏置，修正模型对特定词汇（如“Safe”）的固有偏好。$\alpha$ 作为一个策略旋钮，允许 OS 在部署时灵活权衡查准率（Precision）和查全率（Recall）。
*   **KV Cache 作为进程状态**：
    *   将 LLM 的 KV Cache 类比为传统 CPU 的寄存器状态。
    *   实现了 `kv_checkpoint`（快照）、`kv_restore`（恢复）和 `kv_fork`（派生）操作，支持智能体的上下文切换和分支执行。
*   **内核级强制执行**：
    *   在 **Anima OS**（一个用 8.6 万行 Rust 编写的裸机操作系统）中实现。
    *   治理逻辑运行在 WASM 沙箱边界之下。智能体必须通过内核介导的宿主函数（Host Functions）才能执行操作，确保了治理机制的不可绕过性。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了三个主流模型家族和三个权威安全基准。

### 6. 主要结论与发现
*   **性能卓越**：在裸机环境下，7B 模型的 ProbeLogits 分类延迟仅为 **65ms**，比传统的生成式分类快约 10 倍。
*   **治理效能**：在 ToxicChat 上，基于 Qwen 2.5-7B 的 ProbeLogits（$\alpha=0.0$）F1 分数达到 0.812，显著优于经过微调的 Llama Guard 3（F1=0.675）。
*   **硬件极限**：实验证明 7B 模型的推理速度受限于内存带宽（DDR5 瓶颈），裸机运行已达到理论带宽极限的 85%。
*   **架构优势**：通过将治理置于内核层，实现了“防御深度”属性，即使智能体尝试绕过应用层过滤器，也无法逃脱内核的 Logit 探测。

### 7. 优点与亮点
*   **零参数治理**：无需昂贵的微调，直接利用预训练模型的能力。
*   **极低延迟**：单次前向传播即可完成决策，适合每步操作（Per-action）的实时治理。
*   **系统级创新**：首次将 Logit 向量和 KV Cache 提升为操作系统内核原语，为“AI 原生 OS”提供了清晰的蓝图。
*   **高安全性**：运行在 WASM 沙箱之下，提供了比应用层插件更强的安全保障。

### 8. 不足与局限
*   **GPU 加速缺失**：目前仅支持 CPU，对于更大规模的模型（如 70B+）性能将无法满足实时性要求。
*   **Token 敏感性**：依赖于特定 Token 的 Logit。如果模型的分词器（如 Mistral 的 SentencePiece）将标签词切分为多个 Token，该方法将失效。
*   **校准依赖**：$\alpha$ 参数的选取目前带有一定的后验性（针对特定基准调整），未来需要更自动化的校准机制。
*   **验证范围**：虽然在安全基准上表现良好，但在更复杂的逻辑推理或多步骤治理场景下的表现尚待验证。
