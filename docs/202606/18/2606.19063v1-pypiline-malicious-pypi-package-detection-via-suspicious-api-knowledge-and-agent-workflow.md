## 摘要
检测恶意PyPI包对于维护开源软件供应链的安全至关重要。现有的方法主要依赖规则或传统机器学习，存在可解释性差、难以适应新型攻击的问题。为此，我们提出PYPILINE，一种结合可疑API知识库与智能体（Agent）工作流的新型检测方法。PYPILINE首先对已知恶意包进行静态分析，提取抽象语法树并生成API调用图，从中自动提取并构建结构化的可疑API知识库。在检测阶段，该知识库用于增强推理能力。通过智能体工作流，PYPILINE对未知包进行深度语义分析，并输出结构化、可解释的恶意性评估报告。实验结果表明，PYPILINE在精确率96.7%、召回率99.6%和F1值98.1%上显著优于现有最先进的工具，其精确率超过基线工具5.7至24.2个百分点。此外，我们对恶意包进行了实证研究，系统揭示了流行的攻击策略以及最常被滥用的API。PYPILINE配备了工具调用的AI智能体工作流，用于自动向量数据库检索可疑API知识以及邮件服务器发送分析报告，提供了一个实用、高效、便捷的恶意包检测解决方案，以加强开源生态系统的安全。

## Abstract
The detection of malicious PyPI packages is crucial for maintaining the security of the open source software supply chain. Existing methods, which primarily rely on rules or traditional machine learning, suffer from poor interpretability and difficulty in adapting to novel attacks. To address this, we propose PYPILINE, a novel detection method that combines a suspicious API knowledge base with an Agent workflow. PYPILINE first conducts static analysis on known malicious packages, extracting abstract syntax trees and generating API call graphs, from which it automatically extracts and constructs a structured suspicious API knowledge base. During the detection phase, this knowledge base is used to enhance reasoning capabilities. Through an Agent workflow, PYPILINE performs in depth semantic analysis of unknown packages and outputs a structured, interpretable maliciousness assessment report. The experimental results show that PYPILINE significantly outperforms existing state-of-the-art tools in precision of 96.7\%, recall of 99.6\%, and F1-score of 98.1\%, with its precision surpassing baseline tools by 5.7 to 24.2 percentage points. Additionally, we conducted an empirical study on malicious packages, systematically revealing prevalent attack strategies, as well as the most commonly abused APIs. Equipped with tool-calling AI agent workflows for automated vector database retrieval of suspicious API knowledge and mail server delivery of analysis reports, PYPILINE delivers a practical, efficient, and convenient malicious package detection solution to strengthen open-source ecosystem security.

## 1. 核心问题与研究动机
- **背景与挑战**：开源软件供应链严重依赖 PyPI 等公共仓库，而恶意包数量激增（仅 2023 年就超过 24.5 万个）。攻击者通过域名仿冒、依赖混淆、账户劫持等手段，在安装、导入或运行时触发恶意代码，造成数据泄露或系统失陷。
- **现有方法缺陷**：
  - 基于规则的方法（如 Guarddog、OSSGadget）可解释性差、扩展性弱，难以适应快速演变的攻击。
  - 传统机器学习方法（如 AMALFI、Maloss）决策过程“黑箱”且严重依赖特征工程，对新型攻击泛化能力不足。
  - 动态分析方法开销高，难以大规模快速筛查。
- **研究目标**：提出一种兼具高检测精度、强可解释性、可自动化且能适应新型攻击的恶意 PyPI 包检测方法。

## 2. 方法论
### 核心思想
结合**可疑 API 知识库**与大语言模型（LLM）驱动的**智能体工作流**。离线阶段从已知恶意包中自动提取 API 行为模式并构建知识库；在线阶段将该知识库作为上下文注入 LLM，通过 Agent 对目标包进行深度语义分析与结构化报告生成。

### 关键技术细节
- **离线—知识库构建**：
  - **AST 解析**：对已知恶意包的所有 `.py` 文件进行静态分析，生成抽象语法树。
  - **API 调用图生成**：根据函数调用关系构建有向图 \(G=(V,E)\)，节点为 API，边为调用依赖（考虑显式调用与文件、网络等副作用引发的隐式依赖）。
  - **可疑 API 筛选**：计算每个 API 节点的度中心性，统计其在恶意样本中的全局重要性，结合安全领域先验知识，人工确认并保留 **300 个最可疑的 API**，形成结构化知识库（API 名称 + 恶意行为描述）。
- **在线—智能体检测工作流**：
  - **代码拼接**：将待检包的所有 `.py` 文件按路径顺序拼接为单一文本文档，保留文件名作为段落标题。
  - **Prompt 构建**：定义角色为网络安全专家，注入可疑 API 知识库片段，要求模型分析代码并输出 JSON 格式的恶意性评估报告。
  - **Agent 工具调用**：Agent 可调用**向量数据库**检索高危 API 特征辅助推理，并通过**邮件服务器**自动发送最终报告。
  - **输出格式**：结构化 JSON，包含 `is_malicious`（布尔值）、`confidence`（置信度）、`mal_types`（8 类恶意行为标签，如安装时代码执行、混淆、敏感数据窃取等）、`api_calls`（触发的恶意 API 列表）和 `function`（行为简述）。
- **分类体系**：将恶意行为归纳为 **8 类**：安装时代码执行、混淆、Base64 动态执行、远程可执行文件下载、DLL 劫持、可疑链接、敏感数据窃取、安装命令覆写。

## 6. 主要结论与发现
- **检测性能卓越**：PYPILINE 精确率 96.7%、召回率 99.6%、F1 值 98.1%，精确率较次优工具 MalGuard 高出 5.7 个百分点，较其他工具高 5.7–24.2 个百分点，有效平衡误报与漏报。
- **组件贡献明确**：

- **高可解释性**：输出结构化 JSON 并提供自然语言行为总结与关键代码定位，解决了传统 ML“黑箱”问题，便于安全分析师审计。
- **检测效果突出**：在 F1 和精确率上均大幅超越现有规则/ML 工具，且实验覆盖 23,000+ 样本，结果可信度高。

## 8. 不足与局限
- **模型依赖与成本**：强依赖商用高性能 LLM（如 GPT‑5），API 调用成本高；开源或小型模型性能不足，平衡性能与成本的难度大。
- **输出格式可靠性**：LLM 可能生成不符合 JSON 规范的“格式幻觉”，需额外清洗步骤，影响全自动链路的稳定性。
- **模型幻觉风险**：LLM 可能“虚构”攻击链、误判良性高权限操作为恶意，或输出预定义类别之外的标签，存在假阳性隐患。
- **专为 Python/PyPI 设计**：当前方法仅适用于 Python 生态，对其他语言（如 JavaScript/NPM）需重新构建知识库并调整分析流程，跨语言泛化能力未验证。
- **对抗样本隐忧**：面对专门针对 LLM 或知识库的逃逸攻击（如刻意避开已知可疑 API），方法可能被绕过，文中未讨论鲁棒性测试。
- **知识库构建仍含人工干预**：可疑 API 的最终筛选依赖领域专家确认，虽提升了质量，但自动化程度未达极致。
- **动态行为缺失**：纯静态分析可能遗漏运行时解密的恶意逻辑或条件触发的隐藏行为。
