## 摘要
机构决策——如合规性监管、临床分诊、预授权申诉——需要一种与通用智能体提供的架构不同的 AI 架构。智能体框架通过对话推断权限，从日志中重构问责机制，并产生“静默错误”：即在没有任何人工审核信号的情况下执行的错误判定。我们提出了 Cognitive Core：这是一个受控决策底层框架，由九种类型化的认知原语（检索、分类、调查、验证、质疑、反思、审议、管理、生成）构建而成；它包含一个四层治理模型，其中人工审核是执行的前提条件而非事后检查；一个内生于计算过程的防篡改 SHA-256 哈希链审计账本；以及一个支持声明式和自主推理认知序列的需求驱动委派架构。我们在一个包含 11 个案例的平衡预授权申诉评估集上对三个系统进行了基准测试。Cognitive Core 达到了 91% 的准确率，而 ReAct 为 55%，Plan-and-Solve 为 45%。治理结果更为显著：CC 产生的静默错误为零，而两个基准系统均产生了 5-6 个。我们引入了“可治理性”（governability）——即系统能够可靠地识别何时不应自主行动的能力——作为机构级 AI 除准确率之外的主要评估维度。基准系统以提示词形式实现，代表了受控框架之外的现实部署替代方案。配置驱动的领域模型意味着部署新的机构决策领域仅需 YAML 配置，而无需额外的工程开发能力。

## Abstract
Institutional decisions -- regulatory compliance, clinical triage, prior authorization appeal -- require a different AI architecture than general-purpose agents provide. Agent frameworks infer authority conversationally, reconstruct accountability from logs, and produce silent errors: incorrect determinations that execute without any human review signal. We propose Cognitive Core: a governed decision substrate built from nine typed cognitive primitives (retrieve, classify, investigate, verify, challenge, reflect, deliberate, govern, generate), a four-tier governance model where human review is a condition of execution rather than a post-hoc check, a tamper-evident SHA-256 hash-chain audit ledger endogenous to computation, and a demand-driven delegation architecture supporting both declared and autonomously reasoned epistemic sequences.   We benchmark three systems on an 11-case balanced prior authorization appeal evaluation set. Cognitive Core achieves 91% accuracy against 55% (ReAct) and 45% (Plan-and-Solve). The governance result is more significant: CC produced zero silent errors while both baselines produced 5-6. We introduce governability -- how reliably a system knows when it should not act autonomously -- as a primary evaluation axis for institutional AI alongside accuracy. The baselines are implemented as prompts, representing the realistic deployment alternative to a governed framework. A configuration-driven domain model means deploying a new institutional decision domain requires YAML configuration, not engineering capacity.

### 1. 核心问题与整体含义
*   **研究动机**：当前的通用 AI 智能体（Agent）架构在处理机构决策时存在三大缺陷：权限边界模糊（通过对话推断而非结构化强制）、推理过程不可审计（推理链是事后重构的）、以及频繁产生“静默错误”（Silent Errors，即系统在没有发出任何人工审核信号的情况下执行了错误的决定）。
*   **核心挑战**：机构决策需要受控、可检查且持久的推理。论文指出，高准确率不足以建立机构信任，系统必须具备“可治理性”（Governability），即能够可靠地识别自己何时无法自主做出决定。

### 2. 方法论
论文提出了 **Cognitive Core** 架构，其核心思想是将推理过程分解为受控的原子操作：
*   **九种认知原语（Cognitive Primitives）**：包括检索（retrieve）、分类（classify）、调查（investigate）、验证（verify）、质疑（challenge）、反思（reflect）、审议（deliberate）、管理（govern）和生成（generate）。每个原语都有明确的输入输出契约。
*   **元认知反思（Metacognitive Reflection）**：引入 `reflect` 原语，它不直接处理案例证据，而是分析已有的推理状态。其关键功能是“质疑后守卫”（Post-challenge guard），用于区分合理的推理修正与因压力产生的“谄媚性妥协”（Sycophantic capitulation）。
*   **四层治理模型**：分为 AUTO（自动执行）、SPOT CHECK（抽检）、GATE（挂起等待人工审核）和 HOLD（阻塞）。治理层级锁定在执行基座中，不可被 LLM 绕过。
*   **内生审计账本**：使用 SHA-256 哈希链记录每一个原语操作、编排决策和治理动作，确保推理记录防篡改且与计算过程同步生成。
*   **三层认知状态**：结合机械信号（如引用率）、判断信号（LLM 自报信心）和相干性标志（跨步骤冲突检测）来计算最终的“授权得分”。

### 5. 实验数量与充分性
*   **实验规模**：核心基准测试针对 11 个平衡案例。此外，论文提到在 7 个不同的机构领域（如贷款修改、合规审查、临床分诊等）验证了原语的稳定性。
*   **充分性评价**：虽然 11 个案例的样本量较小，不足以进行大规模统计推断，但对于验证“结构化治理是否能消除静默错误”这一架构属性而言，实验设计是针对性强且客观的。通过平衡案例分布，有效排除了系统因“默认批准偏见”而获得高分的可能性。

### 6. 主要结论与发现
*   **性能表现**：CC 达到了 **91%** 的准确率（10/11），远高于 ReAct（55%）和 Plan-and-Solve（45%）。
*   **零静默错误**：CC 产生的静默错误为 **0**。即使在 CC 犯错的唯一案例中，系统也正确地将其路由到了 GATE（人工审核）环节。相比之下，基准系统产生了 5-6 个静默错误。
*   **偏见发现**：实验证明“批准优先偏见”（Approval Prior Bias）是单次推理系统的顽疾，即使通过 Plan-and-Solve 增加规划步骤也无法消除，必须依靠结构化的治理约束。

### 7. 优点
*   **架构安全性**：将治理作为执行的前提条件，而非事后监控，从底层解决了 Agent 的失控问题。
*   **透明度**：哈希链账本提供了极高的审计价值，符合监管对“步骤级可解释性”的要求。
*   **配置驱动**：通过 YAML 配置即可部署新领域，降低了机构应用 AI 的工程门槛。
*   **元认知机制**：`reflect` 原语有效解决了多智能体协作中常见的“少数意见被抑制”和“信息丢失”问题。

### 8. 不足与局限
*   **样本量限制**：11 个案例的基准测试规模较小，需要更大规模的数据集来验证数值准确率的普适性。
*   **原语完备性**：九种原语是否足以覆盖所有机构决策场景尚未得到形式化证明。
*   **性能开销**：由于引入了多次 LLM 调用（质疑、反思、治理等），其推理成本和延迟会高于简单的单次调用系统。
*   **扩展性**：当前的线程模型在处理极高并发任务时可能存在瓶颈，未来需向异步执行引擎演进。
