## 摘要
智能体系统越来越多地通过第三方技能执行动作，使模型生成的决策能够影响文件、通信渠道以及信息物理设备。这些技能通常包含自然语言规约，定义了访问权限、披露限制、执行特权和所需的前提条件。尽管此类规约描述了技能行为的预期边界，但它们本身并不提供可执行的运行时强制实施。对它们进行强制实施面临着一个情境粒度挑战：即使策略是针对特定任务情境编写的，监视器仍必须决定观察哪些事件、保留哪些状态、在多大执行范围内进行推理，以及在何处进行干预。选择错误的粒度要么会阻止良性执行，要么会遗漏仅跨多个动作才显现的违规行为。然而，大多数现有的强制实施机制都假定固定的时间模型或强制实施点。本文中，我们提出VIGIL，这是一个面向智能体系统的端到端运行时强制实施框架。VIGIL将智能体的实际执行轨迹与来自技能规约、操作员定义约束以及跨多个技能的全局规则的行为策略进行核对。为使此类策略可执行，VIGIL引入了一种策略语言，该语言能捕获关于智能体工具事件的特定情境强制实施需求，包括时序依赖关系、参数约束以及值流条件。该语言与符号评估规则相结合，将策略转换为有限轨迹上的SMT约束，从而使VIGIL能够检测出依赖于事件顺序、参数关系或跨调用值流的违规行为，而非依赖固定的单调用过滤器。在涵盖办公文档、运维和工程任务的真实大模型智能体运行中，VIGIL以超过95%的召回率和低于10%的误报率检测出策略违规。

## Abstract
Agentic systems increasingly act through third-party skills, allowing model-generated decisions to affect files, communication channels, and cyber-physical devices. These skills often include natural-language specifications that define access permissions, disclosure limits, execution privileges, and required preconditions. Although such specifications describe the intended boundaries of skill behavior, they do not by themselves provide executable runtime enforcement. Enforcing them raises a contextual granularity challenge: even when a policy is written for a particular task context, a monitor must still decide which events to observe, what state to retain, how far across the execution to reason, and where to intervene. Choosing the wrong granularity can either block benign executions or miss violations that emerge only across multiple actions. Most existing enforcement mechanisms, however, assume a fixed event model or enforcement point.   In this work, we present VIGIL, an end-to-end runtime enforcement framework for agentic systems. VIGIL checks an agent's actual execution trace against behavioral policies from skill specifications, operator-defined constraints, and global rules spanning multiple skills. To make such policies executable, VIGIL introduces a policy language that captures context-specific enforcement requirements over agent-tool events, including temporal dependencies, argument constraints, and value-flow conditions. The language is paired with symbolic evaluation rules that translate policies into SMT constraints over finite traces, allowing VIGIL to detect violations that depend on event order, argument relationships, or cross-call value flow rather than relying on fixed single-call filters. On real LLM-agent runs spanning office-document, operational, and engineering tasks, VIGIL detects policy violations with over 95% recall and a false-positive rate below 10%.

### 1. 论文核心问题与研究动机

- **背景**：大语言模型（LLM）驱动的智能体（agent）正在从对话助手演变为可执行文件操作、代码修改、物理工作流编排的自主操作者。这些能力通常通过**技能**（skills）打包发布，每个技能附带有自然语言行为规约，用以约束访问权限、数据泄露、执行前提条件等。
- **核心矛盾**：自然语言规约描述了技能“应当遵守”的行为边界，但在运行时却**缺乏自动、可执行的强制手段**，完全依赖智能体自身遵守规约。实际生产环境中已出现大量因 prompt 注入或智能体行为偏差导致的规约违规（如数据泄露、误删资源）。
- **关键粒度难题**：大多数违规并非单个“坏”工具调用，而是**跨多步、依赖事件顺序和参数值流**的非法执行。现有防御手段（沙箱、单调用过滤器、信息流跟踪等）难以捕捉这类跨事件行为违规，因为它们缺乏对执行历史的记忆和对值流、时序依赖的综合判断能力。

### 2. 方法论：VIGIL 的整体设计与技术细节

VIGIL 被设计为一个**端到端运行时参考监控器**，核心思想是对智能体执行轨迹与从规约导出的行为策略进行符号化的符合性判断，并定位违规调用。

- **策略语言**：VIGIL 设计了一套有限轨迹时序策略语言，包含六种预设语法形式（absence、precedence、response、bounded response、resolve、until），以及一个可回退的通用时序体（□Θ）。这些形式支持对事件模式（动作、参数值、输出、状态）施加**时序关系、参数约束和值流绑定**（用变量关联不同事件的同一参数）。
- **抽象与编译**：
  - `ABSTRACT`：将原始工具调用流转换为**带类型的有限事件轨迹**，提取签名（动作集合、参数名、值集合）和事实库（事实原子，包括事件顺序、参数值、状态等）。采用**封闭世界假设**，使未发生的事实可判定性为假。
  - `COMPILE`：根据规约（通过 LLM 辅助）生成符合上述签名词汇的策略语句，并对每条语句进行符号化编译，转化为**量词无关的有限轨迹 SMT 公式**（Rφ）。
- **SMT 验证与违规定位**：
  - 对于每条策略语句 φ，将其要求 Rφ 与事实库 JτK 一起提交给 SMT 求解器（Z3），检查 `unsat(JτK ∧ Rφ)`。若不可满足，则表示轨迹违反该语句。
  - 利用 SMT 求解器的**最小不可满足核心（muc）**直接定位到**违规的事件编号**，为阻断或回滚提供精确见证。
- **在线部署**：在智能体发出工具调用但尚未执行时，VIGIL 基于已记录轨迹前缀进行判定，阻断违规调用，然后由环境决定回滚、重新规划或人工干预。

### 3. 实验设计与对比方法

- **主要数据集**：

  - **AgentDojo**（629 用例）与 **SafeAgentBench**（546 用例）用作跨基准对比，主要评估在单调用可见违规场景下的泛化能力。

### 6. 主要结论与发现

- VIGIL 在主要评测集 SB+SI 上达到 **95.8% 召回率、89.6% 精确率**，F1 值领先最强基线（LLM-as-judge）**15.8 分**，同时平衡误差率仅 5.4%，表明其对违规和良性流的兼顾。
- 消融实验表明，**参数接地**对召回率贡献 **31.9 个百分点**，**策略结构**贡献 5.5 个百分点，二者是系统高准确率的关键。
- 延迟方面，单次 SMT 验证平均仅 **0.27 秒**，端到端约 4.38 秒，瓶颈在 LLM 编译步骤（4.05 秒），但可通过缓存大幅降低。
- 在 216 次真实技能部署运行中，VIGIL 发现 **34 个确认违规**，其中包括 NVIDIA 技能生态中的规约缺陷，已获得供应商确认。

### 7. 优点与亮点

- **轨迹级别的时序强制执行**：突破单调用过滤限制，能检测到依赖事件顺序和跨调用值流的隐匿违规。
- **值流绑定能力**：策略语言可用变量连接不同事件中的同一数据对象（如中间产物），使监控器准确捕捉“验证某物才能使用”这类规范。
- **符号化见证定位**：利用 SMT 最小不可满足核心直接指出违规事件，为自动阻断或人工审计提供了精确位置。
- **策略与执行分离**：规约含义的通用部分与运行时具体实例化解耦，使同一规约可适应不同运行上下文。
- **实际部署验证**：在多个供应商的真实技能包上发现可复现的规约违规和规格缺陷，体现了研究的实际影响。

### 8. 不足与局限

- **可观测性边界**：VIGIL 只能感知**工具调用层面的事件**，如果相关操作完全封装在脚本内部（如 Python 脚本内直接删除文件），则无法监控，造成漏检。
- **编译保真度依赖 LLM**：策略由 LLM 从自然语言规约转化为形式语句，有时会**过度泛化**（丢失限定词如“批量”），引发误报。编译质量直接影响系统精度上限。
- **有限时序模式**：预设的六种形式覆盖常见安全模式，但对某些复杂规范的表达可能需用回退 □Θ 形式，增加编译难度。
- **封闭世界假设前提**：要求事实库完整记录所有工具调用，若记录管道存在遗漏，缺省事实可能导致误判。
- **延迟瓶颈**：LLM 编译是性能瓶颈，虽然可缓存，但对于动态变化的技能或一次性任务，仍可能引入较高延迟。
