## 摘要
提示注入是集成大语言模型应用的首要安全风险，然而迄今为止提出的所有防御措施都已被攻破。我们证明这并非巧合：在缺乏强制控制与数据分离的共享嵌入架构中，完美的提示注入预防在数学上是不可能的。我们将提示系统形式化为提示行动模型，其输出包括具有控制权威性的动作：拒绝决策、工具授权、策略路由和内存写入。我们定义了语义忠实控制（SFC），即这类行为仅取决于不可信输入的含义，而非其编码方式。进而我们通过三项结果证明，在共享流水线内SFC是不可实现的：溯源恢复不可能性（共享表示使得可信与不可信内容在统计上无法区分，受总变差距离的约束）；控制路径暴露（不可信令牌通过决定输出的同一注意力值聚合进入控制相关计算）；以及有限覆盖不变性差距（有限训练无法在无限的语义等价类上认证不变性）。我们在生产级分词器和模型上对每个量进行了实测验证。该结果是结构性的，并非当前防御的缺陷。它映射了冯·诺依曼机器中代码与数据混淆导致缓冲区溢出的情形，这类漏洞花了数十年时间才通过层层防御（DEP、Write-XOR-Execute、ASLR、栈金丝雀，并最终是内存安全语言）得以控制，因为单一机制不足以应对。其含义相同：提示注入无法仅通过更好的流水线内分类或对齐来消除，它需要指令与数据通道的架构分离。我们指出了根本原因以及它所需的解决方案类别。

## Abstract
Prompt injection is the top security risk for LLM-integrated applications, yet every defense proposed so far has been broken. We prove this is not a coincidence: in shared-embedding architectures that lack enforced control-data separation, perfect prompt-injection prevention is mathematically impossible. We formalize prompted systems as Prompted Action Models whose outputs include control-authoritative actions: refusal decisions, tool authorization, policy routing, and memory writes. We define Semantic-Faithful Control (SFC), the property that such behavior depends only on the meaning of untrusted input, not on how it is encoded. We then prove SFC is unachievable within the shared pipeline, via three results: a provenance-recovery impossibility (shared representations make trusted and untrusted content statistically inseparable, bounded by total variation distance); control-path exposure (untrusted tokens enter control-relevant computation through the same attention value-aggregation that determines outputs); and a finite-coverage invariance gap (finite training cannot certify invariance over infinite semantic-equivalence classes). We ground each quantity in measurements on production tokenizers and models.   The result is structural, not a gap in current defenses. It mirrors the code-data confusion in Von Neumann machines that gives rise to buffer overflows, a vulnerability class that took decades of layered defenses (DEP, Write-XOR-Execute, ASLR, stack canaries, and ultimately memory-safe languages) to contain, because no single mechanism sufficed. The implication is the same: prompt injection cannot be eliminated by better in-pipeline classification or alignment alone. It requires architectural separation of instruction and data channels. We identify the root cause and the class of solution it demands.

## 1. 论文的核心问题与整体含义

- **核心问题**：大型语言模型（LLM）集成应用中，提示注入攻击（prompt injection）是最普遍且高危的漏洞，攻击者通过恶意构造的用户数据使模型偏离系统指令。现有防御（如指令微调、RLHF、护栏分类器、电路断路器）均被频繁攻破，多轮攻击在某些模型上成功率高达90%以上。
- **整体含义**：该工作证明这一现状并非防御设计的偶然不足，而是**架构固有的结构性局限**——在缺乏强制“控制-数据”分离的共享嵌入序列模型中，完美的提示注入预防在数学上不可能实现。类比冯·诺依曼架构中代码与数据共享同一存储导致缓冲区溢出数十年无根治方案，提示注入的根因同样是架构层级的表示混淆，仅靠流水线内的分类或对齐训练无法彻底消除，需要**指令与数据通道的架构分离**。

## 2. 论文提出的方法论

- **形式化定义：提示行动模型（Prompted Action Model）**
  - 将系统建模为 \(M = (\Sigma, d, E, f, A)\)，其中 \(\Sigma\) 为词表，\(d\) 为嵌入维度，\(E\) 为共享嵌入函数（指令与用户内容经同一映射进入共享向量空间），\(A\) 为包含控制权威动作的输出空间（token 生成、工具调用、策略路由、内存写入），\(f\) 为从嵌入序列到动作分布的模型函数。
  - 定义带来源标记的提示输入 \(x = (s \| u)\)，\(s\) 为可信指令前缀，\(u\) 为不可信用户内容，拼接后每个位置携带来源变量 \(R_j \in \{\text{trusted}, \text{untrusted}\}\)。关键性质为“共享嵌入公理”：嵌入后无可变类型标记区分可信与不可信向量。

- **安全属性：语义忠实控制（Semantic-Faithful Control, SFC）**
  - 要求控制权威输出（拒绝、工具授权、策略选择、内存写入）仅取决于不可信输入的**语义内容**，而非其编码上的无关变化。即，若 \(u \equiv_{\text{sem}} u'\)（语义等价，仅编码形式不同），则控制行为投影必须一致：\(\Pi_{\text{ctrl}} Y(s\|u) = \Pi_{\text{ctrl}} Y(s\|u')\)。
  - \(\equiv_{\text{sem}}\) 在证明中对参数化实例（如字符混淆、同形异意、base64、无意义后缀等）均成立，且已有通用对抗后缀（GCG）构成对 SFC 的构造性违规。

- **三层证明结构（不可分离性定理）**
  1. **溯源恢复不可能性**：在共享表示下，给定一个表示实例 \(Z\)（嵌入向量或上下文隐藏态），其来自可信或不可信来源的判定贝叶斯最优错误率与各自条件分布的总变差距离（TV）严格相关。由于共享词汇导致嵌入重叠，且实际上下文表示（假设分布不相交的弱条件不成立）**TV < 1**，故完美溯源分类（错误率为0）不可能。
  2. **控制路径暴露**：在标准的因果解码器中，生成位置的隐藏状态（即控制相关节点）通过注意力值聚合同时吸收了可信和不可信位置的贡献。如果模型要利用用户输入，则控制节点对不可信位置的注意力权重不可能为零，因此不可信内容必然进入影响控制权威输出的计算。
  3. **有限覆盖不变性间隙**：语义等价类 \([u]_{\text{sem}}\) 通常是无限或组合巨大的（改写、翻译、格式变化、对抗后缀等）。训练、对齐或红队能覆盖的代表样本集 \(T\) 是有限的，但在缺乏架构级别强制不变性的情况下，模型在 \(T\) 上的正确行为无法保证对所有 \(u' \in [u]_{\text{sem}}\) 都成立。故训练无法消除对未见过编码变体的漏洞。

- **桥接命题**：在通用非退化假设下，若不可信内容的表示级扰动引起控制节点状态变化，则有概率1引起最终控制行为分布的变化（依分布绝对连续的 Lebesgue 测度讨论）。由此表示级漏洞必然转化为可观测的行为级失控。

- **定量边界**：给出序列级对手可达到的贝叶斯最优区分错误下界 \(P^*_{e,n}(B) \ge \frac12 (1 - \delta^*_n(B))\)，其中 \(\delta^*_n(B)\) 为对手在预算 \(B\) 下最小化 TV 距离的结果。随着对手能力增强，区分难度上升至随机猜测水平。

## 3. 实验设计

实验并非用于验证理论定理本身（因其为形式化证明），而是通过实际系统测量理论中关键量的现实存在性和量级。
- **M1 词汇重叠测度**：收集指令语料（四个公开系统提示库）与用户语料（UltraChat-200K 的5万条用户轮次），在 Mistral-7B（32K）、Llama-3.1（128K）、Qwen-2.5（152K）三个分词器上统计词汇重叠度（Jaccard 系数、概率质量重叠、指令覆盖率 \(\rho_{\text{cov}}\)）。结果：\(\rho_{\text{cov}}\) 范围为 61.5%–82.3%，证实指令与用户数据共享大量词汇。
- **M2 表示层重叠**：使用 Llama 3.1 8B Instruct（8-bit 量化），从指令和用户语料各抽取 300 条序列，计算各层隐藏态分布的 MMD（最大均值差异，RBF 核）。发现第0层 MMD ≈0.61（高度重叠），第8层分离程度最大（约0.91），随后在输出相关层（24、31层）MMD 下降至 0.76 左右，**未见完全分离**。同时指出 KV-cache 量化趋势会进一步压缩表示空间，加重该重叠。
- **M3 行为级 SFC 违反**：构建金融服务系统提示，对 Llama 3.1 8B Instruct 与 Mistral 7B Instruct v0.2 进行五种提示注入攻击（角色越狱、多示例引导、伪造管理指令、渐进式升级、虚拟化劫持）。虽然所有攻击均被拒绝（未直接实现拒绝翻转），但输出呈现**长度、前置警告、框架改变等结构性差异**，证明不可信内容已侵入控制路径。
- **M4 权重切除（Abliteration）确认**：引用社区成果，指出安全方向可通过投影移除权重来消除拒绝能力，且该模式在不同对齐模型中普遍成立，印证拒绝行为仅是共享表示空间中的低维特征，与定理预测的“流水线内对齐只能做行为补丁而不能根除架构脆弱性”一致。

实验未涉及与其他防御方法的横向性能对比，因论文目的不是提出新防御，而是论证现有架构的固有上限。

## 6. 主要结论与发现

- **完美提示注入预防在共享嵌入架构的流水线内不可能实现**。根因在于指令与用户数据经过相同嵌入函数进入共享表示空间，类型信息被擦除，导致：
  - 无法完美溯源；
  - 不可信内容必然通过注意力机制进入控制相关计算；
  - 有限训练无法覆盖无限编码等价类。
- 任何仅停留在**表示空间内学习决策边界或不变性**的防御（如对齐训练、护栏、表示工程、电路断路器）都受此结构性上限约束，可被绕过。
- 提示注入问题与冯·诺依曼机器中缓冲区溢出**在架构层面同构**，都是因共享存储/表示导致的数据被解释为控制指令。该漏洞类别的发展史表明，最终控制需依赖**强制性架构分离**（如硬件 NX 位、Write-XOR-Execute），以及**多层纵深防御**，并最终走向**语言级安全**。提示注入也需类似的指令与数据通道分离方案。

## 7. 优点

- **理论上高度原创与扎实**：用严密的形式化框架（提示行动模型、语义忠实控制）给出不可分离性定理，证明路径清晰（溯源不可分、控制路径暴露、有限覆盖间隙）。
- **类比深刻**：将指令-数据混淆与经典计算机安全中的代码-数据混淆类比，借助缓冲区溢出的历史演进推断提示注入的未来防御路径，有效提升了结论的紧迫性和指导价值。
- **实证闭环**：通过对真实分词器和模型的词汇、表示、行为层测量，使理论假设 (词汇重叠、分布不分离等) 落地为可观测事实，增强了定理的实践可信度。
- **明确的安全属性定义**：SFC 区分了“合理依赖用户语义”与“不应受编码形式影响”的界限，为未来安全需求界定提供了清晰标准。
- **开放与负责任**：明确指出了定理的适用边界（不适用有强制分离的架构）和自身实验的局限，并指出了真正有效的解决方向（如 ASIDE 初始化分离）。

## 8. 不足与局限

- **行为实验未直接捕获 SFC 违规的核心**：M3 观察到的输出变化仅是外围现象，拒绝决策未翻转，严格意义上不算 SFC 破坏的直接实验证据，需依赖外部已发表的 GCG 等攻击佐证。
- **实证模型覆盖面有限**：表示层重叠（MMD）只在 Llama 3.1 8B 上进行，未在更大规模、不同系列（如 Gemma、GPT 类）、多模态模型上扩展，样本量也偏小。
- **理论假设个别条件在实践中可能不完全满足**：例如控制路径暴露要求模型必须使用用户输入，而极端情况或有特殊设计的模型可能通过极度限制注意力权重规避，虽然这会丧失功能。
- **对实际防御的建议停留在高层次**：虽然指明了架构分离（如不可伪造的来源标注、独立编码器等）的必然性，但并未提供一种即插即用的技术方案细节，其实施的工程成本、性能影响和可扩展性未评估。
- **没有量化地给出实际系统目前与“完美分离”的距离**：证明完美防御不可能，但并未提供一个可计算的安全度量，来帮助开发者评估“我离理论上限还有多远”。（虽然引用了 Zverev 2025a 的分离度指标与之互补。）
- **von Neumann 类比虽然深刻但具有简化风险**：LLM 的神经网络动态远比内存操作复杂，该类比可能在精确机理上不能完全映射。
