## 摘要
间接提示注入攻击通过在任务执行过程中检索的第三方数据中嵌入恶意指令，劫持基于大语言模型的智能体。现有防御在静态基准上报告了近乎为零的攻击成功率，但近期的自适应评估显示，一旦攻击者被允许针对部署的防御进行优化，这些结果就会崩溃。本工作将这种崩溃归因于两种失败模式。第一，现有防御方法局限于识别特定攻击模式，而非评估每条嵌入指令的意图是否与用户任务相关。第二，基于训练的防御虽然提供了最强的安全-效用权衡，但其对抗样本仅来自少数手工模板，导致防御者无法泛化到该狭窄策略分布之外。为解决这些不足，我们提出RETA，一种基于训练的方法，将防御决策建立在用户任务而非攻击者控制的数据之上。在每个工具输出步骤，防御者进行思维链推理，验证其行为与用户任务一致。利用红队测试，模拟攻击者合成对抗训练数据，并获得基于词典学习的多样性奖励，实现了对注入重构策略的广泛覆盖。这些共同使得防御者能够通过多目标强化学习进行优化，取得更好的安全-效用权衡。在六种黑盒自适应攻击中，RETA将每次攻击的攻击成功率保持在10%以下，在两个目标模型上的平均攻击成功率分别为2.92%和3.75%，同时在攻击下和干净输入上保留了大部分效用。

## Abstract
Indirect prompt injection attacks hijack LLM-based agents by embedding malicious instructions in third-party data that the agent retrieves during task execution. Existing defenses report near-zero attack success rate on static benchmarks, yet recent adaptive evaluations show that these results collapse once the attacker is allowed to optimize against the deployed defense. In this work, we trace this collapse to two failure modes. First, existing defense methods are confined to recognizing specific attack patterns, rather than assessing whether the intent of every embedded instruction is relevant to the user task. Second, training-based defenses, which otherwise offer the strongest safety-utility trade-off, assemble their adversarial examples from a handful of hand-crafted templates, and the resulting defender fails to generalize outside that narrow strategy distribution. To address these gaps, we propose RETA, a training-based method that grounds defense decisions on the user tasks rather than attacker-controlled data. At each tool-output step, the defender undertakes chain-of-thought reasoning verifying that its actions are consistent with the user task. Leveraging red-teaming, a simulated attacker synthesizes adversarial training data and receives a dictionary-learning diversity reward, achieving broad coverage of injection-reformulation strategies. Together, these allow the defender to be optimized via multi-objective reinforcement learning and achieve better safety-utility trade-off. Across six black-box adaptive attacks, RETA keeps every per-attack ASR below 10%, with average ASR of 2.92% and 3.75% on the two target models, while preserving most utility under attack and on clean inputs.

# 基于推理的任务对齐：防御自适应提示注入攻击

### 1. 论文的核心问题与整体意义

*   **研究背景**：大语言模型智能体（LLM-based agents）通过工具调用来检索和操作外部数据，已得到广泛部署。这引入了**间接提示注入**（Indirect Prompt Injection, PI）的严重安全漏洞。攻击者可在第三方数据源（如网页、API 响应）中植入恶意指令，在智能体执行用户任务时劫持其行为。
*   **核心问题**：现有 PI 防御方法在**静态基准测试**（注入模板固定）中表现优异，声称近乎零攻击成功率。然而，当面对**自适应攻击者**（可根据防御反馈迭代优化注入内容）时，这些防御会迅速失效，安全性与任务效用严重下降。论文旨在揭示这种“静态-自适应”鲁棒性差距的根本原因，并设计一种能在保持用户任务效用的同时，有效抵御自适应攻击的防御机制。
*   **研究意义**：本工作深刻揭示了当前 PI 防御的共性缺陷，并提出了新的设计范式和具体方法，强调了从“识别攻击模式”向“强制任务对齐”的转变，对构建更鲁棒的AI智能体安全体系具有重要指导意义。

### 2. 方法论：RETA

论文提出 **RETA**（**RE**asoning-enabled **T**ask **A**lignment，推理赋能的任务对齐），这是一种通过多阶段对抗训练实现的防御方法，其核心思想、设计原则与流程如下：

*   **核心思想与三大设计原则**：
    1.  **基于用户任务推理，而非识别注入模式**：不再试图识别被污染的观察文本是否为“攻击”，而是要求防御者在行动前进行**思维链推理**（Chain-of-Thought），明确判断下一步的**工具调用集合**是否源于可信的用户任务（User Task），而非观察内容中的恶意指令。这要求防御满足两个语义条件：
        *   **C1**：识别出观测内容中试图重定向智能体的部分。
        *   **C2**：证明计划的工具调用集仅从用户任务派生，独立于观测历史中的指令性内容。
    2.  **训练覆盖攻击策略，而非特定攻击实例**：为应对对抗训练分布单一的问题，**红队（Red-teaming）阶段**通过**策略词典**（Strategy Dictionary）来量化注入攻击的**策略多样性**，并给予生成未充分解释、具有新策略的成功注入更高奖励，以此鼓励攻击者探索更广泛的攻击重构方式。
    3.  **采用对抗强化学习（Adversarial RL, ARL）作为训练范式**：提示注入防御是一个**轨迹级**（trajectory-level）问题，防御者必须在多步交互中既拒绝注入任务，又完成用户任务。因此，防御者通过强化学习在完整的交互轨迹上被优化，仅当同时满足安全性和效用性时才获得奖励。

*   **关键技术流程（两阶段训练）**:
    1.  **第一阶段：基于红队测试的对抗样本生成（Red-Teaming）**
        *   **初始化**：攻击者模型通过少量监督微调（SFT），学习基础的攻击表述能力。
        *   **策略词典学习**：
            *   将每个成功的注入攻击通过LLM提炼为“**策略记录**”（Strategy Record，包含名称和解释）。
            *   利用**字典学习**（Dictionary Learning）对所有策略记录的嵌入向量进行稀疏重构，得到一组可解释的**词典基元**（Dictionary Primitives）。
            *   这些基元代表了攻击者使用的核心策略。一个策略记录可以由多个基元的稀疏组合来解释。
        *   **多样性奖励**：对于新生成的注入，通过判断其策略是否能被现有词典充分解释，计算**可解释性得分**（Explainability Score）。得分越低（越无法解释），其**多样性奖励**越高。
        *   **攻击者优化**：攻击策略通过**GDPO**（Group reward-Decoupled Normalization Policy Optimization）算法进行优化，除了攻击成功奖励外，还加入了多样性奖励，引导其探索新的攻击策略。
    2.  **第二阶段：防御者对抗强化学习**
        *   **初始化**：防御者模型首先在精心构造的“边界案例”上进行格式调优，学习在工具调用前表达任务对齐推理的过程。
        *   **防御者优化**：使用第一阶段收集的多样化攻击样本，通过强化学习优化防御策略。防御者同时接收三项奖励：
            *   **安全奖励（Security Reward）**：注入任务未完成（ASR=0）。
            *   **效用奖励（Utility Reward）**：可信的用户任务成功完成。
            *   **思维链奖励（CoT Reward）**：防御者在决策步骤中产生的推理满足C1和C2条件。
        *   这三项奖励同样通过GDPO算法进行联合优化，平衡安全与效用。

### 3. 实验设计

    *   **迁移评估集**：使用了 ASB 和 InjecAgent 两个不同结构的基准，以测试防御的泛化能力。

### 6. 主要结论与发现

1.  **现有防御存在严重“静态-自适应”鲁棒性差距**：在静态攻击下表现优异的防御，面对自适应攻击时，平均攻击成功率显著上升（在两个模型上平均上升15.8个百分点），最高可达50%以上。
2.  **防御失效的两大根本原因**：
    *   **观察层面的决策**：防御仅检查检索到的文本是否“像”已知攻击，而忽略了其意图是否与用户任务对齐。攻击者词句层面的改写即可绕过。
    *   **对抗训练的模式坍塌**：基于训练的方法，其对抗样本策略分布狭窄，导致防御模型记忆了攻击模板，无法泛化到未见过的攻击策略。
3.  **RETA 实现了卓越的安全-效用平衡**：
    *   **高鲁棒性**：在六种自适应攻击下，将每次攻击的成功率均控制在 10% 以下，平均攻击成功率在两个模型上低至 2.92% 和 3.75%。
    *   **高任务效用**：在强力抵御攻击的同时，其攻击下效用和干净效用均接近甚至超过未防御的基线模型。
    *   **强泛化能力**：在跨基准的迁移测试中，RETA 相比最强基线 MetaSecAlign 显著降低了 ASR。

### 7. 优点

*   **思想深刻，设计新颖**：准确地识别了现有防御“重模式识别、轻任务对齐”和“训练策略单样本坍塌”的共性缺陷，并提出了创新性的解决思路。
*   **方法论闭环与优雅**：两阶段设计（红队探索防御弱点 -> 防御者针对性学习）形成了一个完整的对抗性自我提升闭环。将字典学习引入红队以量化策略多样性，是技术上的一大亮点。
*   **全面且严谨的评估**：不仅对比了多种范式的防御，覆盖了多种自适应攻击，还通过详细的消融和迁移实验验证了各组件的有效性和方法的泛化能力，结论非常扎实。
*   **兼顾安全与效用**：不是简单地拒绝或过滤，而是通过任务对齐推理，在完成用户任务的前提下抵御攻击，更符合实际应用需求。

### 8. 不足与局限

*   **评估范围与真实性**：
    *   评估局限于单个用户-智能体会话，未考虑攻击存在于持久化工具输出、内存状态或多轮/多会话交互中的更现实场景。
    *   自适应攻击仅在40个案例的子集上评估，尽管筛选标准保证了其难度，但全面性仍有不足。
*   **成本与延迟**：RETA 引入的任务对齐推理显著增加了推理阶段的 token 消耗和延迟，这在成本敏感或对实时性要求高的应用中可能成为瓶颈。
*   **效用边界问题**：分析表明，RETA 偶尔会表现出“过度防御”，将信任的用户任务所必需的工具输出参数也视为不安全而拒绝，导致用户任务失败。效用指标相比部分基线并无绝对优势。
*   **比较范围的限制**：未与改变执行模型的系统级防御（如 CaMeL）进行对比，论文给出的理由是当前基准测试无法为二者提供公平的比较平台。
