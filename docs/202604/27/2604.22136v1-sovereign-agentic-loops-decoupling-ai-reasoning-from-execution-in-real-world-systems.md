## 摘要
大语言模型（LLM）代理越来越多地发出改变真实系统的 API 调用，然而许多当前的架构直接将随机的模型输出传递给执行层。我们认为这种耦合带来了安全风险，因为在执行时无法假设模型的正确性、上下文感知能力和对齐性。我们引入了主权代理循环（Sovereign Agentic Loops, SAL），这是一种控制平面架构，其中模型发出带有理由的结构化意图，而控制平面在执行前根据真实的系统状态和策略验证这些意图。SAL 结合了限制模型访问身份敏感状态的混淆膜（obfuscation membrane），以及用于可审计性和回放的加密链接证据链（Evidence Chain）。我们对 SAL 进行了形式化，并证明在所述假设下，它提供了受策略约束的执行、身份隔离和确定性回放。在针对云基础设施的 OpenKedge 原型中，SAL 在策略层拦截了 93% 的不安全意图，通过一致性检查拒绝了剩余的 7%，在我们的基准测试中防止了不安全执行，并增加了 12.4 毫秒的中值延迟。

## Abstract
Large language model (LLM) agents increasingly issue API calls that mutate real systems, yet many current architectures pass stochastic model outputs directly to execution layers. We argue that this coupling creates a safety risk because model correctness, context awareness, and alignment cannot be assumed at execution time. We introduce Sovereign Agentic Loops (SAL), a control-plane architecture in which models emit structured intents with justifications, and the control plane validates those intents against true system state and policy before execution. SAL combines an obfuscation membrane, which limits model access to identity-sensitive state, with a cryptographically linked Evidence Chain for auditability and replay. We formalize SAL and show that, under the stated assumptions, it provides policy-bounded execution, identity isolation, and deterministic replay. In an OpenKedge prototype for cloud infrastructure, SAL blocks 93% of unsafe intents at the policy layer, rejects the remaining 7% via consistency checks, prevents unsafe executions in our benchmark, and adds 12.4 ms median latency.

*   **研究动机**：当前的 AI 智能体架构通常将 LLM 的输出直接视为可执行命令（如 API 调用）。由于 LLM 具有**随机性（Stochastic）**，且往往在信息不全或上下文模糊的情况下运行，这种“推理与执行紧耦合”的模式存在巨大风险：智能体可能因误解上下文而删除关键数据库或违反安全策略。
*   **核心问题**：如何防止随机的 AI 推理过程直接导致不可逆的现实系统破坏，同时在利用外部强大模型（如 GPT-4）时保护本地系统的隐私和主权？
*   **整体含义**：论文提出了“解耦原则”，即推理模型应产生**可验证的意图（Intent）**而非**直接执行权**。

### 2. 方法论：SAL 架构
SAL 建立了一个主权控制平面，将推理与执行完全隔离，核心流程如下：
*   **混淆膜（Obfuscation Membrane）**：在将系统状态发送给外部模型前，通过投影函数 $\Pi$ 剥离身份敏感信息（如 IP 地址、具体 ID），仅保留结构化拓扑。这实现了信息论上的隔离，确保模型无法获取真实身份。
*   **意图协议（Intent Protocol）**：模型不直接调用 API，而是输出“结构化意图 + 自然语言理由（Justification）”。
*   **确定性评估引擎（Evaluation Engine）**：
    *   **策略对齐**：使用 Open Policy Agent (OPA) 等工具检查意图是否违反硬性安全规则。
    *   **一致性检查**：对比模型给出的“理由”与“真实系统状态”。例如，若模型理由是“CPU 过高”，但实时监控显示 CPU 正常，则拒绝执行。
*   **受控执行与证据链（Evidence Chain）**：仅在验证通过后，由控制平面使用短期凭证执行操作，并将整个决策链（状态、意图、理由、结果）通过加密哈希链接成不可篡改的账本，支持确定性回放和审计。
