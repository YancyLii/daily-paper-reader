## 摘要
现代AI代理实现（如前沿编码代理）在运行时链接多个工具，由此产生的安全表面是单工具防护栏无法解决的，因为单独允许的工具在组合时可能违反组织策略。我们提出动态安全控制组合器（DSCC），一种针对多工具代理链的组合安全两阶段方法。第一阶段在会话检出时，最严格集合（MRS）算法将每个工具的安全策略组合成单一有效策略，并具备形式化单调不变性，即扩展链只会收紧结果，从而在任何工具执行前阻止不兼容组合。任何工具调用的输出将其分类约束传播到会话级污点状态，使得后续调用必须满足迄今所见最严格约束。第二阶段在运行时，系统通过单调污点状态跟踪代理所触及数据的敏感性，若累积暴露会使后续工具调用违反策略，则撤销会话。这两个阶段共同构成深度防御：静态组合防止不安全链启动，运行时污点跟踪捕获因具体数据而产生的违规。我们在32个工具上提供了参考实现，这些工具受16条与NIST SP 800-53对齐的策略治理，并在两种组合模式下评估。在默认许可模式下，允许的组合被划分为分类级集群，阻止79.2%的策略对和95.5%的三元组。替代污点模式允许渗漏边界内的混合分类链，分别阻止42.5%和60.5%。我们讨论了部署多工具代理的组织的治理影响，包括效用-安全权衡以及操作化链感知策略所需的变化。

## Abstract
Modern AI agent implementations such as frontier coding agents chain multiple tools at runtime that create a security surface that per-tool guardrails are unable to address, as individually permitted tools can violate organizational policies when composed. We propose the Dynamic Security Control Compositor (DSCC), a two-phase approach to compositional security for multi-tool agent chains. In Phase 1, at session checkout, a Most Restrictive Set (MRS) algorithm composes per-tool security policies into a single effective policy with a formal monotonicity invariant that extending a chain can only tighten the result, blocking incompatible combinations before any tool executes. Outputs of any tool call propagate their classification constraints into a session-level taint state, so subsequent invocations must satisfy the most restrictive constraints seen so far. In Phase 2, at runtime, the system tracks the sensitivity of data the agent touches through a monotonic taint state and revokes the session if the accumulated exposure would make a subsequent tool call a policy violation. Together, these phases provide defense in depth, where static composition prevents unsafe chains from starting, and runtime taint tracking catches violations that emerge from the specific data used. We provide a reference implementation on 32 tools governed by 16 NIST SP 800-53 aligned policies and evaluate it under two composition modes. In the default clearance mode, permitted combinations are partitioned into classification-level clusters, blocking 79.2% of policy pairs and 95.5% of triples. The alternative taint mode admits mixed-classification chains within the exfiltration boundary, blocking 42.5% and 60.5% respectively. We discuss the governance implications for organizations deploying multi-tool agents, including the utility-security tradeoff and the changes needed to operationalize chain-aware policies.

- **典型例子**：一个文件读取工具和一个HTTP客户端各自合法，但组合后代理可能读取内部文档并POST到外部服务器，造成数据泄露；或者一个处理保密数据的工具输出被未授权处理敏感信息的工具使用。
- **威胁现状**：已有研究发现82.7%的代理工具漏洞需要多工具执行才能触发，且攻击序列可由多个无害步骤组合而成，攻击成功率超过90%。这说明**组合授权缺口**已成为一个紧迫的安全表面。
- **整体含义**：必须引入**链感知的组合安全控制**，在工具被调用前和实施中动态评估整个工具链的聚合权限，阻止超越组合策略的行为。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
### 核心思想
提出**动态安全控制组合器（DSCC）**，一种两阶段防御体系：
- **第一阶段（会话检出）**：静态组合各工具的声明式安全策略，产生一个统一的、最严格的有效策略，并预先拒绝不兼容的工具链。
- **第二阶段（运行时执行）**：跟踪代理实际接触的数据敏感度（污点），若后续调用会导致策略冲突则即时撤销会话。

### 关键技术细节与算法流程
#### 工具策略模型（六元组）
每个工具需携带声明式策略 `P_i = (B_i, C_i, φ_i, d_i, Z_i, τ_i)`：
- `B_i`：控制绑定，将NIST控制（如AC-4）映射到允许/限制/拒绝级别。
- `C_i`：数据分类（Public < Internal < Confidential < Restricted）。
- `φ_i`：传输禁止标志（1表示数据必须留在内部）。
- `d_i`：流方向（入站/出站等）。
- `Z_i`：允许的网络区域。
- `τ_i`：最大会话存活时间。

#### 第一阶段：最严格集合（MRS）算法
给定工具链 `T1,...,Tn`，MRS执行四步组合，任一失败即拒绝：

1. **兼容性检查**：检测策略间是否存在无法满足的矛盾（例如一个工具禁止外部传输，另一个却要求公网操作）。
2. **控制分辨率**：对每个NIST控制，取所有工具声明中**最严格的级别**（Allow < Restrict < Deny），Deny由单工具可导致链拒绝。
3. **数据流组合**：
   - **分类高水位**：`C_eff = max(C_0, max_i C_i)`，会话敏感度只能上升。
   - **许可检查**（仅默认的clearance模式）：要求每个工具的 `C_i` ≥ `C_eff`，否则拒绝（不允许低等级工具处理高敏感数据）；taint模式则跳过此项，允许混合分类但会提升高水位。
   - **禁止传播**：`φ_eff = max_i φ_i`，一旦某工具禁止外传，整个链都禁止。
   - **分类边界规则**：若 `C_eff ≥ Confidential` 且链中包含外发工具，直接拒绝（安全底线）。
   - **网络区域交集**：`Z_eff = ⋂_i Z_i`，若交集为空中则拒绝（可能产生两两兼容但三路不兼容的情况）。
4. **DENY强制执行**：检查解析得到的Deny级控制是否与组合后的数据流冲突（如边界保护Deny与外发同时存在）。
- **结果**：输出有效控制集 `E`（包含组合后的控制级别、分类、禁止标志、区域、TTL），并满足**单调性不变量**：添加新工具只会收紧策略，永不放松。

#### 第二阶段：运行时执行模型
- **资源分类**：代理访问的每个数据资源都有敏感度标签（同上述四级分类）及外传禁止标志。
- **会话污点状态**：`S = (C_s, φ_s)`，初始为第一阶段计算值或最低值。每次资源访问时，根据资源标签进行单调更新：`C_s ← max(C_s, C_r)`，`φ_s ← max(φ_s, φ_r)`。
- **执法检查**：每次工具调用前评估三个卫士，任一触发则撤销会话：
  1. 会话污点有外传禁止 **且** 请求的工具是外发的。
  2. 当前资源本身禁止外传 **且** 链中有外发工具。
  3. 第一阶段已计算出的数据流禁止外传 **且** 当前工具是外发的。
- **单调性扩展**：污点更新算子与静态组合算子同构，保证运行时约束只能收紧。

### 两种工作模式
- **清除模式（default）**：严格按分类等级分区，不允许跨级组合。
- **污点模式**：允许混合分类链执行，但通过高水位污点和外传边界来兜底。

## 6. 论文的主要结论与发现
- **组合安全缺口真实且严重**：多工具链的聚合权限问题无法被单工具护栏解决，必须引入组合策略机制。
- **MRS算法有效且严格**：
  - 在清除模式下，强制形成了分类级别集群，阻止了所有跨级组合，策略对阻止率79.2%，三元组阻止率95.5%。绝大部分阻止由许可检查完成。
  - 在污点模式下，三个较高安全级（Internal, Confidential, Restricted）合并成一个许可区，阻止率降至42.5%（对）和60.5%（三元组），但仍确保**数据渗出边界**被守住：任何被标记为高于Public的数据无法到达外发工具。
- **单调性保证深度防御**：无论是在第一阶段静态组合还是第二阶段运行时更新，有效约束都只增不减，构建了一个可预测且无法放松的安全边界。
- **两阶段互补**：静态相阻止不安全的链启动，运行时相根据实际接触的数据动态撤销，能够捕获仅靠声明策略无法预见的违规。

## 7. 优点：方法或实验设计上的亮点
- **首次形式化解决代理工具链的组合授权问题**：明确定义了工具策略模型和组合算子，给出了形式化单调性不变量，理论基础扎实。
- **实用的两阶段深度防御**：将“启动前预检”与“运行时污点跟踪”相结合，兼顾了静态可预测性和动态实时性。
- **灵活的部署模式**：提供清除与污点两种模式，允许组织根据自身的风险偏好和授权粒度在“严格分区”和“混合但防渗出”之间选择。
- **拒绝原因可解释**：每次阻止都精确定位到冲突的控制和要求，有利于开发者和安全团队迭代策略，支持联邦式策略治理。
- **实验枚举齐全**：对给定策略空间进行了穷尽式组合分析，清晰揭示了算法行为模式（如清除模式下的聚类结构、区域交集导致的三路不可见失败），为后续研究提供了可验证的基线。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等
- **缺乏与现有系统的对比评估**：完全没有与MCP安全方案、基于能力的系统或其他代理安全框架进行实验对比，无法说明相对于其他方法的优势或劣势。
