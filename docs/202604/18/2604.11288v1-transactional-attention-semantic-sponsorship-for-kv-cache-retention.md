## 摘要
在 K=16 个 Token（4K 上下文的 0.4%）时，现有的所有 KV 缓存压缩方法在凭据检索任务上的准确率均为 0%。其失败模式在于“休眠 Token”：即凭据、API 密钥和配置值等，这些 Token 接收到的注意力几乎为零，但在生成阶段却至关重要。由于这些 Token 缺乏驱逐策略所依赖的统计信号，任何基于注意力分数、重构损失或学习保留门控的方法都无法保留它们。我们提出了事务性注意力（Transactional Attention, TA），这是一种赞助机制，通过结构化锚点模式（例如 “key:”、“password:”）保护相邻的承载值 Token 免遭驱逐。在 K=16 时，TA 实现了 100% 的凭据检索率，而六个基准模型（H2O、TOVA、SnapKV、StreamingLLM、PyramidKV、DynamicKV）均为 0%，且在 200 次函数调用测试中保持了 100% 的准确率。TA-Fast 是一种无注意力变体，可减少 52% 的内存开销，并与 SDPA 和 FlashAttention 兼容。TA 与现有的压缩方法正交，且增加的延迟开销不到 1%。

## Abstract
At K=16 tokens (0.4% of a 4K context), every existing KV-cache compression method achieves 0% on credential retrieval. The failure mode is dormant tokens: credentials, API keys, and configuration values that receive near-zero attention but become essential at generation time. Because these tokens lack the statistical signals that eviction policies rely on, no method based on attention scores, reconstruction loss, or learned retention gates retains them. We introduce Transactional Attention (TA), a sponsorship mechanism in which structural anchor patterns (e.g., "key:", "password:") protect adjacent value-bearing tokens from eviction. TA achieves 100% credential retrieval at K=16 where six baselines (H2O, TOVA, SnapKV, StreamingLLM, PyramidKV, DynamicKV) achieve 0%, and sustains 100% accuracy across 200 function-calling trials. TA-Fast, an attention-free variant, reduces memory overhead by 52% and is compatible with SDPA and FlashAttention. TA is orthogonal to existing compression methods and adds less than 1% latency overhead.

### 1. 核心问题与整体含义
*   **研究动机**：现有的 KV 缓存压缩方法（如 H2O, SnapKV 等）主要依赖注意力分数或重构损失来决定保留哪些 Token。然而，在实际应用（如 AI 编码助手、智能体工作流）中，存在大量“休眠标记”（Dormant Tokens），如 API 密钥、密码、配置参数等。
*   **核心矛盾**：这些 Token 在输入后很长一段时间内注意力分数接近于零，但在生成阶段（如调用 API 时）必须被精确提取。现有的统计学剔除策略会因其“不活跃”而将其误删，导致在极低缓存预算（如仅保留 0.4% 的 Token）下，所有基准方法的准确率均降至 0%。

### 2. 方法论
*   **核心思想**：引入“语义赞助”（Semantic Sponsorship）机制。不再被动地观察注意力信号，而是主动识别具有结构化特征的“锚点”（Anchors），并由锚点为其后的关键值 Token 提供“保护券”，确保它们不被剔除。
*   **关键技术细节**：
    1.  **锚点检测（Anchor Detection）**：通过正则匹配（Regex）、语义识别（如“remember”、“said”）或嵌入向量相似度，识别出预示关键信息即将出现的标记（如 `key:`）。
    2.  **赞助分配（Voucher Allocation）**：一旦检测到锚点，系统会为后续的 $L$ 个 Token（通常 $L=6$）分配一个指数衰减的“赞助分”（Voucher Score）。
    3.  **综合效用评分**：Token 的最终保留优先级由注意力分数、新近度、锚点状态、赞助分等加权计算得出。
    4.  **TA-Fast 变体**：为了兼容 FlashAttention 等高效后端，TA-Fast 舍弃了注意力分数项，仅依靠位置和语义模式进行保留决策，大幅降低了内存开销。
*   **算法流程**：在每一步生成中，检测新 Token 是否为锚点 $\rightarrow$ 更新后续 Token 的赞助分 $\rightarrow$ 计算所有 Token 的效用评分 $\rightarrow$ 保留评分最高的 $K$ 个 Token，剔除其余部分。
