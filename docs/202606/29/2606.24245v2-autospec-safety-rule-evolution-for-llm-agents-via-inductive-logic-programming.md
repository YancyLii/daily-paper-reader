## 摘要
大语言模型（LLM）智能体通过将语言模型与外部工具和环境集成，越来越多地自动化复杂任务。然而，其自主性带来了重大安全风险：智能体可能执行破坏性命令、泄漏敏感数据或违反领域约束。现有的安全方法面临一个根本性权衡：手工制定的规则可解释但脆弱，过于保守的规则会阻止安全操作（高假阳性），而宽松的规则会遗漏不安全行为（高假阴性）。神经分类器缺乏安全关键部署所需的可解释性。我们提出AutoSpec，一个通过反例引导的归纳合成（CEGIS）在归纳逻辑程序设计（ILP）指导下，根据用户的安全/不安全标注自动进化已部署的专家设计安全规则的框架。从专家规则和一系列带标注的执行轨迹开始，AutoSpec迭代地评估规则、挖掘假阳性和假阴性反例、使用ILP学习可以区分它们的谓词、生成候选规则编辑，并验证候选项以选择最佳修订。关键洞见在于，ILP能够高效识别出在假阴性中频繁出现但在假阳性中很少出现（或反之）的谓词，从而极大地裁剪了规则编辑的指数级搜索空间。这个过程持续到收敛，生成可解释的规则，平衡了精确率和召回率。我们在涵盖代码执行和具身智能体领域的291条执行轨迹上评估AutoSpec。AutoSpec将两个领域的规则F1分别提升至0.98和0.93，在保持高召回率的同时实现了高达94%的假阳性减少，并在4–5次迭代内收敛。ILP引导的方法相比启发式CEGIS实现了最高4.8倍的F1提升。学习到的规则可读、可审计，并能泛化到未见过的场景。

## Abstract
Large language model (LLM) agents increasingly automate complex tasks by integrating language models with external tools and environments. However, their autonomy poses significant safety risks: agents may execute destructive commands, leak sensitive data, or violate domain constraints. Existing safety approaches face a fundamental tradeoff: hand-crafted rules are interpretable but brittle, with overly conservative rules blocking safe operations (high false positives) while permissive rules miss unsafe behaviors (high false negatives). Neural classifiers lack the interpretability required for safety-critical deployments.   We present AutoSpec, a framework that automatically evolves deployed expert-designed safety rules from user safe/unsafe annotations through counterexample-guided inductive synthesis (CEGIS) guided by inductive logic programming (ILP). Starting from the expert rules and a stream of annotated traces, AutoSpec iteratively evaluates rules, mines false-positive and false-negative counterexamples, uses ILP to learn which predicates discriminate them, generates candidate rule edits, and verifies candidates to select the best revision. The key insight is that ILP efficiently identifies predicates that appear frequently in false negatives but rarely in false positives (or vice versa), dramatically pruning the exponential search space of rule edits. This continues until convergence, producing interpretable rules that balance precision and recall.   We evaluate AutoSpec on 291 execution traces spanning code execution and embodied agent domains. AutoSpec raises rule F1 to 0.98 and 0.93 across the two domains, achieving up to 94% false positive reduction while maintaining high recall, and converges within 4-5 iterations. The ILP-guided approach achieves up to 4.8x higher F1 than heuristic CEGIS. The learned rules are human-readable, auditable, and generalize to unseen scenarios.

### 2. 论文提出的方法论

- **核心思想（ILP‑Guided CEGIS）：**
  - 将用户标注的不安全或安全动作视为反例（Counterexample）。对于被专家规则错误放行的不安全行为（假阴性，FN）和错误拦截的安全行为（假阳性，FP），通过归纳逻辑编程（ILP）自动发现能够区分两类错误的关键谓词，进而有指导性地生成规则编辑候选，经过迭代验证，收敛到高 F1 的规则集。
- **关键技术细节：**
  - **规则表示：** 安全规则基于谓词库P中的布尔谓词构建，规则在事件级别触发，轨迹安全由任一规则触发判定。编辑操作包括：添加合取项（AddConjunct）、添加例外（AddException）、放松（Relax）、添加析取分支（AddDisjunct）。
  - **四阶段迭代流程（Algorithm 1）：**
    1.  **评估与反例挖掘：** 使用当前规则集 R 对标注轨迹 D 分类，找出 FN 和 FP。
    2.  **ILP引导的谓词学习：** 将 FN 作为正例，FP 作为负例，背景知识为轨迹中各事件的谓词取值，调用 ILASP 求解器学习一个能覆盖正例且不覆盖负例的最简谓词组合。从该组合中提取区分性谓词，并确定其编辑方向（用于覆盖 FN 则提升召回，用于排除 FP 则提升精确）。
    3.  **候选生成与选择：** 根据 ILP 建议的谓词和方向，通过单步编辑、混合修复链、失败驱动分支等策略生成多个候选规则集合。使用字典序分数 `S(R') = (F1, Prec, -U, -|R'|)` 在标注集上评估并贪心选择最佳候选。
    4.  **验证与终止：** 若候选分数提升则接受，否则停止迭代。最终收敛条件包括达到目标分数、无可改进候选或达到最大迭代次数。
  - **搜索空间剪枝：** ILP 通过在反例上直接学习区分性谓词，将指数级的编辑空间大幅缩减为一个短小的候选谓词列表，从而显著优于启发式或随机搜索。

### 3. 实验设计

- **数据集与场景：**

### 7. 优点

- **问题定义新颖：** 首次将安全规则维护建模为一个连续的、标注驱动的进化问题，贴合实际部署需求。
- **方法设计巧妙：** 将 ILP 集成到 CEGIS 循环中，利用其从少量反例中直接学习区分性逻辑公式的能力，高效地裁剪庞大的编辑空间。编辑算子设计精炼，仅需少数几种操作即可覆盖扩缩规则的路径。
- **平衡可解释性与自适应性：** 输出是可直接部署、可审计的符号规则，同时具备了学习系统的自适应能力，是该领域一个关键权衡的突破。
- **实验评估全面：** 覆盖代码和物理具身两个领域，包含规则质量、搜索效率、泛化能力、人为可用性及消融研究，形成了一个完整且令人信服的评估链。
- **轻量且部署友好：** 无需 GPU，计算开销极小，可异步运行，不侵入智能体执行路径，易于与现有分层防御架构集成。

### 8. 不足与局限

- **领域覆盖有限：** 仅验证了代码执行和具身智能体两个领域，对于其他常见智能体形态（如 Web 浏览器、数据库操作智能体）的表现有待验证。
- **标注依赖：** 依赖于轨迹级别的安全标注，若标注噪声大或分布不均（如某些不安全模式极少出现），可能影响 ILP 的学习质量。
- **谓词库依赖性：** 方法的效果高度依赖于初始谓词库的覆盖度。对于谓词无法表达的安全属性，系统无法学习出相应规则。虽然论文提到未解决的反例会指明缺失谓词，但这仍是一个潜在瓶颈。
- **时序属性支持：** 当前谓词评估基于单个事件和局部上下文，虽然可以内嵌历史/状态，但缺乏原生的时序算子（如 LTL）。论文将此列为未来工作。
- **最优性保证：** 因底层是 NP-hard 问题，算法仅保证单调改进和局部最优，不保证找到全局最优规则集。
