## 摘要
提示组合式智能体系统的实践者报告了一种反复出现的失效模式：编辑一个提示模块会无声地改变其他模块的行为，尽管它们之间没有共享变量或可执行依赖关系。我们将此现象形式化为组合行为泄漏（CBL）：共享上下文窗口的模块间的干扰。CBL 由架构上的非隔离性导致：Transformer 自注意力机制在串联模块之间没有提供形式上的边界。我们通过一个可复用的三通道协议，在一个已部署的职位评估智能体（Claude Sonnet 4.6，144 次试验）上探测了 CBL，该协议从体量、内容和形式三个方面扰动非焦点模块。只有内容通道产生了可检测的配对效应（Cohen's d = 0.63，自助法 95% 置信区间不含零）；没有推荐结果发生翻转——这是一种低于标准问答阈值的状态，但会在部署智能体所做出的数千次决策中累积。CBL 与已知的智能体失效维度（对抗注入、认知退化、多智能体故障传播、隐私泄漏）正交。我们贡献了一个操作型定义、一套可复用协议、一组可证伪的预测以及系统类别特征描述，确立了跨模块干扰测量作为提示组合式智能体评估的必要条件。

## Abstract
Practitioners of prompt-composed agentic systems report a recurring failure mode: editing one prompt module silently shifts the behavior of others despite no shared variable or executable dependency. We formalize this as compositional behavioral leakage (CBL): interference between modules sharing a context window. CBL is enabled by architectural non-isolation: transformer self-attention provides no formal boundary between concatenated modules. We probe CBL on a deployed job-evaluation agent (Claude Sonnet 4.6, 144 trials) through a reusable three-channel protocol that perturbs non-focal modules along volume, content, and form. Only the content channel produces a detectable paired effect (Cohen's d = 0.63, bootstrap 95% CI excluding zero); no recommendation flipped -- a sub-threshold regime invisible to standard QA but compounding across the thousands of decisions a deployed agent makes. CBL is orthogonal to known agent-failure axes (adversarial injection, cognitive degradation, multi-agent fault propagation, privacy leakage). We contribute an operational definition, a reusable protocol, a falsifiable prediction set, and a system-class characterization, establishing cross-module interference measurement as a requirement for prompt-composed agent evaluation.

## 1. 论文的核心问题与整体含义
- **研究背景**：提示组合式智能体系统（prompt‑composed agentic systems）通过将自然语言模块（如人格、技能、规则）在运行时拼接为 LLM 的上下文来驱动行为，已被广泛采用（如 OpenClaw、AGENTS.md 等）。然而，实践者反复观察到一种隐性故障：编辑某个模块（如技能定义）后，会无声地改变其他完全无关模块的输出，即使它们之间没有共享变量或代码依赖。
- **核心问题**：该现象被作者形式化为 **组合行为泄漏（Compositional Behavioral Leakage, CBL）**——共享同一上下文窗口的提示模块之间发生的非预期行为干扰。其根源在于 Transformer 自注意力机制的 **架构非隔离性**：跨模块的全局注意力没有形式化的模块边界，分隔符（如 Markdown 标题）仅是书写约定，无法保证行为独立。
- **整体含义**：CBL 与已知的智能体失效模式（对抗注入、认知退化、多智能体故障传播、隐私泄漏）正交，是一种全新的评估维度。子阈值漂移虽不触发决策翻转，却会在成千上万次调用中累积，带来难以察觉的可靠性风险。因此，跨模块干扰测量必须成为提示组合式智能体评测的**必要条件**。

## 2. 论文提出的方法论
### 2.1 操作型定义
设 \(M = \{m_1, \dots, m_n\}\) 为部署的提示模块集，经拼接后送入 LLM 的上下文。对非焦点模块 \(m_j\) 做编辑 \(e\)，得到 \(M(j \leftarrow e)\)。焦点模块 \(m_i\) 针对输入 \(x\) 的可观测得分或决策记作 \(s_i(x; M)\)。**如果编辑 \(e\)（原本不应影响 \(m_i\) 的行为）导致配对偏移的 bootstrap 置信区间不含零，则系统表现出 CBL：**
\[
\Delta_i(e) = \mathbb{E}_x\left[ s_i(x; M(j \leftarrow e)) - s_i(x; M) \right]
\]
更严格的组合一致性检验 \(B(m_i \mid M) \approx B(m_i \mid \{m_i\})\) 可视为该回归观点的特例。本文聚焦于**部署系统场景**：局部修改非焦点模块是否会在完整部署提示下悄然改变焦点模块行为。

### 2.2 三通道扰动协议
为区分不同干扰来源，作者设计了可复用的三通道协议，对非焦点模块施加三种扰动：
- **C1 体量通道（Volume）**：增加一个完全无关的 200 行“食谱评估”模块，仅改变上下文长度。
- **C2 内容通道（Content）**：在共享规则文件中添加一个语义上与所有目标职位无关的“专业厨师”原型及评价说明。
- **C3 形式通道（Form）**：对非焦点模块进行意义保留的结构性变化（标题层级、列表格式、emoji 标记、章节顺序重排）。
基线 **C0** 为部署时的原始系统提示。C2 是主要测试条件；C1 和 C3 作为对照，排除“任意额外上下文导致退化”或“格式敏感性”的解释。

### 2.3 统计推断
- **效应量**：Cohen’s \(d\)，基于职位级别配对均值（\(N = 12\)）。
- **置信区间**：对 \(\Delta\) 进行 10,000 次自助重采样，计算 95% CI。
- **辅助检验**：Wilcoxon 符号秩检验，Bonferroni 校正。

## 3. 实验设计
- **被测试系统**：`career-ops`，一个已部署的提示组合式职位评估智能体，其焦点模块对职位描述（JD）在多个维度上打分（1–5）。主要结局变量为 **cv‑match** 评分，即候选人与岗位要求对齐程度。
- **模型与环境**：Claude Sonnet 4.6（通过 API 调用）。无训练，仅推理。
- **实验素材**：12 份真实职位描述；每个条件独立运行 3 次，总计 144 次试验。
- **对比条件**：基线 C0 与三种扰动条件 C1、C2、C3。比较方法为同一 JD 在不同条件下的配对得分差异，聚焦于 cv‑match 维度。
- **Benchmark 与基线**：不存在标准 benchmark，本文以部署系统的原始提示作为唯一基线，通过内置修改构造多条件对比。

## 5. 实验数量与充分性
- **实验数量**：12 个职位描述 × 4 个条件（C0–C3）× 3 次运行 = **144 次试验**。每个条件内部不同运行间的组内相关系数（ICC）高达 0.925，保证了配对分析的统计效力。
- **充分性讨论**：
  - 作为**概念证实**（existence proof），实验规模足以展示 CBL 现象，且主要结果（C2 的 d = 0.63，CI 不含零）具有统计学意义。
  - 但样本量（12 个 JD）较小，仅覆盖一个系统和一个模型，无法代表所有提示组合式智能体。作者明确指出其性质为“存在性质的证明”，并提出了**多系统、多模型的复制计划**和三条可证伪预测，需要后续工作来检验通用性。
  - 对照设计（C1、C3 未出现显著效应）有效排除了体积和格式的解释，强化了内容通道干扰的归因，增强了实验的客观性。
- **公平性**：使用同一模型、相同评分标准、随机化运行顺序（隐含），统计方法透明，无数据依赖偏差。但 API 模型的不透明性限制了内部因果机制的深度解释。

## 6. 论文的主要结论与发现
- **存在 CBL**：仅在语义内容通道（C2）检测到显著的配对效应，Cohen’s \(d = 0.63\)，\(\Delta = +0.17\)，bootstrap 95% CI \([+0.03, +0.31]\) 不含零。这意味着添加一个完全不相关的原型能系统性提高 cv‑match 评分。
- **对照条件无显著干扰**：C1 体量增加（d = –0.11）和 C3 格式改变（d = –0.29）的置信区间均包含零，排除了“上下文变长导致普遍退化”或“格式敏感”的竞争解释，将泄漏锁定在语义内容层面。
- **子阈值漂移**：没有任何推荐结果发生反转，但分数分布发生系统性偏移——这是一种**低于常规问答评估阈值的状态**，在多步决策中可能累积成高影响误差。
- **系统类特征**：将 CBL 与已有失效模式区分开，确立了提示组合式智能体作为一个独特的系统类别（行为逻辑以文本形式由注意力解析）。
- **测试框架**：提出针对提示模块的四类回归测试：组合一致性、模块交互回归、格式扰动鲁棒性和模型迁移测试，并指出当前所有被调研系统（OpenClaw、career‑ops、OpenHands、aider）均未实施这些测试。

## 7. 优点
- **问题新颖且务实**：捕获了实际大规模部署中反复出现的“指令渗漏”现象，并给予严格定义，填补了评测盲区。
- **协议设计精巧**：三通道扰动有效分离体量、内容、形式的影响，通过明确对照排除干扰；统计方法严谨（自助法置信区间 + 效应量），可复用性强。
- **可证伪预测**：为社区提供了具体、可检验的假说（跨模型差异、语义距离梯度、阈值突破缩放），无需新测量设施即可在现有系统产物上开展复制研究。
- **操作型定义与系统定位**：将 CBL 与注入、退化、隐私泄漏明确区分，厘清了概念边界，有助于后续标准化评测。

## 8. 不足与局限
- **单一系统和模型**：仅基于一个评估智能体与一个闭源模型，结果未必能推广到不同领域、不同架构或不同提示组合风格的智能体。
- **样本规模有限**：12 个职位描述的样本量虽然在统计上足以检出中等到大效应，但覆盖面窄，无法刻画更细粒度的交互（如不同提示模块组合的交互效应）。
- **缺乏内部机制分析**：依赖 API，无法进行注意力权重、隐藏状态等内部分析，只能从行为结果推断干扰，留下了“为什么”的空白。
- **测试可行性的挑战**：作者虽提出了组合空间巨大（如 OpenClaw 的 10K+ 技能）的回归测试需求，但仅给出采样策略方向，未实现或验证实际可操作的监控方案。
- **实验设置单一**：扰动幅度（仅添加一个原型）和类型固定，未探索不同干预强度、语义距离或模块顺序的梯度变化，暂时无法估计 CBL 在多大程度上会被放大或抑制。
- **闭源模型影响**：结果对特定模型版本敏感，模型更新后可能改变；作者也承认模型迁移测试是必需的，但未在此实验中展示。
