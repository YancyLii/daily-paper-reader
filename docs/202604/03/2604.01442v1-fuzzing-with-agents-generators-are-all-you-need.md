## 摘要
现代基于生成器的模糊测试技术将轻量级输入生成器与覆盖率引导的变异相结合，作为探索目标程序深层执行路径的一种方法。先前研究中的一种补充方法侧重于创建高度定制的、特定领域的生成器，这些生成器编码了足以到达深层程序状态的结构和语义逻辑；其挑战在于编写和测试这些复杂生成器的开销。我们研究了 AI 编程智能体是否能够自动合成此类针对特定目标的生成器，以及生成的生成器是否足够强大，从而完全消除对覆盖率引导和变异的需求。我们的方法 Gentoo 由一个大语言模型（LLM）编程智能体组成（该智能体被授予终端访问权限以及模糊测试目标及其库的源代码），指令其迭代地合成和优化输入生成器，并可选地提供细粒度的谓词级覆盖率反馈。我们在 7 个真实 Java 库的模糊测试目标上，针对人工编写的生成器评估了 Gentoo 的三种配置。研究结果表明，在 7 个基准测试中的 4 个上，智能体合成的生成器比人工编写的基准生成器实现了统计学上显著更高的分支覆盖率。关键的是，覆盖率引导和变异策略的使用对于智能体合成的生成器在统计学上并无显著益处，但对所有人工编写的生成器却意义重大，这表明智能体生成器中编码的结构和语义逻辑使得覆盖率引导在很大程度上变得不再必要。

## Abstract
Modern generator-based fuzzing techniques combine lightweight input generators with coverage-guided mutation as a method of exploring deep execution paths in a target program. A complimentary approach in prior research focuses on creating highly customized, domain-specific generators that encode structural and semantic logic sufficient enough to reach deep program states; the challenge comes from the overhead of writing and testing these complex generators. We investigate whether AI coding agents can automatically synthesize such target-specific generators, and whether the resulting generators are strong enough to obviate the need for coverage guidance and mutation entirely.   Our approach, Gentoo, is comprised of an LLM coding agent (provided terminal access and source code of the fuzz target and its library) instructed to iteratively synthesize and refine an input generator, and optionally provided fine-grained predicate-level coverage feedback. We evaluate three configurations of Gentoo against human-written generators on fuzz targets for 7 real-world Java libraries. Our findings show that agent-synthesized generators achieve statistically significantly higher branch coverage than human-written baseline generators on 4 of 7 benchmarks. Critically, the use of coverage guidance and mutation strategies is not statistically significantly beneficial for agent-synthesized generators, but is significant for all human-written generators, suggesting that structural and semantic logic encoded in the agent generators makes coverage guidance largely unnecessary.

#### 1. 核心问题与整体含义（研究动机和背景）
模糊测试（Fuzzing）的核心挑战在于生成既符合结构规范又具备语义意义的输入。传统的**覆盖率引导的灰盒模糊测试（CGF）**依赖于随机变异，难以突破深层逻辑检查；而**基于生成器的模糊测试**虽然能生成结构化输入，但编写高质量、针对特定领域的生成器（如 CSmith）需要极高的人工成本。
本论文探讨了：**AI 编程智能体（Agent）能否自动合成这些复杂的生成器？** 进一步地，如果智能体合成的生成器足够强大，是否可以完全取代复杂的覆盖率引导和变异机制？

#### 2. 方法论：Gentoo 框架
论文提出了 **Gentoo** 系统，利用 LLM 编程智能体自动迭代开发 JQF 风格的 Java 输入生成器。
*   **核心思想**：赋予智能体终端访问权限、目标源码和迭代反馈，使其像人类开发者一样编写、测试和调试生成器。
*   **关键技术细节**：
    *   **谓词反馈（Predicate Feedback）**：不仅告诉智能体是否覆盖了某行，还提供细粒度的分支信息。
    *   **静态支配者分析（Static Dominator Analysis）**：利用控制流图（CFG）计算“分支支配得分”，识别出控制最多未探索代码的“守门员”谓词。
    *   **三种配置**：
        1.  **Gentoo-Base**：基础智能体，仅通过模糊测试统计数据和观察输入进行优化。
        2.  **Gentoo-S**：引入静态分析工具生成的谓词记录，指导智能体关注高价值分支。
        3.  **Gentoo-L**：由 LLM 自身分析源码识别重要谓词，无需外部静态分析工具。
*   **迭代流程**：阅读源码 -> 合成初始生成器 -> 运行短时模糊测试 -> 获取谓词覆盖报告 -> 识别未覆盖的高价值分支 -> 细化生成器代码。

#### 3. 实验设计
*   **数据集/场景**：选择了 7 个真实的 Java 库，涵盖字节码解析（BCEL）、加密协议（Bouncy Castle）、编译器（ChocoPy, Google Closure）、压缩（BZip2）、JSON 解析（Gson）和 JS 解释器（Rhino）。
*   **Benchmark**：对比了先前研究（Zest, JQF 等）中由**人类专家编写的生成器**。
*   **对比维度**：
    *   智能体生成器 vs. 人类生成器。
    *   覆盖率引导模式（Zest 算法）vs. 纯随机采样模式。
    *   不同 Gentoo 配置间的消融实验。

#### 6. 主要结论与发现
*   **超越人类**：在 7 个基准测试中的 4 个上，智能体合成的生成器在分支覆盖率上**显著优于**人类编写的基准（平均提升 11%–21%）。
*   **“生成器即所需”**：对于智能体合成的生成器，**覆盖率引导和变异几乎没有额外收益**（提升通常小于 3% 且不显著）。这表明智能体已经将复杂的结构和语义逻辑直接编码进了生成器中，使其不再依赖模糊测试引擎的变异能力。
*   **反馈的价值**：谓词反馈在 74% 的运行中能有效改进生成器，但在最终覆盖率上，基础智能体表现也相当强劲。

#### 7. 优点（亮点）
*   **范式转变**：提出了从“改进模糊测试引擎”到“利用 Agent 自动合成高质量生成器”的思路转变。
*   **自主性强**：Gentoo 智能体具备完整的“编写-编译-运行-调试”闭环能力，能够处理复杂的语义约束（如 ChocoPy 的类型系统）。
*   **实用性**：证明了在无需复杂变异算法的情况下，仅靠高质量生成器即可达到极深的代码覆盖。

#### 8. 不足与局限
*   **API 发现能力不足**：在 BCEL 测试中，智能体未能发现库自带的高级构建 API，而是尝试手动编写字节流，导致表现远逊于人类。
*   **成本与速度**：虽然比人工便宜，但调用顶级 LLM 仍有一定金钱成本；且智能体生成的复杂输入有时会导致执行吞吐量（Execs/s）下降。
*   **潜在偏差**：LLM 在预训练阶段可能见过这些开源库的代码，虽然论文通过检查生成代码的差异性进行了部分缓解，但无法完全排除数据泄露风险。
*   **应用限制**：目前仅在 Java 环境下验证，对于 C/C++ 等内存管理更复杂或缺乏高质量反射/分析工具的语言，效果尚待观察。
