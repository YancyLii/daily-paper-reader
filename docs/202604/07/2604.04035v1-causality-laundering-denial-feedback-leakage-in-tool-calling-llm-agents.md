## 摘要
调用工具的大语言模型（LLM）智能体可以读取私有数据、调用外部服务并触发真实世界的动作，这在工具执行环节产生了安全问题。我们识别出一种拒绝反馈泄露模式，并将其称为“因果洗钱”（causality laundering）。在这种模式下，攻击者探测受保护的操作，从拒绝结果中学习，并通过随后看似无害的工具调用将推断出的信息外泄。仅靠扁平的溯源追踪（flat provenance tracking）无法捕获此类攻击，因为泄露的信息源于被拒绝操作的因果影响，而非直接的数据流。我们提出了智能体引用监视器（Agentic Reference Monitor, ARM），这是一个运行时强制执行层，通过咨询涵盖工具调用、返回数据、字段级溯源和被拒绝操作的溯源图来调解每一次工具调用。ARM 通过完整性格（integrity lattice）传播信任，并利用来自被拒绝操作节点的反事实边（counterfactual edges）增强溯源图，从而实现对传递性数据依赖和拒绝诱导的因果影响的强制管控。在针对三个代表性攻击场景的受控评估中，ARM 拦截了扁平溯源基准模型所遗漏的因果洗钱、传递性污点传播以及混合溯源字段滥用，同时仅增加了亚毫秒级的策略评估开销。这些结果表明，感知拒绝的因果溯源是保障调用工具的智能体系统安全的一种有效抽象。

## Abstract
Tool-calling LLM agents can read private data, invoke external services, and trigger real-world actions, creating a security problem at the point of tool execution. We identify a denial-feedback leakage pattern, which we term causality laundering, in which an adversary probes a protected action, learns from the denial outcome, and exfiltrates the inferred information through a later seemingly benign tool call. This attack is not captured by flat provenance tracking alone because the leaked information arises from causal influence of the denied action, not direct data flow. We present the Agentic Reference Monitor (ARM), a runtime enforcement layer that mediates every tool invocation by consulting a provenance graph over tool calls, returned data, field-level provenance, and denied actions. ARM propagates trust through an integrity lattice and augments the graph with counterfactual edges from denied-action nodes, enabling enforcement over both transitive data dependencies and denial-induced causal influence. In a controlled evaluation on three representative attack scenarios, ARM blocks causality laundering, transitive taint propagation, and mixed-provenance field misuse that a flat provenance baseline misses, while adding sub-millisecond policy evaluation overhead. These results suggest that denial-aware causal provenance is a useful abstraction for securing tool-calling agent systems.

这篇论文由独立研究员 Mohammad Hossein Chinaei 于 2026 年发表，探讨了工具调用型大语言模型（LLM）智能体中一种新型且隐蔽的安全威胁。以下是对该论文的结构化深度总结：

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的 LLM 智能体安全防御机制（如污点追踪、数据流分析）主要关注**成功的工具调用**和**返回的数据流**。然而，论文发现**被拒绝的操作（Denied Actions）**本身也会泄露敏感信息。
*   **因果洗钱（Causality Laundering）**：这是一种新型攻击模式。攻击者诱导智能体执行一个会被拦截的受保护操作（如读取敏感文件），智能体通过“拒绝反馈”（如“权限拒绝”而非“文件不存在”）推断出敏感信息，随后通过另一个看似无害的合法工具调用（如发送邮件）将该推断外泄。
*   **研究背景**：随着 MCP（Model Context Protocol）等协议的普及，智能体权限日益扩大。传统的扁平溯源（Flat Provenance）无法捕捉这种不经由数据流、而经由“因果影响”产生的泄露。

### 2. 方法论：Agentic Reference Monitor (ARM)
*   **核心思想**：将“被拒绝的操作”视为溯源图中的一等公民，通过引入“反事实边”来追踪拒绝行为对后续操作的潜在因果影响。
*   **关键技术细节**：
    *   **分层策略管道**：ARM 包含四层防御：
        1.  **硬边界层 (L1)**：执行无条件检查（如敏感路径过滤、凭据检测、负载大小限制）。
        2.  **图感知溯源层 (L2G)**：核心层，通过溯源图查询信任传播和反事实链。
        3.  **模式衍生约束层 (L3)**：根据工具的 JSON Schema 自动生成防御规则（如限制可选参数长度）。
        4.  **手动策略层 (L4)**：执行操作员定义的基于能力的访问控制。
    *   **溯源图 (Provenance Graph)**：节点包括工具调用、返回数据、数据字段和**拒绝动作**。边包括直接输出、输入、字段归属以及关键的**反事实边 (Counterfactual Edge)**。
    *   **完整性晶格 (Integrity Lattice)**：定义了五个信任等级（系统指令 > 用户输入 > 信任工具 > 不信任工具 > 工具描述）。信任在图中按“最小化原则”传播。
    *   **反事实启发式**：当一个动作被拒绝后，ARM 会自动在拒绝节点与紧随其后的下一个工具调用之间建立反事实边，从而标记潜在的因果泄露。

### 3. 实验设计
*   **实验场景**：设计了三个具有代表性的攻击场景：
    1.  **因果洗钱**：探测敏感文件是否存在，被拒绝后通过邮件外泄推断结果。
    2.  **传递性污点链**：通过中间工具（如格式化工具）对不信任数据进行“洗稿”，试图绕过简单的来源检查。
    3.  **混合溯源字段误用**：利用结构化输出中受信任字段（如姓名）和不受信任字段（如邮箱）的混合来实施攻击。
*   **基准模型 (Benchmark)**：**扁平溯源基准 (Flat Provenance)**。该基准仅检查参数值是否出现在历史记录中，不具备图结构或拒绝感知能力。
*   **对比方法**：对比了扁平溯源层（L2）与图感知溯源层（L2G）在相同 L1、L3、L4 约束下的表现。

### 4. 资源与算力
*   **硬件环境**：实验在配备 **Apple M4 Pro 芯片、48 GB RAM** 的设备上运行。
*   **性能表现**：ARM 的策略评估开销极低，中位延迟**小于 1 毫秒**（针对包含数十个调用、数百个节点的会话图）。
*   **训练时长**：ARM 是一个确定性的运行时监控器，**不需要训练**，因此不涉及 GPU 训练时长。

### 5. 实验数量与充分性
*   **实验规模**：
    *   针对 3 个核心攻击场景进行了详细的对比测试。
    *   包含一个由 **45 个单元测试**组成的测试套件，覆盖了溯源图构建、信任传播、反事实链检测等。
*   **充分性评价**：作为原型验证，实验在**功能覆盖**上是充分的，成功证明了图感知模型在处理复杂因果关系上的优越性。但由于是受控实验，缺乏在大规模真实世界数据集（如 AgentDojo）上的泛化性评估。

### 6. 论文的主要结论与发现
*   **拒绝反馈是真实的安全威胁**：实验证明，仅拦截非法调用是不够的，必须监控拦截行为对智能体后续决策的影响。
*   **图结构优于扁平历史**：通过图遍历可以有效识别传递性污点和字段级风险，而扁平模型在这些场景下全部失效。
*   **确定性防御的可行性**：ARM 证明了可以在不引入额外 LLM 推理开销的情况下，通过确定性的图算法实现高级别的安全保障。

### 7. 优点
*   **视角独特**：首次形式化了“因果洗钱”概念，填补了智能体安全领域对拒绝反馈研究的空白。
*   **性能卓越**：亚毫秒级的延迟使其非常适合集成到实时生产环境中。
*   **防御深度**：分层架构结合了硬编码规则、自动衍生约束和动态溯源分析，提供了多重保障。
*   **确定性**：不依赖 LLM 来检测攻击，避免了防御者自身被注入或绕过的风险。

### 8. 不足与局限
*   **启发式偏差**：反事实边基于“时间邻近性”建立，这可能导致**误报**（拦截了紧随拒绝后的正常调用）或**漏报**（攻击者通过多步无关操作延迟外泄）。
*   **字段级标注依赖**：字段级溯源的有效性高度依赖于集成者对工具输出结构的准确标注。
*   **单智能体限制**：目前仅针对单智能体场景，尚未解决多智能体协作中的复杂因果溯源问题。
*   **评估规模**：缺乏大规模、自动化的红队测试或真实攻击样本库的验证。
