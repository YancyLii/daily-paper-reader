## 摘要
大语言模型越来越多地在来自具有不同权威级别的异构来源的多重指令下运行，包括系统策略、用户请求、工具输出和检索到的上下文。虽然先前关于指令层级的工作强调了尊重指令优先级的重要性，但其主要关注对抗性攻击，而忽视了现实应用中出现的良性但常见的指令冲突。在这些场景下，当指令发生部分或隐式冲突时，模型不仅必须避免安全违规，还必须保持任务效用和行为一致性。我们提出了神经符号层级对齐（NSHA），通过显式建模和强制执行指令优先级来实现层级指令遵循。在推理阶段，我们引入了求解器引导的推理，将指令解析公式化为一个约束满足问题，使模型能够在层级约束下推导出最大一致的适用指令集。在训练阶段，NSHA 利用自动构建的监督信号，将基于求解器的决策蒸馏到模型参数中。我们在规则遵循、任务执行、工具使用和安全性方面评估了我们的方法，涵盖了单轮和多轮交互，并表明 NSHA 在此类冲突下显著提高了性能，同时在参考设置中保持了具有竞争力的效用。

## Abstract
Large language models increasingly operate under multiple instructions from heterogeneous sources with different authority levels, including system policies, user requests, tool outputs, and retrieved context. While prior work on instruction hierarchy highlights the importance of respecting instruction priorities, it mainly focuses on adversarial attacks and overlooks the benign but common instruction conflicts that arise in real-world applications. In such settings, models must not only avoid security violations but also preserve task utility and behavioral consistency when instructions partially or implicitly conflict. We propose Neuro-Symbolic Hierarchical Alignment (NSHA) for hierarchical instruction-following by explicitly modeling and enforcing instruction priorities. At inference time, we introduce solver-guided reasoning that formulates instruction resolution as a constraint satisfaction problem, enabling the model to derive a maximally consistent set of applicable instructions under hierarchical constraints. At training time, NSHA distills solver-based decisions into model parameters using automatically constructed supervision. We evaluate our approach on rule following, task execution, tool use, and safety, covering both single-turn and multi-turn interactions, and show that NSHA significantly improves performance under such conflicts while maintaining competitive utility in reference settings.

随着 LLM 应用于复杂代理（Agent）场景，模型需同时处理来自系统策略、用户请求、工具输出和检索上下文（RAG）的指令。这些指令往往具有不同的**权威等级（Authority Levels）**。
*   **核心问题**：当不同来源的指令发生冲突时（例如系统要求输出 JSON，用户要求输出纯文本），模型往往无法正确识别优先级，导致行为失准或安全违规。
*   **研究动机**：现有研究多关注对抗性攻击（如提示词注入），忽视了现实应用中常见的“良性冲突”。论文旨在通过逻辑一致性显式建模指令优先级，确保模型在复杂环境下依然能严格遵循高优先级指令。

### 2. 论文提出的方法论：神经符号层级对齐（NSHA）
论文提出了 **NSHA** 框架，结合了符号逻辑的严谨性和神经网络的泛化性，分为推理阶段和训练阶段：

*   **推理阶段（Solver-guided Inference）**：
    1.  **指令原子化（Atomizer）**：将原始上下文分解为离散的原子指令，并标注权威等级（系统 > 用户 > 工具/文档）。
    2.  **神经冲突扫描（Neural Conflict Scanning）**：利用 NLI（自然语言推理）模型或小型 LLM 检测指令对之间的语义冲突，构建冲突矩阵。
    3.  **层级优化求解（Z3 Optimizer）**：将冲突解析建模为 **MaxSMT（最大约束满足）问题**。利用 Z3 求解器在保证高优先级指令的前提下，推导出最大一致的适用指令子集。
    4.  **求解器引导生成**：仅使用筛选后的指令集引导模型生成最终回复。

*   **训练阶段（Hierarchical Alignment Training）**：
    *   **数据构建**：利用求解器自动标注 Alpaca 等数据集，构建包含“对齐”和“冲突”场景的偏好对。
    *   **核心算法（HCAL 损失）**：提出**层级一致性对齐损失（Hierarchy-Consistent Alignment Loss）**。该损失函数结合了标准偏好损失（Lpref）、语义损失（LSL，惩罚违反层级的概率）和 KL 散度约束，将求解器的逻辑推理能力蒸馏到模型参数中。

### 3. 实验设计
*   **数据集与场景**：采用 **IHEval** 协议，涵盖四个领域：
    *   **规则遵循**：单轮/多轮格式约束。

    *   **任务执行**：语言检测、翻译、动词提取等基础任务。
    *   **安全性**：防御提示词劫持（Hijack）和系统提示词提取。
*   **Benchmark**：对比了 Reference（无冲突）、Aligned（协同）和 Conflict（冲突）三种设置。
*   **对比方法**：Base 模型、CoT（思维链）、NS（纯推理版神经符号）、NSHA-SFT、NSHA-DPO 以及 NSHA-HCAL。
*   **实验模型**：Qwen3-4B-it、Llama-3.1-8B、Qwen-2.5-14B-it。

### 4. 资源与算力
*   **硬件**：使用 **NVIDIA A100 (80GB)** GPU。
*   **配置**：
    *   较小模型（4B/8B）使用单张 A100。

### 5. 实验数量与充分性
*   **实验规模**：构建了 **43,380** 个训练样本（30,594 个冲突案例，12,786 个对齐案例）。
*   **充分性**：实验涵盖了从 4B 到 14B 不同规模的骨干模型，测试了单轮与多轮对话的鲁棒性，并针对系统/用户指令的强弱进行了消融实验（System/User Strength Swaps）。
*   **客观性**：通过引入“参考设置（Reference）”确保了模型在提升冲突处理能力的同时，没有牺牲基础的通用任务能力。

### 6. 论文的主要结论与发现
*   **冲突是性能杀手**：在冲突场景下，原生模型（如 Llama 3.1）的准确率会从 80%+ 骤降至 20% 以下。
*   **NSHA 效果显著**：NSHA-DPO 在多数任务中表现最优，显著提升了模型在冲突下的规则遵循和安全性。
*   **多轮一致性**：实验发现，模型在多轮对话中容易出现“层级漂移”，而 NSHA 能有效维持长程的系统优先级。
*   **能力边界**：虽然 NSHA 提升了鲁棒性，但最终性能仍受限于骨干模型的基础推理能力（Base Capability）。

### 7. 优点
*   **逻辑严密性**：引入 SMT 求解器为指令遵循提供了可解释的符号化保障。
*   **训练与推理解耦**：通过 HCAL 损失将复杂的逻辑约束蒸馏进模型，使得模型在推理时无需依赖外部求解器即可获得高性能。
*   **场景全面**：不仅关注安全攻击，更深入探讨了实际应用中复杂的良性指令冲突。

### 8. 不足与局限
*   **冲突检测成本**：推理阶段依赖 GPT-5-mini 等模型进行冲突检测，增加了推理延迟和成本。
*   **原子化依赖**：指令原子化（Atomizer）目前依赖手工规则和启发式方法，对于极度模糊或隐含的指令可能拆解不准。
*   **蒸馏损失**：在 Llama 3.1-8B 上，NSHA-SFT 出现了明显的性能退化（Reference 分数下降），说明小模型在吸收复杂逻辑约束时存在稳定性风险。
