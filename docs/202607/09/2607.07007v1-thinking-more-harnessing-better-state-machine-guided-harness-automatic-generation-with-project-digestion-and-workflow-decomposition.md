## 摘要
高质量的模糊测试桩对于有效的灰盒模糊测试至关重要。虽然大型语言模型（LLMs）为自动化这一任务提供了可能，但现有的一轮生成方法由于粗粒度的函数定位和不一致的生成工作流，存在幻觉和覆盖率不足的问题。我们提出了SynapseFlow，一种自动测试桩生成器，通过两个关键创新来解决这些局限性：数据流感知的函数聚合，以及分阶段、支持回滚的生成工作流分解。SynapseFlow首先分析源代码，构建结构流图并提取连贯的函数三元组。然后，它通过一个分阶段回滚算法指导的四阶段分解过程来合成测试桩，以确保正确性。我们在25个真实世界的开源软件项目上评估了SynapseFlow。实验结果表明，SynapseFlow优于最先进的工具（OSS-Fuzz-Gen、CKGFuzzer、PromeFuzz），分支覆盖率分别提高了3.07$\times$、1.71$\times$和4.26$\times$，缺陷检测率分别提高了1.77$\times$、1.51$\times$和1.36$\times$。最重要的是，SynapseFlow发现了7个此前未被报告的缺陷（其中5个已分配CVE编号），证明了其在真实世界缺陷发现中的实际有效性。

## Abstract
High-quality fuzz harnesses are essential for effective gray-box fuzzing. While Large Language Models (LLMs) offer promise for automating this task, existing one-turn generation methods suffer from hallucinations and inadequate coverage due to coarse-grained function targeting and misaligned generation workflows. We present SynapseFlow, an automatic harness generator that addresses these limitations through two key innovations: dataflow-aware function aggregation and a staged, rollback-enabled generation workflow decomposition. SynapseFlow first analyzes source code to construct Structural Flow Graphs and extract coherent Function Triplets. It then synthesizes harnesses via a decomposed four-stage process governed by a staged rollback algorithm to ensure correctness. We evaluated SynapseFlow on 25 real-world open-source software projects. The experimental results indicate that SynapseFlow outperforms state-of-the-art tools (OSS-Fuzz-Gen, CKGFuzzer, PromeFuzz), achieving 3.07$\times$, 1.71$\times$, and 4.26$\times$ higher branch coverage, and 1.77$\times$, 1.51$\times$, and 1.36$\times$ higher bug detection rates, respectively. Most importantly, SynapseFlow discovered 7 previously unreported bugs (5 assigned CVEs), demonstrating its practical effectiveness in real-world bug discovery.

### 2. 论文提出的方法论

论文提出了 **SynapseFlow** 框架，包含两个核心阶段，将精细的程序分析与分阶段、可纠错的LLM生成相结合。

*   **阶段一：基于数据流聚合的函数分组**
    *   **核心思想**：摒弃孤立的函数测试，通过分析源码将功能相关、数据依赖的函数聚合为最小的、连贯的**函数三元组（Function Triplet, FT）**。
    *   **关键技术细节**：
        1.  **函数语义标注**：结合轻量级静态分析和LLM推理，将函数分为三类：
            *   **输入流函数 (ISF)**：消费非结构化外部数据（如`from_memory`）的入口函数。
            *   **处理函数 (PRF)**：对已初始化的数据结构进行变换或读取的函数。
            *   **辅助函数 (HPF)**：负责数据结构生命周期管理的函数（如分配、释放）。
        2.  **结构流图 (SFG) 构建**：构建一个有向图，节点为数据结构类型，边为代表函数，描述数据如何在函数间流转和转换。
        3.  **函数三元组 (FT) 提取**：以每个唯一的 ISF 为锚点，在SFG上执行前向/后向数据流分析，提取所有可达的、语义相关的PRF和HPF，形成一个`(ISF, Set<PRF>, Set<HPF>)`的FT。一个FT代表一个最小的、可自洽测试的功能单元。

*   **阶段二：带分阶段回滚的测试桩合成**
    *   **核心思想**：将一个复杂的测试桩生成任务分解为一系列更简单、可验证的子任务，并引入回滚机制来遏制和修复错误，避免LLM幻觉的级联放大。
    *   **四阶段生成工作流**：
        1.  **文档生成**：为FT内的每个函数生成结构化的API文档。
        2.  **片段缝合**：针对SFG中共享输入/输出结构的函数组，生成局部的、正确的代码片段。
        3.  **粗略代码组装**：按照数据流顺序，迭代合并所有局部片段，形成一个完整的、可执行的原型代码。
        4.  **代码优化与转换**：将原型代码封装为标准测试桩入口 `LLVMFuzzerTestOneInput`，删除文件IO等有害操作，完成最终代码。
    *   **分阶段回滚算法 (Staged Rollback Algorithm)**：
        *   **机制**：当最终测试桩（阶段4输出）无法通过编译或基本运行测试时，无需全部重来，而是**回滚到前一阶段（如阶段3）进行局部再生**。
        *   **优势**：这是一种成本效益高的错误恢复机制，能有效隔离并修正错误，避免因单一错误导致整体生成失败，显著节省了LLM推理时间和资源。
