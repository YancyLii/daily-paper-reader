## 摘要
大语言模型（LLMs）是回答用户查询的强大工具，但它们仍然极易受到越狱攻击的影响。现有的护栏（guardrail）方法通常依赖内部特征或文本响应来检测恶意查询，这要么会引入显著的延迟，要么会受到文本生成随机性的影响。为了克服这些局限性，我们提出了 SelfGrader，这是一种轻量级的护栏方法，它利用 Token 级 Logits 将越狱检测建模为一个数值评分问题。具体而言，SelfGrader 在一组紧凑的数值 Token（NTs）（例如 0-9）中评估用户查询的安全性，并将它们的 Logit 分布解释为内部安全信号。为了使这些信号与人类对恶意程度的直觉保持一致，SelfGrader 引入了一种双视角评分规则，同时考虑查询的恶意性和良性，从而产生一个既能反映有害性又能同时降低误报率的稳定且可解释的分数。在多种越狱基准测试、多个 LLM 以及最先进的护栏基准上进行的广泛实验表明，SelfGrader 在 LLaMA-3-8B 上的攻击成功率（ASR）最高降低了 22.66%，同时保持了显著更低的内存开销（最高降低 173 倍）和延迟（最高降低 26 倍）。

## Abstract
Large Language Models (LLMs) are powerful tools for answering user queries, yet they remain highly vulnerable to jailbreak attacks. Existing guardrail methods typically rely on internal features or textual responses to detect malicious queries, which either introduce substantial latency or suffer from the randomness in text generation. To overcome these limitations, we propose SelfGrader, a lightweight guardrail method that formulates jailbreak detection as a numerical grading problem using token-level logits. Specifically, SelfGrader evaluates the safety of a user query within a compact set of numerical tokens (NTs) (e.g., 0-9) and interprets their logit distribution as an internal safety signal. To align these signals with human intuition of maliciousness, SelfGrader introduces a dual-perspective scoring rule that considers both the maliciousness and benignness of the query, yielding a stable and interpretable score that reflects harmfulness and reduces the false positive rate simultaneously. Extensive experiments across diverse jailbreak benchmarks, multiple LLMs, and state-of-the-art guardrail baselines demonstrate that SelfGrader achieves up to a 22.66% reduction in ASR on LLaMA-3-8B, while maintaining significantly lower memory overhead (up to 173x) and latency (up to 26x).

#### 1. 核心问题与整体含义
*   **研究背景**：大语言模型（LLMs）尽管经过了安全对齐，但仍极易受到越狱攻击（Jailbreak Attacks）的威胁。
*   **核心问题**：现有的防御护栏（Guardrails）存在三大痛点：
    1.  **高延迟/高开销**：基于内部特征（如激活值、梯度）的方法计算成本巨大。
    2.  **不稳定性**：基于生成文本（关键词匹配）的方法受解码随机性影响，且关键词列表难以穷举。
    3.  **鲁棒性差**：分类器模型往往针对特定攻击训练，难以应对未见的自适应攻击。
*   **整体含义**：本文提出 **SelfGrader**，旨在通过将安全检测转化为一个简单的“数值评分问题”，利用模型输出的原始概率分布（Logits）来实现轻量、稳定且高效的防御。

#### 2. 方法论
SelfGrader 的核心思想是利用模型对特定数字 Token 的预测概率来推断其内部对查询安全性的真实判断。
*   **数值 Token (NT) 提取**：选取一组紧凑的数字 Token（如 0-9 或 0-100）。通过特定的提示词（Prompt）要求模型对用户查询的恶意程度打分，但不实际生成文本，而是提取模型在第一个输出位置上，这些数字 Token 对应的 Logits。
*   **双视角评分规则 (DPL)**：
    *   **恶意视角 ($s^{(+)}$)**：询问“该查询有多恶意？”，得分越高越危险。
    *   **良性视角 ($s^{(-)}$)**：询问“该查询有多安全？”，得分越高越安全。
    *   **综合评分**：通过公式 $s_{DPL} = \lambda s^{(+)} + (1-\lambda)(Q - 1 - s^{(-)})$ 结合两个视角，其中 $\lambda$ 是平衡因子（默认 0.5）。这种设计能有效降低误报率（FPR）。
*   **鲁棒性增强**：引入 **尾部修剪 (Tail Trimming)** 技术，剔除 Logit 分布中概率极低的噪声 Token，并进行重新归一化，使评分更聚焦于高置信度的信号。
*   **决策逻辑**：设定阈值 $\tau_D$，若综合评分超过阈值则拦截查询，返回预设的安全回复。

*   **实验规模**：论文进行了极其详尽的实验，包括针对 9 种不同类型的越狱攻击（手动、优化、自适应、多轮等）的测试。
*   **消融实验**：针对 ICL（上下文学习）示例的作用、双视角评分的必要性、数字 Token 数量 $Q$、修剪参数 $k$ 以及权重 $\lambda$ 均做了深入分析。
*   **特殊场景**：还测试了针对 Token 分割攻击（Emoji Attack）的鲁棒性，以及在数学和代码任务上的误报率。

#### 6. 主要结论与发现
*   **高效性**：相比于最强的基线方法，SelfGrader 的内存开销降低了高达 **173 倍**，延迟降低了 **26 倍**。
*   **防御性能**：在 Llama-3-8B 上，平均攻击成功率（ASR）显著下降，部分场景下 ASR 降低了 **22.66%**。
*   **稳定性**：通过 Logits 空间而非文本空间操作，SelfGrader 能够免疫因 Token 划分变化（如 Emoji 攻击）带来的语义干扰。
*   **效用保持**：在数学和代码任务中几乎没有引入额外的误报，证明了其在保障安全的同时不会损害模型的通用能力。

#### 7. 优点
*   **轻量且即插即用**：不需要微调模型，也不需要复杂的梯度计算，非常适合实际生产环境部署。
*   **确定性强**：避开了 LLM 文本生成的随机性问题，Logits 提供的信号比生成的“Yes/No”更细致、更稳定。
*   **双视角设计**：巧妙地结合了恶意和良性两个维度的判断，极大地平衡了安全性与可用性。

#### 8. 不足与局限
*   **Tokenizer 依赖**：该方法依赖于模型分词器中是否存在连续或独立的数字 Token。例如，Vicuna 仅支持 0-9 的独立 Token，限制了评分的分辨率（分辨率越高效果越好）。
*   **白盒风险**：如果攻击者完全了解 SelfGrader 的数字评分逻辑，可能会针对性地构造让数字 Logits 偏移的对抗性样本（虽然论文中未深入讨论此类针对性攻击）。
*   **提示词敏感性**：尽管有 ICL 示例，但防御效果仍可能受到护栏提示词（Guardrail Prompt）质量的影响。
