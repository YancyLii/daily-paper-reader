## 摘要
我们通过将检索控制直接嵌入到生成过程中，重新审视了检索增强生成（RAG）。我们不再将检索视为外部干预，而是在词元级（token-level）解码中表达检索决策，从而在无需额外控制器或分类器的情况下实现端到端协作。在“检索即生成”的范式下，我们提出了 GRIP（Generation-guided Retrieval with Information Planning），这是一个统一的框架，模型通过发射控制词元（control-token）来调节检索行为。GRIP 的核心是“自触发信息规划”，它允许模型在单个自回归轨迹中决定何时检索、如何重构查询以及何时终止。这种设计将检索与推理紧密耦合，并支持具有即时证据整合的动态多步推理。为了监督这些行为，我们构建了一个结构化训练集，涵盖了可回答、部分可回答和多跳查询，每种查询都与特定的词元模式对齐。在五个问答（QA）基准测试上的实验表明，GRIP 超越了强大的 RAG 基准模型，并在参数量显著减少的情况下，其性能可与 GPT-4o 媲美。

## Abstract
We revisit retrieval-augmented generation (RAG) by embedding retrieval control directly into generation. Instead of treating retrieval as an external intervention, we express retrieval decisions within token-level decoding, enabling end-to-end coordination without additional controllers or classifiers. Under the paradigm of Retrieval as Generation, we propose \textbf{GRIP} (\textbf{G}eneration-guided \textbf{R}etrieval with \textbf{I}nformation \textbf{P}lanning), a unified framework in which the model regulates retrieval behavior through control-token emission. Central to GRIP is \textit{Self-Triggered Information Planning}, which allows the model to decide when to retrieve, how to reformulate queries, and when to terminate, all within a single autoregressive trajectory. This design tightly couples retrieval and reasoning and supports dynamic multi-step inference with on-the-fly evidence integration. To supervise these behaviors, we construct a structured training set covering answerable, partially answerable, and multi-hop queries, each aligned with specific token patterns. Experiments on five QA benchmarks show that GRIP surpasses strong RAG baselines and is competitive with GPT-4o while using substantially fewer parameters.

### 1. 核心问题与整体含义（研究动机和背景）
传统的检索增强生成（RAG）通常将检索视为一个外部的、单次的干预过程：先检索文档，再固定上下文进行生成。这种方式在处理需要多步推理、存在歧义或信息需求随推理过程动态变化的复杂查询时表现不佳。
*   **研究动机**：现有的动态检索方法要么依赖启发式的阈值（如不确定性），要么依赖外部控制器或多阶段管道，导致检索决策与模型的内部推理状态脱节。
*   **核心目标**：提出“检索即生成”（Retrieval as Generation）范式，将检索控制直接嵌入到词元（Token）级的解码过程中，使模型能够像生成文本一样自主决定何时检索、搜什么以及何时停止。

### 2. 方法论：核心思想与关键技术
论文提出了 **GRIP**（Generation-guided Retrieval with Information Planning）框架，其核心思想是通过扩展模型的输出词表，引入特定的**控制词元**来管理检索行为。

*   **关键技术细节**：
    *   **控制词元**：引入了 `[RETRIEVE]`（触发检索）、`[INTERMEDIARY]`（输出中间状态）、`[ANSWER]`（开始最终回答）和 `[SOLVED]`（终止生成）。
    *   **自触发信息规划**：模型在自回归过程中，根据当前上下文判断信息是否充足。若不足，则发射 `[INTERMEDIARY]` 和 `[RETRIEVE]`，并紧接着生成一个新的查询语句（Query Reformulation），检索到的新证据会被反馈到输入中继续推理。
    *   **结构化监督训练**：将训练数据分为四种类型（Type-α：直接回答；Type-β：需要检索；Type-γ：多跳规划；Type-θ：答案整合），通过教师引导（Teacher Forcing）让模型学习不同场景下的词元发射模式。
    *   **两阶段训练流程**：
        1.  **监督微调（SFT）**：在结构化行为示例上训练，建立基础的检索规划能力。
        2.  **强化学习（RL/DAPO）**：使用规则驱动的奖励（答案忠实度奖励 + 控制准确性奖励）来优化模型，减少过度检索并提高停止决策的可靠性。

### 6. 主要结论与发现
*   **性能卓越**：GRIP 在所有基准测试中均显著优于现有的开源 RAG 框架，且在参数量远小于 GPT-4o 的情况下，性能与其相当。
*   **任务感知自适应**：模型学会了根据任务调整检索频率（例如在多跳任务 HotpotQA 中检索更多，在常识任务 NQ 中检索更少）。
*   **检索质量提升**：通过动态生成的查询（Query Reformulation），检索到的文档包含正确答案的概率显著高于原始查询。
*   **能力外推**：虽然训练时最大检索步数为 3，但在推理时增加预算（如设为 10），模型仍能通过更多步检索持续提升性能。
*   **通用能力保留**：在非 RAG 任务（如代码、逻辑推理）上，GRIP 的性能下降极小，优于传统的 SFT-RAG 方法。

### 7. 优点
*   **高度集成**：将检索决策内化为 Token 生成，无需外部复杂的分类器或启发式规则。
*   **可解释性强**：检索时机和查询意图直接体现在生成的文本轨迹中。
*   **效率与效果平衡**：通过 RL 优化，模型学会了“见好就收”，有效减少了冗余检索带来的计算开销。
*   **灵活性**：支持推理时的动态预算调整，具有良好的外推能力。

### 8. 不足与局限
*   **检索接口敏感性**：模型表现仍受限于外部检索器的性能、分块策略（Chunking）和 Top-K 设置。
*   **训练数据构建成本**：Type-γ（多跳）数据的构建依赖于高性能教师模型（如 GPT-4o-mini）来生成高质量的查询重构示例，尽管实验证明可以使用开源模型替代，但仍需一定的标注成本。
*   **推理延迟**：虽然减少了冗余检索，但多步“生成-检索-再生成”的循环在处理极复杂问题时，端到端延迟仍高于单次 RAG。
