## 摘要
近期，用于测试用例生成的大语言模型通过提示工程驱动的变异提高了分支覆盖率。然而，它们仍缺乏引导模型走向特定高风险执行分支的原则性机制，限制了其在发现细微漏洞和安全风险方面的有效性。我们提出了 GLMTest，这是首个用于定向测试用例生成的程序结构感知 LLM 框架。它利用图神经网络和语言模型，将代码属性图与代码语义无缝集成，从而以执行分支为条件来引导测试用例的生成。这种结构化约束实现了可控且针对分支的测试用例生成，从而有可能增强对漏洞和安全风险的发现能力。在真实项目上的实验表明，基于 Qwen2.5-Coder-7B-Instruct 模型构建的 GLMTest 在 TestGenEval 基准测试中将分支准确率从 27.4% 提升至 50.2%，优于 Claude-Sonnet-4.5 和 GPT-4o-mini 等最先进的 LLM。

## Abstract
Recent advances in large language models for test case generation have improved branch coverage via prompt-engineered mutations. However, they still lack principled mechanisms for steering models toward specific high-risk execution branches, limiting their effectiveness for discovering subtle bugs and security vulnerabilities. We propose GLMTest, the first program structure-aware LLM framework for targeted test case generation that seamlessly integrates code property graphs and code semantics using a graph neural network and a language model to condition test case generation on execution branches. This structured conditioning enables controllable and branch-targeted test case generation, thereby potentially enhancing bug and security risk discovery. Experiments on real-world projects show that GLMTest built on a Qwen2.5-Coder-7B-Instruct model improves branch accuracy from 27.4% to 50.2% on TestGenEval benchmark compared with state-of-the-art LLMs, i.e., Claude-Sonnet-4.5 and GPT-4o-mini.

#### 1. 核心问题与整体含义（研究动机和背景）
自动化测试生成对于软件正确性和安全性至关重要。虽然大语言模型（LLM）在生成测试用例方面取得了进展，但现有方法（如提示工程或简单的微调）存在两个核心缺陷：
*   **缺乏可控性：** LLM 的随机性使得难以引导其覆盖特定的、高风险的执行分支。
*   **结构理解不足：** 纯文本表示无法充分捕捉代码对象间的复杂依赖关系和程序结构。
本文提出了 **GLMTest**，旨在通过结合程序结构（代码属性图，CPG）与文本语义，实现**定向（Targeted）**的分支覆盖测试生成。

#### 2. 方法论：核心思想与关键技术
GLMTest 的核心思想是将程序的图结构特征直接注入 LLM 的嵌入空间，使模型在生成测试时受到特定执行路径的约束。
*   **代码属性图（CPG）建模：** 将程序 $S$ 转化为异构多关系图 $G=(V, E)$，包含抽象语法树（AST）、控制流图（CFG）和数据流图（DFG）等信息。
*   **分支掩码（Branch Mask）：** 根据目标执行分支 $b$，生成一个二进制掩码 $m$，标识出与该分支相关的节点集合 $V_b$。
*   **异构图神经网络（GNN）：** 使用 $K$ 层异构图注意力网络（GAT）提取节点嵌入。GNN 模块与 LLM 联合训练，将结构信息转化为“分支感知结构嵌入”。
*   **结构-文本融合：** 在 LLM 的输入提示词中引入特殊的占位符标记（如 `<|graph_pad|>`），并将 GNN 输出的结构嵌入替换到这些位置，与文本嵌入拼接后输入 LLM。
*   **端到端训练：** 采用监督微调（SFT）策略，利用真实世界项目的“程序-分支-测试用例”三元组数据，通过交叉熵损失函数同时优化 GNN 和 LLM 参数。
