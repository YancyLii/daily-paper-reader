## 摘要
使用工具的基于大型语言模型（LLM）的智能体面临两种不同的安全失效：未经授权的外部操作，以及在最终输出检查介入前敏感明文在运行时内部的暴露。现有防御通常只保护一个边界，要么是规划器/运行时，要么是动作汇，因此它们自身无法同时确保两个表面。我们提出SecureClaw，一种双边界架构，它将授权置于效应汇，将明文限制置于读取边界。敏感读取通过一个可信网关，该网关将原始值替换为不透明句柄，并在评估部署中，使用有界摘要作为显式的降级接口。改变外部状态的写操作遵循PREVIEW→COMMIT协议，其中只有可信执行器可以提交由策略授权的确切规范请求。运行时仍可基于摘要和符号引用进行规划，但不能直接解引用秘密或执行副作用。在AgentDojo、AgentLeak和Agent Security Bench（ASB）上，SecureClaw是我们在统一测试框架中评估的唯一防御方法，它同时保留了可用的任务实用性，并在ASB上达到0%的攻击成功率（ASR），在AgentDojo上达到0.64%的ASR，在AgentLeak的攻击对等通道上整体泄漏率为3.23%，该通道测量最终输出和内部中继泄漏。

## Abstract
Tool-using large language model (LLM) agents face two distinct security failures: unauthorized external actions and exposure of sensitive plaintext inside the runtime before any final output check can intervene. Existing defenses usually protect one boundary, either the planner/runtime or the action sink, and therefore do not by themselves secure both surfaces. We present SecureClaw, a dual-boundary architecture that places authorization at the effect sink and plaintext confinement at the read boundary. Sensitive reads pass through a trusted gateway that replaces raw values with opaque handles and, in the evaluated deployment, bounded summaries as an explicit declassification interface. Writes that change external state follow a PREVIEW$\rightarrow$COMMIT protocol in which only a trusted executor may commit the exact canonical request authorized by policy. The runtime can still plan over summaries and symbolic references, but cannot directly dereference secrets or perform side effects. Across AgentDojo, AgentLeak, and Agent Security Bench (ASB), SecureClaw is the only defense we evaluate in a common harness that simultaneously retains usable task utility and achieves 0\% attack success rate (ASR) on ASB, 0.64\% ASR on AgentDojo, and 3.23\% overall leak on AgentLeak's attacked parity lane, which measures final-output and internal-relay leakage.

### 1. 论文的核心问题与整体含义
- **核心问题**：基于工具使用的大语言模型（LLM）智能体存在两类相互独立的安全失效：  
  - **效应面**：未经授权的外部动作（如发送邮件、修改记录）能够到达外部汇点。  
  - **明文面**：敏感数据在运行时内部以明文形式暴露，可通过内部信道（跨智能体消息、共享内存等）泄露，即使最终输出被审计也无法阻止。
- **研究动机**：现有防御（如IPIGuard、DRIFT、Faramesh）通常只保护单一边界——要么是规划/运行时，要么是动作汇——无法同时解决两种失效。需要一种架构将“外部效应授权”和“运行时明文限制”作为两个独立安全表面进行控制。
- **整体含义**：SecureClaw通过双边界设计，将授权置于效应汇（写路径），将明文限制置于读取边界（读路径），使不可信运行时既不能提交未经授权的效应，又不能直接持有原始敏感值，从而在贴近实际部署的条件下兼顾安全性与任务可用性。

### 2. 论文提出的方法论
#### 核心思想
- **双边界分离**：  
  - **读边界**：敏感数据通过可信网关，以不透明句柄（handle）和有界摘要（bounded summary）返回运行时，防止明文进入运行时内部信道。  
  - **写边界**：任何效应性操作必须经过`PREVIEW → COMMIT`协议，仅由可信执行器在策略授权后提交精确的规范请求。
- **信任模型**：运行时完全不可信；网关、句柄存储、策略引擎和执行器是可信计算基。

#### 关键技术细节
- **读路径——不透明句柄限制**：
  - 敏感读取时，网关将明文存入受保护的句柄表，生成高熵句柄（如`hid = HMAC-SHA256(khandle, nonce‖session‖object)`）返回运行时。
  - 运行时仅接收句柄和显式界定的摘要（基于模式感知的有界摘要算子，如“发票来自供应商A，金额$4.2k，周五到期”），作为显式降级接口。
  - 句柄无法被运行时直接解引用，跨智能体消息和内存中只能携带符号引用，从而切断内部中继泄露路径。
- **写路径——PREVIEW→COMMIT执行边界**：
  - 运行时提出效应性请求（意图、可见输入），网关补全认证调用者、会话和上下文（收件人、信道等）形成规范请求ρ。
  - 网关对规范请求计算绑定摘要`h = HMAC-SHA256(kbind, Canon(ρ))`，并查询策略引擎获得与摘要绑定的授权构件。
  - 执行器独立重新计算h，验证授权构件、新鲜度、重放状态，并在检查手柄解析限制后提交外部效应，任何失败均“失败关闭”。
- **拒绝感知恢复**：当提交被拒绝时，返回结构化拒绝信息（仅含粗动作类别、原因码和安全提示），引导工作流向安全替代路径（如从“发送”切换为“起草待确认”），保障可用性而不削弱安全不变式。
- **形式化保障**：论文附录给出定理1（请求绑定授权完整性）、定理2（仅句柄参考下的明文限制）和定理3（有界摘要接口的信息泄露量化），分别形式化效应面和明文面的安全保证。

### 6. 主要结论与发现
- **安全分解有效性**：SecureClaw是唯一在统一工具框架下同时实现ASB 0% ASR、AgentDojo 0.64% ASR、AgentLeak奇偶通道泄露3.23%的防御，且保持较高任务实用率。
- **互补不可替代**：消融实验表明，移除句柄导致内部信道泄露复现，而移除执行边界则不影响明文限制但无法防止未授权提交；绕过套件证实执行边界能抵御完全对抗的运行时。两个机制分别保护不同表面，缺一不可。
- **可用性源于安全延续而非削弱**：拒绝感知恢复将ASB实用率从70%提升至86%，AgentLeak风格子集从40%提升至66.7%，均无ASR增加，说明高实用率来自安全替代路径而非放松不变式。
- **残余失效转移**：少数残余失败主要集中在策略允许区域内的语义对齐问题（如错误主客体）和最终输出通道（C1）的降级压力，而非内部中继泄露或提交绕过，表明问题已从安全旁路转向更细粒度的策略表达和降级控制。

### 7. 优点
- **双边界首创**：首次系统性分离并同时解决效应授权和运行时明文限制两个独立的安全表面，突破了单一边界防护的局限。
- **架构清晰可验证**：采用PREVIEW→COMMIT协议和不透明句柄设计，将安全执行点从不可信运行时移至可信组件，提供形式化定理支撑。
- **拒绝感知恢复**：在不损害安全的前提下，通过结构化的安全提示保持攻击情境下的任务可用性，实用性强。
- **统一工具框架的公平对比**：在相同基准、相同模型、相同评分器下对比多种基线，揭示架构性收益，而非依赖于提示过滤或局部加固。
- **细致的安全分析**：通过消融、绕过、恢复等补充实验，清晰展示每个机制的独立贡献和局限，并量化了有界摘要的泄露特性。

### 8. 不足与局限
- **残余失败仍需策略改进**：少量AgentDojo攻击成功源于粗粒度的策略允许（如允许动作族但可被攻击者选择收件人），AgentLeak的C1泄露表明降级接口可能传递敏感信息，需要更丰富的策略绑定（如主客体、阶段）和更严格的摘要控制。
- **对分类和配置的依赖**：安全保证基于敏感字段的正确分类和效应汇点的完全中介，若分类错误或汇点未覆盖，则保证失效，部署需配合审计和保守的策略。
- **跨模型泛化验证有限**：仅进行小规模跨模型抽查（50行），未在更多模型家族或超大规模设置下进行全面评估，模型不确定性可能影响稳定结论。
- **未讨论复杂攻击场景**：尽管绕过套件覆盖了承诺路径的对抗篡改，但对多智能体协同攻击、侧信道（如摘要的长度或时间差异）未做深入分析。
- **可用性损失尚存**：在AgentDojo上良性任务实用率由Plain的70.10%降至60.82%，尽管相比其他防御较好，但仍有一定损失，可能与摘要信息有限和保守的提交策略有关。
- **分布式策略引擎隐私保障**要求非共谋假设，且单查询隐私模型较窄，未在实验中充分评估。
