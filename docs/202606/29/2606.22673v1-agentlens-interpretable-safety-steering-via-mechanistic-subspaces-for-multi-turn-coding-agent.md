## 摘要
基于大型语言模型（LLM）的编码代理展现出卓越的自主能力，但在与外部环境的多轮交互中也引入了显著的安全与滥用风险。现有安全机制主要依赖外部护栏，在执行过程中进行细粒度行为控制的能力有限。与此同时，近期针对LLM安全的机制可解释性方法大多局限于单轮或越狱式的问答场景，难以捕捉多轮代理执行中不断演变的风险动态。本文从内部视角研究多轮编码代理的安全性，提出AgentLens（机制子空间干预与引导）——一种白盒防御框架，能够对编码代理进行运行时安全检测与表征层面的缓解。不同于传统的代理护栏，AgentLens 从步骤级隐藏表征中检测有害执行状态，并通过干预单层内的10维子空间来减轻不安全行为。为支持该研究，我们引入了机制代理安全（MAS）基准，该基准包含了使用LLaMA-3.1-8B、Qwen-2.5-7B和Gemma-2-9B在194个任务上全面标注的多轮执行轨迹。大量实验表明，AgentLens 实现了强大的安全检测性能，提供了关于前瞻性风险预期的初步证据，并大幅减少了编码代理的有害行为，为将机制可解释性应用于动态LLM代理安全奠定了基础。代码见：https://github.com/EddyLuo1232/AgentLens

## Abstract
Coding agents based on large language models (LLMs) demonstrate remarkable autonomous capabilities, but they also introduce significant safety and misuse risks during multi-turn interactions with external environments. Existing safety mechanisms mainly rely on external guardrails, which have a limited ability to perform fine-grained behavioral control during execution. Meanwhile, recent mechanistic interpretability methods for LLM safety are mostly confined to single-turn or jailbreak-style QA settings, limiting their ability to capture the evolving risk dynamics of multi-turn agent execution. In this paper, we investigate the safety of multi-turn coding agents from an internal perspective. We propose AgentLens (Mechanistic Subspace Intervention and Steering), a white-box defense framework that performs runtime safety detection and representation-level mitigation for coding agents. Unlike conventional agent guardrails, AgentLens detect harmful execution states from step-level hidden representations and mitigate unsafe behavior by intervening in a 10-dimensional subspace within a single layer. To support this research, we introduce the Mechanistic Agent Safety (MAS) benchmark, comprising comprehensively annotated multi-turn execution trajectories across 194 tasks using LLaMA-3.1-8B, Qwen-2.5-7B, and Gemma-2-9B. Extensive experiments show that AgentLens achieves strong safety detection performance, provides preliminary evidence for lookahead risk anticipation, and substantially reduces harmful actions of the coding agent, establishing a foundation for applying mechanistic interpretability to dynamic LLM agent safety. ## 2. 方法论

### 2.1 核心思想
提出白盒防御框架 **AgentLens**，它不修改模型权重，而是在推理时通过两个阶段工作：
1. **检测阶段**：利用步骤级隐藏表征训练一个轻量级线性探针（linear probe），判断当前交互状态是否有害。
2. **缓解阶段**：若检测到有害步骤，则从探针权重中选取最具判别力的维度（Top‑K），构造稀疏引导向量，对编码代理的内部表征进行干预，使其远离有害行为。

### 2.2 关键技术细节

- **步骤级线性探针**：
  - 对冻结的 LLM，在选定层 ℓ 提取最后一个 token 的隐藏表征 \(z_t^{(ℓ)}\)。
  - 训练二分类逻辑回归探针 \(g_{\phi}(z) = \sigma(w^T z + b)\)，标签为当前步骤是否直接执行有害操作（如破坏性命令、持久化安装、权限篡改等）。
  - 训练时使用了反事实增强（counterfactual augmentation）：构造与恶意命令语法相似但语义良性的步骤，防止探针只记住表面词汇。

- **安全检测（当前步 + 前瞻）**：
  - 当前步检测：探针对当前步骤的隐藏状态打分，超过阈值则判定为有害。
  - 前瞻（lookahead）检测：将紧邻有害执行步之前的一步标记为「即将发生有害行为」，用同一探针测试其能否提前预测风险。

- **风险缓解（Representation Steering）**：
  - 从探针权重矩阵 \(W \in \mathbb{R}^{2 \times d}\) 中计算每个维度的类别区分度（有害与良性的权重差绝对值），选取 Top‑K（K=10）个维度。
  - 构造引导向量 \(v \in \mathbb{R}^d\)（选定维度为 1，其余为 0）。
  - 在目标层 ℓ 对所有 token 的隐藏状态加性干预：\(\tilde{H}_t^{(ℓ)}(\alpha) = H_t^{(ℓ)} + \alpha \mathbf{1}_{n_t} v^\top\)，α 为引导强度。
  - **自适应 α 搜索**：采用一个外部 LLM 评委（DeepSeek‑V4‑Pro）迭代调用工具（AdjustAlpha / SubmitEvaluation），以安全性与实用性加权得分（λs=0.6, λu=0.4）寻找最优 α，防止生成崩溃或无效输出。
  - 引导方向固定为正向（使模型更倾向于安全拒绝），由训练集校准确定。

- **整体算法流程**：
  1. 每步提取隐藏状态 → 探针预测有害标签。
  2. 若判为安全（ŷ=0），直接输出动作。
  3. 若判为有害（ŷ=1），触发缓解阶段：计算引导向量，自适应搜索 α，通过表征干预生成安全动作。

### 2.3 概念形式化
- 单轮 LLM vs 多轮编码代理：论文给出了两者形式定义（定义 2.1、2.2），强调安全评估必须基于完整轨迹而非孤立动作。

## 6. 主要结论与发现

- **线性可解码性**：多轮编码代理的中间层隐藏表征中存在可用简单线性探针提取的安全信号，在早期-中间层（LLaMA‑3.1‑8B 第12层、Qwen‑2.5‑7B 第13‑17层、Gemma‑2‑9B 第14层）达到最佳检测性能。
- **检测性能**：AgentLens 当前步检测平均准确率 97.32%，显著优于所有 LLM‑as‑Judge 和护栏基线（提升 3‑6 个百分点）；前瞻检测平均准确率 94.54%（在可用数据上），提供了隐蔽状态包含未来风险信号的初步证据。
- **缓解效果**：AgentLens 将三个模型的平均攻击成功率从 85.99% 降至 13.36%（绝对降幅 72.63%），远强于 RepE 和 Self Reminder；但会带来一定生成质量代价（平均崩溃率 18.54%）。
- **因果效应**：负向引导实验表明，将引导方向反转可将安全拒绝步扭转为执行有害命令（ASR 高达 100%），证明发现的子空间具有因果控制作用，而非仅相关。
- **分布外鲁棒性（提示注入）**：当前探针未能可靠检测间接提示注入状态，但相同的引导方向仍能有效抑制有害行为（ASR 从 86.7% 降至 6.7%），表明检测与控制可解耦，表征级引导具有跨分布的控制能力。

## 7. 优点

- **内部视角创新**：首次将机制可解释性应用于多轮编码代理安全，从步骤级隐藏状态直接进行安全检测与干预，超越了传统外部护栏。
- **轻量且有效**：仅需一个线性探针和一个 10 维稀疏引导向量，无需微调模型权重，即可实现优异的检测与缓解。
- **因果验证严谨**：通过负向引导（逆转安全拒绝）提供了强因果证据，增强了框架可信度。
- **基准贡献**：提出的 MAS 基准可作为可复现的测试平台，包含严格标注和反事实增强，支持步骤级别的安全分析。
- **自适应控制**：采用外部 LLM 评委搜索最优引导强度，在安全与实用性之间寻求平衡，避免一刀切式干预。

## 8. 不足与局限

- **白盒访问依赖**：需访问模型中间层激活值，不适用于黑盒或仅提供 API 的模型。
- **前瞻检测数据稀疏**：受限于当前开源模型的长程编码能力，自然产生的前瞻样本很少，导致该部分评估不充分、结论薄弱。
- **缓解副效应**：表征引导有时会导致生成崩溃（重复、截断），影响可用性，且自适应搜索引入额外推理延迟和外部 LLM 依赖。
- **标注歧义**：步骤级标签存在固有模糊性（准备性步骤可能携带潜在风险），当前标签规则以是否直接执行攻击为准，可能低估某些中间步骤的风险。
- **提示注入检测失败**：探针在间接注入任务上泛化能力不足，需要额外探测器辅助。
- **任务集限制**：恶意轨迹收集困难，许多开源小模型难以完成长程复杂攻击，导致部分模型的样本量较小（如 Gemma‑2‑9B 训练集仅 146 步）。
- **缺乏多模态或更复杂环境**：实验局限于 Bash 操作，未覆盖更丰富的工具使用或 Web 代理场景。
- **未讨论部署后的持续学习或对抗适应性**：属静态探针加固定引导，可能在对抗性环境中被规避。
