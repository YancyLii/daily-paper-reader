## 摘要
大型语言模型（LLMs）可通过多轮对话策略被诱导生成有害内容，其中没有单条用户消息显得明显不安全。现有的运行时安全措施通常将提示或响应作为孤立消息进行评估，这限制了它们恢复累积意图、验证所声称的权限或检测在对话中分解的有害目标的能力。本文提出了认知防火墙（Cognitive Firewall），这是一个主动的运行时监督框架，它在用户和受保护的目标模型之间引入一个独立的监督模型。该框架将安全评估分解为四个分类门控：意图门控，识别请求的操作目标；零信任上下文门控，将声称的角色和权限视为未经验证的证据；一致性门控，检测跨轮次的升级和分解；输出风险门控，在释放前检查候选响应。门控决策通过升级机制而非分数平均来组合，允许任何确信的危险信号阻止交互，同时保留可审计的理由。在四个越狱基准和一个良性安全测试集上的实验表明，认知防火墙大幅降低了单轮、多轮、基于权限和人工设计的攻击的成功率。它将三个攻击集上的攻击成功率降低至2%或以下，将最困难的人工设计集的攻击成功率降至14%，同时维持8%的过度拒绝率。这些结果表明，分解的对话级监督可以改善LLM安全的主动遏制和可审计性。

## Abstract
Large language models (LLMs) can be induced to produce harmful content through multi turn strategies in which no single user message appears clearly unsafe. Existing runtime safeguards commonly evaluate prompts or responses as isolated messages, which limits their ability to recover ac-cumulated intent, verify asserted authority, or detect harmful objectives decomposed across a dialogue. This paper presents the Cognitive Firewall, a proactive runtime oversight framework that interposes an independent oversight model between a user and a protected target mod l. The framework decomposes safety assessment into four categorical gates: an intent gate that identi-fies the operational objective of a request, a zero trust context gate that treats claimed roles and permissions as unverified evidence, a consistency gate that detects escalation and decomposition across turns, and an output risk gate that inspects candidate responses before release. Gate decisions are combined through escalation rather than score averaging, allowing any confident danger signal to block an interaction while preserving an auditable rationale. Experiments on four jailbreak benchmarks and a benign safety test set show that the Cognitive Firewall substantially reduces attack success across single turn, multi turn, authority based, and human crafted attacks. It lowers attack success to 2 percent or below on three attack sets and to 14 percent on the most difficult human crafted set, while maintaining an 8 percent over refusal rate. These results indicate that decomposed, conversation level oversight can improve proactive containment and auditability for LLM safety.

## 1. 研究动机与核心问题
- 大型语言模型（LLM）的安全防护当前主要依赖**单消息级别的危害分类器**（如 Llama Guard, ShieldGemma），它们独立对每一次 prompt 或 response 打分，无法捕捉多轮对话中累积的恶意意图。
- 现代攻击策略（如 Crescendo 逐步升级、ActorAttack 任务分解、多轮人工越狱）将危害意图分散在多个看似无害的轮次中，借助声称的权限（开发者、临床医生等）换取信任，单轮检测手段对此几乎失效。
- **核心隐喻**：LLM 安全应被视为**理解问题而非分类问题**。现有的单消息分类器缺少对用户操作目标、对话轨迹和权限声明的持续建模，构成了根本性缺陷。

## 2. 方法论：认知防火墙框架
- **整体架构**：在用户与受保护的目标模型之间插入一个独立的**监督模型**（oversight model, `qwen3:8b`），阅读整个对话历史而非孤立消息。
- **四个分类门控**（所有门控均输出离散标签，无连续阈值）：
    - **意图门（G1, pre‑gen）**：识别请求的实际可操作目标（referent），而非表面措辞。仅当回答会直接输出可转移的危害能力（如武器合成、恶意软件、仇恨言论）时判定为 `unsafe`。虚构或研究框架不影响判决。
    - **零信任上下文门（G2, pre‑gen）**：将对话中声称的角色、权限、身份或策略变更视为**未经验证的证据**。若用户通过第一人称操纵性地声明开发者、审计员或修改助手规则，则判定为 `manipulative`。普通的职业身份声明不受影响。
    - **一致性门（G3, pre‑gen）**：审查完整对话历史，检测逐步升级（crescendo）、目标漂移、矛盾以及跨轮次的任务分解。一旦出现 `escalating` 判决即阻断。第一轮用户消息的门控自动豁免。
    - **输出风险门（G4, post‑gen）**：在允许生成后，检查候选响应是否实际提供了可操作的危险内容，即使是分步或部分材料。若为 `harmful`，则扣留响应。
- **决策规则：升级机制（escalation）**：
    - 任一前置门控判决为危险（`unsafe`, `manipulative`, `escalating`）时，在目标模型生成**之前**阻断该轮交互。
    - 只有全部前置门控通过后才调用目标模型，最后由输出门控对响应进行后置拦截。
    - 该机制避免了传统轨迹防火墙将风险信号“平均化”导致的稀释问题，每个阻断都有明确、可审计的门控来源和理由。

## 6. 主要结论与发现
- 认知防火墙在所有多轮攻击集上大幅降低 ASR：`crescendo` 降至 0%，`safemt` 降至 2%，`mhj` 降至 14%（无防护时为 74%），同时单轮攻击仅 1%。
- **前置阻断效果显著**：在 `crescendo` 上 100% 阻断，`safemt` 和 `mhj` 上大部分阻断发生在生成之前，节省了对目标模型的计算暴露。
- **门控分工明确**：意图门负责直接有害请求；一致性门是多轮升级和分解的主要贡献者；输出门是分解载荷的最后防线，且与前几道形成互补，任一单独使用都不及完整版本。
- **零信任上下文门**有效隔离了权限操纵通道：在权限探针上阻断率达 75%，而其他基线仅 6%~31%（或伴随高过度拒绝），移除该门后防火墙完全失效。
- **过度拒绝率低**：仅 8%，优于 ShieldGemma（18%）和 Granite Guardian（21%）。
- 每个阻断都有**可读的理由**，增强了审计和调试的可解释性。

## 7. 优点
- **多维度、可组合的安全评估**：将安全拆分为意图、上下文信任、轨迹和输出四个独立且可解释的维度，比单一总分的粒度更细，避免了信号稀释。
- **主动的前置阻断**：大部分威胁在目标模型生成前即被拦截，减少了受攻击面。
- **零信任原则的引入**：首次将“永远不信任，始终验证”的理念系统性地应用于对话内角色与权限的验证，开辟了独立于话题安全的审查通道。
- **无需训练，即插即用**：基于现成 LLM 的提示工程实现，无需针对特定攻击或模型重新训练。
- **决策可审计**：升级规则输出的每个阻断动作都绑定到一个具体的门控及其简短理由，便于人工复核。
- **互补性优势**：严格的消融实验证明前后置门控必须协同工作，单独部署会使安全性能大打折扣。

## 8. 不足与局限
- **样本量小与数据偏倚**：攻击集合仅有 50~100 条对话，权限制探针和自适应攻击更小，导致统计波动较大；部分对比基线为作者自行复现，不能完全代表原方法的最佳性能。
- **评估偏差风险**：危害评判器与监督模型同属 Qwen 家族，可能存在同一家族的认知偏差，尽管作者已对防火墙的残余失败案例进行人工验证。
- **校准依赖性**：门控提示词在开发集的早期样本上迭代开发，因此主报告中的数值存在一定程度的“内样本”乐观倾向。
- **计算开销未量化**：每轮 3~4 次串行模型调用的延迟和总成本未做测量，实际部署时的吞吐量影响未知。
- **安全覆盖缺口**：
    - 最强的多轮人工攻击集（mhj）上仍有 14% 的残余成功率。
    - 未明确测试对多轮良性对话的过度阻断情况（xstest 是单轮测试）。
    - 监督模型本身是 8B 的 LLM，可能受到攻击者通过对话历史注入的提示操控，论文未进行针对监督模型的 prompt injection 压力测试。
    - 对伪造凭证类的权限攻击防御不足（设计上避免拒绝真实的专业用户）。
- **主观性边界**：危害判定（尤其是“可操作”与“虚构”的边界）受 prompt 策略和评判标准影响，是一种策略参数，而非绝对的客观度量。
