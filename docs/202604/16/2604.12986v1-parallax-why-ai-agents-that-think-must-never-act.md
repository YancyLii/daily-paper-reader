## 摘要
自主 AI 智能体正在迅速从实验工具转变为运营基础设施，预计到 2026 年底，80% 的企业级应用将嵌入 AI 副驾驶（copilots）。随着智能体获得执行现实世界动作的能力（如读取文件、运行命令、发起网络请求、修改数据库），一个根本性的安全漏洞也随之出现。目前主流的智能体安全方法依赖于提示词级护栏（prompt-level guardrails）：即在与试图缓解的威胁相同的抽象层级上运行的自然语言指令。本文认为，对于具备执行能力的智能体而言，基于提示词的安全机制在架构上是不充分的，并由此引入了 Parallax，这是一种基于四个原则的自主 AI 安全执行范式：认知-执行分离（Cognitive-Executive Separation），从结构上防止推理系统执行动作；具有分级确定性的对抗性验证（Adversarial Validation with Graduated Determinism），在推理和执行之间插入一个独立的、多层级的验证器；信息流控制（Information Flow Control），通过智能体工作流传播数据敏感性标签，以检测与上下文相关的威胁；以及可逆执行（Reversible Execution），捕获破坏前的状态，以便在验证失败时进行回滚。我们展示了 OpenParallax，这是一个使用 Go 语言编写的开源参考实现，并使用“假设受损评估”（Assume-Compromise Evaluation）对其进行了评估。该方法完全绕过推理系统，以测试在智能体完全受损情况下的架构边界。在涵盖 9 个攻击类别的 280 个对抗性测试用例中，Parallax 在默认配置下拦截了 98.9% 的攻击且误报率为零，在最高安全配置下则拦截了 100% 的攻击。当推理系统受到侵害时，提示词级护栏无法提供任何保护，因为它们仅存在于受损系统内部；而无论如何，Parallax 的架构边界都能保持稳固。

## Abstract
Autonomous AI agents are rapidly transitioning from experimental tools to operational infrastructure, with projections that 80% of enterprise applications will embed AI copilots by the end of 2026. As agents gain the ability to execute real-world actions (reading files, running commands, making network requests, modifying databases), a fundamental security gap has emerged. The dominant approach to agent safety relies on prompt-level guardrails: natural language instructions that operate at the same abstraction level as the threats they attempt to mitigate. This paper argues that prompt-based safety is architecturally insufficient for agents with execution capability and introduces Parallax, a paradigm for safe autonomous AI execution grounded in four principles: Cognitive-Executive Separation, which structurally prevents the reasoning system from executing actions; Adversarial Validation with Graduated Determinism, which interposes an independent, multi-tiered validator between reasoning and execution; Information Flow Control, which propagates data sensitivity labels through agent workflows to detect context-dependent threats; and Reversible Execution, which captures pre-destructive state to enable rollback when validation fails. We present OpenParallax, an open-source reference implementation in Go, and evaluate it using Assume-Compromise Evaluation, a methodology that bypasses the reasoning system entirely to test the architectural boundary under full agent compromise. Across 280 adversarial test cases in nine attack categories, Parallax blocks 98.9% of attacks with zero false positives under its default configuration, and 100% of attacks under its maximum-security configuration. When the reasoning system is compromised, prompt-level guardrails provide zero protection because they exist only within the compromised system; Parallax's architectural boundary holds regardless.

*   **核心挑战**：随着 AI 智能体从“生成内容”转向“执行动作”（如操作文件、运行 Shell 命令、发起网络请求），传统的**提示词级护栏（Prompt-level Guardrails）**暴露出根本性缺陷。
*   **研究动机**：作者指出提示词防御存在“同质化失效”风险，即防御指令与攻击载荷（如提示词注入）处于同一计算层级，无法在推理系统受损时提供保护。
*   **背景**：2026 年企业级 AI 应用激增，智能体具备了“代理权（Agency）”，一旦被劫持，可能导致数据外泄、系统破坏等不可逆后果。论文旨在建立一种**架构级**的安全范式，而非依赖于模型自身的道德对齐。

#### 2. 论文提出的方法论：Parallax 范式
Parallax 范式由四个核心原则组成，旨在实现“架构安全”：
*   **认知-执行分离 (CES)**：将系统分为“推理进程”（Agent，不可信，无执行权）和“引擎进程”（Engine，可信，负责执行）。两者通过 gRPC 接口通信，推理系统只能“提议”动作，不能直接“执行”。
*   **具有分级确定性的对抗性验证 (AV)**：引入 **Shield** 验证层，分为四级：
    *   **Tier 0 (策略层)**：基于 YAML 的硬编码规则，毫秒级响应。
    *   **Tier 1 (分类器层)**：启发式规则 + DeBERTa 模型，检测已知攻击模式。
    *   **Tier 2 (LLM 评估层)**：使用独立的、受限的 LLM 进行上下文安全评估。
    *   **Tier 3 (人工层)**：高风险动作需用户手动确认。
*   **信息流控制 (IFC)**：为数据打上敏感度标签（如 PUBLIC, RESTRICTED），标签随操作传播。如果智能体试图将敏感数据写入网络请求，Shield 会拦截。
*   **可逆执行 (Reversible Execution)**：通过 **Chronicle** 系统在执行破坏性动作前捕获状态快照，支持一键回滚。
*   **动态工具表面缩减 (DTSR)**：智能体默认不加载任何工具，仅在需要时动态加载特定工具组，缩小攻击面。

#### 3. 实验设计
*   **评估方法论**：提出**假设受损评估 (Assume-Compromise Evaluation, ACE)**。该方法直接绕过 LLM 推理层，将恶意工具调用直接注入执行边界，模拟推理系统完全沦陷的最坏情况。
*   **数据集/场景**：构建了包含 **280 个对抗性测试用例**的测试集，涵盖 9 大攻击类别（直接/间接注入、多步上下文操纵、工具链攻击、混淆攻击、多智能体渗透等）。
*   **对比方法**：

    *   **Config B**：仅提示词护栏（在 ACE 模式下自动失效，证明了其脆弱性）。
    *   **Config C**：Parallax 架构（分为默认配置和最高安全配置）。

#### 4. 资源与算力
*   **模型使用**：Tier 1 使用了本地部署的 **DeBERTa** 模型（ONNX 格式）；Tier 2 评估使用了 **Claude Haiku 4.5** API。

#### 5. 实验数量与充分性
*   **实验规模**：共进行了 **337 组实验**（280 组对抗测试 + 50 组误报测试 + 7 组人工干预测试）。

*   **拦截率**：在推理系统完全受损的情况下，Parallax 默认配置拦截了 **98.9%** 的攻击，误报率为 **0%**；最高安全配置实现了 **100%** 的拦截率。

#### 7. 优点（亮点）
*   **范式转移**：从“教 AI 做好人”转向“在物理架构上限制 AI 做坏事”，借鉴了操作系统特权分离的成熟思想。
*   **ACE 评估法**：为智能体安全测试提供了一个严苛且科学的标准。
*   **实用性**：OpenParallax 提供了 Go 语言的参考实现，且支持 MCP 协议，易于集成到现有工作流。
*   **可恢复性**：引入 Chronicle 快照机制，为安全防御提供了“最后一道防线”。

#### 8. 不足与局限
*   **延迟问题**：Tier 1 和 Tier 2 的验证延迟（秒级）对于实时性要求极高的场景可能存在挑战。
*   **脚本内容不透明**：Shield 目前主要检查命令本身（如 `python script.py`），但对脚本内部生成的恶意逻辑检测能力有限。

*   **外部副作用**：对于已发出的邮件或 API 调用，可逆执行（Chronicle）无法实现物理回滚。
