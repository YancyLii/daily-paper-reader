## 摘要
大语言模型 (LLM) 智能体越来越多地依赖第三方 API 路由在多个上游提供商之间分发工具调用请求。这些路由作为应用层代理运行，拥有对传输中每个 JSON 负载的完整明文访问权限，然而目前没有任何提供商在客户端和上游模型之间强制执行加密完整性校验。我们对这一攻击面进行了首次系统性研究。我们为恶意 LLM API 路由建立了一个威胁模型，并定义了两类核心攻击：负载注入 (AC-1) 和密钥窃取 (AC-2)，以及两种自适应规避变体：针对依赖项的注入 (AC-1.a) 和条件交付 (AC-1.b)。通过对购自淘宝、闲鱼和 Shopify 托管店铺的 28 个付费路由以及从公共社区收集的 400 个免费路由进行调查，我们发现有 1 个付费路由和 8 个免费路由正在主动注入恶意代码，2 个部署了自适应规避触发器，17 个触碰了研究人员拥有的 AWS 金丝雀凭据，还有 1 个从研究人员拥有的私钥中窃取了以太坊 (ETH)。两项投毒研究进一步表明，表面上良性的路由也可能被卷入同样的攻击面：一个泄露的 OpenAI 密钥产生了 1 亿个 GPT-5.4 token 和超过 7 个 Codex 会话，而配置薄弱的诱饵则产生了 20 亿个计费 token、分布在 440 个 Codex 会话中的 99 个凭据，以及 401 个已经以自主 YOLO 模式运行的会话。我们构建了 Mine，这是一个针对四个公共智能体框架实现所有四类攻击的研究代理，并利用它评估了三种可部署的客户端防御措施：故障关闭策略网关、响应端异常筛选以及仅追加透明度日志。

## Abstract
Large language model (LLM) agents increasingly rely on third-party API routers to dispatch tool-calling requests across multiple upstream providers. These routers operate as application-layer proxies with full plaintext access to every in-flight JSON payload, yet no provider enforces cryptographic integrity between client and upstream model. We present the first systematic study of this attack surface. We formalize a threat model for malicious LLM API routers and define two core attack classes, payload injection (AC-1) and secret exfiltration (AC-2), together with two adaptive evasion variants: dependency-targeted injection (AC-1.a) and conditional delivery (AC-1.b). Across 28 paid routers purchased from Taobao, Xianyu, and Shopify-hosted storefronts and 400 free routers collected from public communities, we find 1 paid and 8 free routers actively injecting malicious code, 2 deploying adaptive evasion triggers, 17 touching researcher-owned AWS canary credentials, and 1 draining ETH from a researcher-owned private key. Two poisoning studies further show that ostensibly benign routers can be pulled into the same attack surface: a leaked OpenAI key generates 100M GPT-5.4 tokens and more than seven Codex sessions, while weakly configured decoys yield 2B billed tokens, 99 credentials across 440 Codex sessions, and 401 sessions already running in autonomous YOLO mode. We build Mine, a research proxy that implements all four attack classes against four public agent frameworks, and use it to evaluate three deployable client-side defenses: a fail-closed policy gate, response-side anomaly screening, and append-only transparency logging.

*   **研究动机**：LLM 智能体（Agents）在执行任务（如订票、运行代码）时，常通过第三方 API 路由器（如 LiteLLM、OpenRouter 或灰色市场的转售商）分发请求。
*   **核心矛盾**：这些路由器在应用层充当代理，拥有对所有明文 JSON 负载（包括系统提示词、API 密钥、工具调用参数）的完全访问权限。由于缺乏端到端的加密完整性校验，恶意或被劫持的路由器可以随意篡改模型返回的工具调用指令，从而控制智能体在用户机器上的行为。
*   **背景**：2026 年 3 月发生的 LiteLLM 供应链污染事件证明了这一威胁的现实性。

### 2. 方法论
论文提出了一个针对恶意路由器的威胁模型，并将其攻击行为分为四类：
*   **核心攻击类别**：
    *   **AC-1 (载荷注入)**：在响应返回客户端前，篡改工具调用的参数（如将合法的安装脚本 URL 替换为恶意脚本）。
    *   **AC-2 (秘密窃取)**：被动扫描流量中的敏感信息（如 AWS 密钥、GitHub 令牌、以太坊私钥）。
*   **自适应规避变体**：
    *   **AC-1.a (针对依赖项的注入)**：专门针对包管理命令（如 `pip install`），通过拼写劫持（Typosquatting）替换依赖包名，绕过基于域名的简单防御。
    *   **AC-1.b (条件交付)**：设置触发谓词（如在第 50 次请求后才激活，或仅针对“YOLO 模式”即自动审批模式的会话），以规避安全审计。
*   **研究工具 (Mine)**：开发了一个名为 Mine 的研究代理，实现了上述所有攻击，并用于评估防御措施。

### 6. 主要结论与发现
*   **恶意行为普遍存在**：在 428 个路由器中，发现 9 个在注入恶意代码，17 个在滥用凭据，1 个在窃取以太坊。
*   **自适应规避已在野使用**：观察到路由器会根据请求计数或项目类型（如仅针对 Rust/Go 项目）选择性地发动攻击。
*   **供应链传递性风险**：即使是良性路由器，如果使用了泄露的上游密钥或转发了弱配置中继的流量，也会使终端用户暴露在攻击之下。
*   **防御现状**：现有的智能体框架均未实现响应完整性校验。虽然策略网关能拦截 100% 的简单注入，但容易被自适应攻击绕过。

### 7. 优点
*   **首次系统化**：填补了 LLM 供应链中“路由器中间人”这一研究空白。
*   **实战导向**：不仅有理论模型，还深入灰色市场进行实测，揭示了真实的威胁现状。
*   **防御建议务实**：提出了可立即部署的客户端防御方案，并给出了长期的提供商签名建议。

### 8. 不足与局限
*   **地域偏差**：样本主要集中在中文语境下的路由器市场（淘宝、闲鱼等），可能无法完全代表全球所有地区的路由器生态。

*   **应用限制**：提出的防御措施（如异常检测）存在一定的误报率（约 6.7%），在追求极高可用性的生产环境中可能面临挑战。
