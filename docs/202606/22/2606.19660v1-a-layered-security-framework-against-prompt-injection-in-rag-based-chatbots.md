## 摘要
在大型语言模型（LLM）部署中，提示注入被OWASP LLM应用十大风险评为最严重的漏洞，然而现有的防御措施仅在孤立的流水线阶段运行且不够完善。输入过滤器无法检查检索到的文档，而输出监控器则无法防止恶意载荷到达模型。因此，基于检索增强生成（RAG）的聊天机器人仍然易受间接注入攻击，其中被污染的知识库文档会危害所有查询检索到该文档的用户。我们提出了一个三层框架，在整个推理流水线中拦截直接和间接的提示注入。第一层使用基于规则的模板库和微调的语义异常分类器筛查用户输入。第二层在上下文组装期间强制执行基于来源的指令层级，防止检索内容覆盖操作者策略。第三层在交付前使用策略规则引擎和语义漂移检测器审计模型输出。连续的审计循环汇总结构化日志，并支持重训练以使分类器适应新兴攻击模式。该框架与模型无关，并作为中间件部署，无需修改底层LLM。在GPT-4o、Llama 3和Mistral 7B上对5,080个样本的评估表明，该框架将攻击成功率（ASR）从71.4%降低到11.3%，比最佳单层基线高出27.3个百分点，比已发表的护栏系统高出23.8个百分点，同时保持4.8%的假阳性率和61.2毫秒的中位延迟开销。消融研究证实，所有三层都提供了互补的保护，且其组合效果超过了各部分贡献之和。

## Abstract
Prompt injection is ranked as the most critical vulnerability in large language model (LLM) deployments by the OWASP Top 10 for LLM Applications, yet existing defenses operate at isolated pipeline stages and remain incomplete. Input filters cannot inspect retrieved documents, while output monitors cannot prevent malicious payloads from reaching the model. Consequently, retrieval-augmented generation (RAG) chatbots remain vulnerable to indirect injection, where a poisoned knowledge-base document compromises every user whose query retrieves it. We present a three-layer framework that intercepts both direct and indirect prompt injection throughout the inference pipeline. Layer 1 screens user input using a rule-based pattern library and a fine-tuned semantic anomaly classifier. Layer 2 enforces a provenance-based instruction hierarchy during context assembly, preventing retrieved content from overriding operator policy. Layer 3 audits model output using a policy rule engine and semantic drift detector before delivery. A continuous audit loop aggregates structured logs and supports retraining to adapt the classifier to emerging attack patterns. The framework is model-agnostic and deploys as middleware without modifying the underlying LLM. Evaluation on 5,080 samples across GPT-4o, Llama 3, and Mistral 7B shows that the framework reduces Attack Success Rate (ASR) from 71.4\% to 11.3\%, outperforming the best single-layer baseline by 27.3 percentage points and a published guardrail system by 23.8 percentage points, while maintaining a 4.8\% false positive rate and a median latency overhead of 61.2 ms. Ablation studies confirm that all three layers provide complementary protection and that their combined effect exceeds the sum of individual contributions.

### 2. 方法论：三层安全框架
论文提出了一种**模型无关、以中间件形式部署**的三层防御框架，无需修改底层LLM。框架在推理管线的三个关键节点协同工作：
*   **第一层：输入筛查**
    *   **核心思想**：在用户输入进入系统的最早阶段拦截直接注入攻击。
    *   **关键技术**：
        1.  **基于规则的模板匹配**：维护一个包含已知攻击模式的规则库（如“忽略之前的指令”），并对输入进行归一化（Unicode折叠、Base64解码等）以对抗混淆。匹配后可执行放行（带标记）、清洗或阻断三种处置。
        2.  **语义异常检测**：使用一个经微调的轻量级语义分类器（基于`sentence-transformers/all-MiniLM-L6-v2`），计算输入的“注入概率得分” \( s_1 \)。若得分超过阈值 \( \tau_1 \)，则标记为可疑并传递元数据至第三层。
*   **第二层：权限约束的上下文组装**
    *   **核心思想**：在组装提示上下文时，通过强制执行指令层级来解决LLM无法区分指令与数据这一根本性架构缺陷。
    *   **关键技术**：
        1.  **指令层级**：定义三个权限等级：**操作者层**（系统提示，最高权限）、**检索层**（知识库文档，参考数据）、**用户层**（用户输入，最低权限）。每个层级的操作权限有明确界定（见表5）。
        2.  **来源标记**：在上下文中为每个片段插入`[SYS]`、`[RET]`、`[USR]`等标签，并附加元指令，明确告知模型拒绝低权限指令覆盖高权限指令。
        3.  **检索内容扫描**：在将检索到的文档块送入上下文前，使用简化版的第一层规则库进行预扫描，移除或标记`[FLAGGED]`含有明显指令覆盖模式的块。
*   **第三层：输出审计**
    *   **核心思想**：作为最后一道防线，在模型生成回复后、交付用户前进行安检，无论注入攻击来自哪个渠道。
    *   **关键技术**：
        1.  **策略规则引擎**：基于攻击者目标，设定一系列布尔谓词进行检测，包括：**系统提示相似度**（防数据泄露，余弦相似度 \( \geq \theta_{sp} \) 则阻断）、**角色令牌检测**（防行为操纵）、**有害内容评分**（使用Llama Guard 2，得分 \( \geq \tau_3 \) 则阻断）。
        2.  **语义漂移检测**：计算模型回复与部署时采样的合规回复之间的嵌入余弦距离。当漂移超过阈值 \( \tau_{drift} \) 时，不直接阻断，而是触发升级审查，以减少对合法回复的误伤。
*   **持续审计循环**
    *   **功能**：这不是第四层，而是一个跨层运营组件，收集所有层的结构化日志，支持会话级和群体级的攻击警报，并将确认的误报和漏报案例反馈给第一层分类器进行定期重训练，以适应新的攻击模式。

### 8. 不足与局限
*   **静态威胁模型**：假设攻击者为黑盒，未评估面对了解防御机制内部（如分类器阈值、规则库）的自适应白盒攻击者时的鲁棒性。

*   **知识库环境的局限性**：评估所用的知识库是受控的产品文档，未测试在面对内容更不可控的现实世界知识库（如社交媒体验证）时的表现。
*   **阈值敏感性**：多个检测阈值依赖于与测试集同分布的验证集校准，在真实生产环境中可能面临分布偏移导致误报率上升的风险。
*   **并发延迟未知**：报告的延迟是在单请求、专用硬件下测得的。在高并发生产环境中，共享的编码器模型可能成为瓶颈，导致延迟增加。
*   **隐式指令检测不足**：错误分析显示，间接注入中的隐式软指令是第二大绕过原因（25%），当前基于规则的检索文档扫描器对此类攻击力不从心。
