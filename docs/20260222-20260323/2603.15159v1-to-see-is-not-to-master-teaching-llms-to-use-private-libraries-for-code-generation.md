## 摘要
大语言模型（LLMs）在代码生成方面展现出强大的潜力，但在面向私有库的代码生成（其目标是使用私有库中的 API 生成代码）方面仍存在局限性。现有方法主要依赖于检索私有库 API 文档，并在推理时将相关知识注入上下文。然而，我们的研究表明这是不够的：即使提供了准确的所需知识，LLMs 仍然难以有效地调用私有库 API。为了解决这一局限性，我们提出了 PriCoder，这是一种通过自动合成数据来教 LLMs 调用私有库 API 的方法。具体而言，PriCoder 将私有库数据合成建模为图构建过程，并在两个图算子之间交替进行：（1）渐进式图演化（Progressive Graph Evolution），通过从基础样本逐步合成更多样化的训练样本来提高数据多样性；（2）多维图剪枝（Multidimensional Graph Pruning），通过严格的过滤流水线提高数据质量。为了支持严格的评估，我们基于测试模型不熟悉的近期发布库构建了两个新的基准测试。在三个主流 LLMs 上的实验表明，PriCoder 显著提升了面向私有库的代码生成能力，在许多设置下 pass@1 的增益超过 20%，同时对通用代码生成能力的影响微乎其微。我们的代码和基准测试已在 https://github.com/contact-eniacode/PriCoder 公开发布。

## Abstract
Large Language Models (LLMs) have shown strong potential for code generation, yet they remain limited in private-library-oriented code generation, where the goal is to generate code using APIs from private libraries. Existing approaches mainly rely on retrieving private-library API documentation and injecting relevant knowledge into the context at inference time. However, our study shows that this is insufficient: even given accurate required knowledge, LLMs still struggle to invoke private-library APIs effectively.   To address this limitation, we propose PriCoder, an approach that teaches LLMs to invoke private-library APIs through automatically synthesized data. Specifically, PriCoder models private-library data synthesis as the construction of a graph, and alternates between two graph operators: (1) Progressive Graph Evolution, which improves data diversity by progressively synthesizing more diverse training samples from basic ones, and (2) Multidimensional Graph Pruning, which improves data quality through a rigorous filtering pipeline. To support rigorous evaluation, we construct two new benchmarks based on recently released libraries that are unfamiliar to the tested models. Experiments on three mainstream LLMs show that PriCoder substantially improves private-library-oriented code generation, yielding gains of over 20% in pass@1 in many settings, while causing negligible impact on general code generation capability. Our code and benchmarks are publicly available at https://github.com/contact-eniacode/PriCoder.

*   **整体含义**：论文旨在通过自动化的数据合成与微调流程，让模型从“查阅手册”转变为“真正掌握”私有库的 API 调用逻辑。

### 2. 方法论：PriCoder 框架
PriCoder 将数据合成建模为一个**合成图（Synthesis Graph）**的构建过程，包含两个核心算子：
*   **渐进式图演化（Progressive Graph Evolution）**：
    *   **初始演化**：从 API 节点出发，随机采样 API 规格作为种子，生成基础的“需求-代码”样本节点。
    *   **迭代演化**：将多个已有的基础样本节点进行组合，促使模型生成需要协同调用多个 API 的复杂、多样化需求，从而探索更广的 API 组合空间。
*   **多维图剪枝（Multidimensional Graph Pruning）**：
    *   **语法校验**：使用 AST 解析器剔除无法编译的代码。
    *   **执行校验**：利用 LLM 自动生成单元测试，在沙箱环境中运行，剔除运行报错或断言失败的样本。
    *   **功能校验（LLM-as-a-Judge）**：再次调用 LLM 评估需求与代码的语义一致性，确保逻辑合理。
*   **训练与部署**：在清洗后的高质量合成数据集上对模型进行微调（如 LoRA）。该方法与 RAG 正交，可结合使用。

### 3. 实验设计
*   **数据集/基准测试**：

    *   *注：选择这些库是因为它们在主流模型训练截止日期之后发布，可视为“私有库”。*
    *   **HumanEval**：用于测试通用编程能力是否退化。
*   **对比方法**：

### 6. 主要结论与发现
*   **性能飞跃**：PriCoder 在多个设置下将 **pass@1 提升了 20% 以上**。例如，在 NdonnxEval 上，Qwen-7B 的 pass@1 从 23.37% 提升至 47.99%。
*   **RAG 的局限**：实验证实，单纯依靠 RAG 提升有限，模型在拥有文档的情况下依然会犯低级调用错误。
*   **通用能力保持**：在 HumanEval 上的测试显示，PriCoder **几乎不损害（甚至在某些情况下提升了）**模型的通用编程能力，因为它强化了模型遵循语法和测试驱动开发的习惯。
*   **可扩展性**：性能随合成数据规模增加而稳步提升，且使用较弱的模型合成数据也能让较强的模型受益。

### 7. 优点
*   **直击痛点**：敏锐捕捉到 LLM 在私有库场景下“看得见、用不对”的瓶颈。
*   **自动化闭环**：构建了从 API 文档到高质量训练数据的全自动流水线，极大地降低了人工标注成本。
*   **评估严谨**：避开了已被 LLM 训练过的旧库，通过选取 2024/2025 年的新库构建 Benchmark，保证了评估的真实性。
*   **正交性**：该方法不排斥 RAG，而是作为 RAG 的底层能力补充，具有很强的实战部署价值。

### 8. 不足与局限
*   **计算成本**：虽然 50 小时在企业级可接受，但对于频繁更新的库，每次更新都重新微调可能存在滞后性。
*   **LLM 依赖性**：数据质量高度依赖于“LLM-as-a-Judge”和单元测试生成的准确性，如果合成模型本身极弱，可能无法生成有效的测试用例。
*   **语言覆盖**：目前主要针对 Python 进行了验证，对于其他静态编译语言（如 C++、Java）在复杂私有环境下的表现尚待进一步探索。
*   **库的规模**：实验中的库 API 数量在数百个量级，对于拥有数万个 API 的超大型企业私有框架，图演化的效率可能面临挑战。
