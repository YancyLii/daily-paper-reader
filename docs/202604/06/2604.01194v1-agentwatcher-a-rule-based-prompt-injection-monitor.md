## 摘要
大语言模型（LLMs）及其应用（如智能体）极易受到提示词注入攻击。现有的最先进提示词注入检测方法存在以下局限性：（1）其有效性随上下文长度的增加而显著下降；（2）缺乏定义何为提示词注入的明确规则，导致检测决策具有隐式性、不透明且难以推理。在本研究中，我们提出了 AgentWatcher 来解决上述两个局限性。针对第一个局限性，AgentWatcher 将 LLM 的输出（例如智能体的动作）归因于一小组具有因果影响的上下文片段。通过将检测重点集中在相对较短的文本上，AgentWatcher 可以扩展到长上下文场景。针对第二个局限性，我们定义了一组规则来明确哪些构成或不构成提示词注入，并使用一个监控 LLM 基于归因文本对这些规则进行推理，从而使检测决策更具可解释性。我们在工具使用智能体基准测试和长上下文理解数据集上进行了全面评估。实验结果表明，AgentWatcher 能够有效检测提示词注入，并在无攻击情况下保持效用。代码可在 https://github.com/wang-yanting/AgentWatcher 获取。

## Abstract
Large language models (LLMs) and their applications, such as agents, are highly vulnerable to prompt injection attacks. State-of-the-art prompt injection detection methods have the following limitations: (1) their effectiveness degrades significantly as context length increases, and (2) they lack explicit rules that define what constitutes prompt injection, causing detection decisions to be implicit, opaque, and difficult to reason about. In this work, we propose AgentWatcher to address the above two limitations. To address the first limitation, AgentWatcher attributes the LLM's output (e.g., the action of an agent) to a small set of causally influential context segments. By focusing detection on a relatively short text, AgentWatcher can be scalable to long contexts. To address the second limitation, we define a set of rules specifying what does and does not constitute a prompt injection, and use a monitor LLM to reason over these rules based on the attributed text, making the detection decisions more explainable. We conduct a comprehensive evaluation on tool-use agent benchmarks and long-context understanding datasets. The experimental results demonstrate that AgentWatcher can effectively detect prompt injection and maintain utility without attacks. The code is available at https://github.com/wang-yanting/AgentWatcher.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）智能体（Agents）在现实场景中的广泛应用，**提示词注入（Prompt Injection）**攻击成为了严重的安全性威胁。攻击者通过在外部上下文（如邮件、网页）中嵌入恶意指令，诱导智能体执行非授权操作（如转账、泄露隐私）。
现有的检测方法存在两大核心痛点：
*   **长上下文失效**：随着输入文本变长，检测器的性能显著下降，难以处理智能体任务中常见的长文档。
*   **缺乏可解释性**：现有方法多依赖隐式表示或黑盒模型，无法给出明确的拒绝理由，且难以根据特定业务逻辑定制防御规则。

#### 2. 方法论：核心思想与关键技术
论文提出了 **AgentWatcher** 框架，其核心流程分为两个阶段：

*   **阶段一：因果重要上下文归因（Context Attribution）**
    *   **核心思想**：不检测全部文本，而是只检测对智能体当前动作（Action）产生因果影响的关键片段。
    *   **技术细节**：利用归因 LLM 的注意力权重，识别出接收到异常高注意力的“汇聚标记”（Sink Tokens）。
    *   **算法流程**：使用滑动窗口（大小为 $w_s$）计算窗口内的平均注意力得分，定位得分最高的 $K$ 个窗口，并向左右扩展（$w_l, w_r$）以保持语义完整，最终拼接成精简的归因文本 $C^*$。

*   **阶段二：基于规则的推理检测（Rule-based Detection）**
    *   **核心思想**：引入一个“监控 LLM”（Monitor LLM），根据一组显式的、可定制的规则 $R$，对归因文本 $C^*$ 进行逻辑推理。
    *   **技术细节**：监控模型不仅输出“是/否”的二元判断，还会生成推理过程（Reasoning），说明违反了哪条规则，并提取出恶意指令。
    *   **模型优化**：采用 **GRPO（组相对策略优化）** 算法对监控模型进行微调，使用基于 BLEU 分数的奖励函数，鼓励模型准确提取注入指令并引用规则。

#### 6. 主要结论与发现
*   **高效防御**：AgentWatcher 在所有基准测试中均表现优异，能将 ASR 降低至 1% 左右，同时实用性损失极小（通常 < 4%）。
*   **长文本优势**：在长达数万 token 的上下文中，AgentWatcher 凭借归因机制，依然能保持稳定的检测能力，而基准方法性能大幅下滑。
*   **跨模型泛化**：该方法作为插件式工具，对 GPT-4o、Claude-3、Gemini-2.0 等多种后端模型均有效。
*   **规则自动化**：实验证明，利用 LLM 自动生成的规则集与人工编写的规则具有相当的检测效力。

#### 7. 优点
*   **可解释性极强**：通过“推理过程 + 规则引用”的模式，让安全管理员能够理解为何某个请求被拦截。
*   **可扩展性好**：归因机制解决了长文本检测的算力瓶颈和精度稀释问题。
*   **高度灵活**：支持用户根据业务需求（如特定工具的白名单）自定义规则，平衡安全与业务需求。

#### 8. 不足与局限
*   **计算延迟**：由于引入了额外的监控 LLM 推理步骤，单次检测耗时较长（约 8-10 秒），可能不适用于对实时性要求极高的交互场景。
*   **归因依赖**：检测效果高度依赖于第一阶段归因的准确性。如果注入指令极其隐蔽且未引起注意力权重波动，可能会被漏掉。
*   **模糊地带挑战**：在某些复杂场景下（如 AgentDyn 案例），良性指令（如代他人操作）与恶意注入在特征上高度相似，模型仍存在误报风险。
