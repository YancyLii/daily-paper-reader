## 摘要
大型语言模型（LLM）代理在运行时越来越多地通过加载代理技能来扩展其能力，这些技能将自然语言规范（SKILL.md）与可执行脚本和资源配对。由于技能的行为依赖于自然语言指令和可执行代码，评估其安全性需要跨模态推理，从而产生新的语言与代码攻击面。攻击者可以在SKILL.md中呈现良性的工作流程，同时嵌入隐式指令，引导代理窃取敏感文件，即使脚本看似无害。这一攻击面尚未得到充分研究；先前的工作仅将技能视为提示注入向量或静态代码工件，使得跨模态交互引发的攻击在很大程度上未被测量。在我们的评估中，开源和商业技能扫描器仅分别检测到2%-8%和9%-17%的此类攻击。为弥补这一差距，我们引入SkillMutator，这是首个用于检测代理技能的安装时语言与代码跨模态攻击的基准。它模拟了涵盖13种攻击类别的对抗性突变过程，利用扫描器反馈迭代优化恶意技能，使注入的行为与合法工作流程难以区分。我们进一步提出一个四阶段推理轨迹蒸馏框架，将前沿教师模型的推理痕迹蒸馏到较小的开放权重模型中。这产生了一个可本地部署的扫描器，避免了第三方数据暴露和过多的API成本。在最强SkillMutator子集（n=76）上，我们蒸馏的模型（Qwen2.5-Coder-7B-Instruct）将检测率从17.1%提高到88.2%，超过了GPT-4o-mini（23.7%）和GPT-5.4-mini（79.0%），并达到前沿水平GPT-5.4（86.8%）。这些结果表明，在实际防御跨模态攻击时，无需依赖昂贵的前沿模型是可行的。

## Abstract
Large language model (LLM) agents increasingly extend their capabilities at runtime by loading Agent Skills, which pair natural-language specifications (SKILL.md) with executable scripts and resources. Because a skill's behavior relies on both natural-language instructions and executable code, assessing its safety requires cross-modal reasoning, creating a new language-and-code attack surface. Attackers can present a benign workflow in SKILL.md while embedding implicit directives that steer the agent to exfiltrate sensitive files, even if the scripts appear harmless. This attack surface remains understudied; prior work treats skills merely as prompt-injection vectors or static code artifacts, leaving attacks emerging from cross-modal interactions largely unmeasured. In our evaluation, open-source and commercial skill scanners detect only 2%-8% and 9%-17% of such attacks, respectively. To address this gap, we introduce SkillMutator, the first benchmark for install-time detection of language-and-code cross-modal attacks on Agent Skills. It emulates an adversarial mutation process across 13 attack categories, iteratively refining malicious skills using scanner feedback to make injected behaviors indistinguishable from legitimate workflows. We further propose a four-phase reasoning-trajectory distillation framework to distill frontier-teacher traces into smaller open-weight models. This produces a locally deployable scanner avoiding third-party data exposure and excessive API costs. On the strongest SkillMutator subset (n=76), our distilled model (Qwen2.5-Coder-7B-Instruct) improves detection from 17.1% to 88.2%, surpassing GPT-4o-mini (23.7%) and GPT-5.4-mini (79.0%), and reaching frontier-level GPT-5.4 (86.8%). These results show practical defense against cross-modal attacks is feasible without relying on costly frontier models.

-   **现状与缺口**：
    -   现有防御（规则扫描器、商业SAST/SCA、提示注入检测器）仅能捕获2%–17%的此类攻击，因为它们将技能视为孤立的提示注入向量或静态代码工件，无法分析跨模态交互。
    -   依赖前沿LLM进行扫描虽有效，但会带来高成本、API调用延迟和第三方内容暴露风险，在企业或受限网络环境中不实用。
    -   因此，需要一种能在安装时（而非运行时）对技能进行本地、低成本、高精度的跨模态安全检测方案。

## 2. 方法论

论文提出了两大部分：**SkillMutator基准生成框架**和**本地可部署LLM扫描器的四阶段推理轨迹蒸馏框架**。

### 2.1 SkillMutator：攻击基准生成

该框架通过模仿适应性攻击者，从良性技能出发生成扫描器难以察觉的恶意变异技能。

-   **输入**：良性技能包（包含SKILL.md、可执行脚本、资源文件）。
-   **四阶段变异流程**：
    1.  **技能分析**：LLM提取技能的声明目的、使用的工具库、受信操作、敏感资源及可修改文件列表。
    2.  **隐蔽性感知的攻击选择**：针对13种攻击类别（来自真实技能市场的威胁分类），评估哪些攻击能与该技能的正常流程无缝整合（隐蔽得分>80分），并保留至少3种以保证多样性。
    3.  **场景生成**：LLM为每个选中类别设计具体攻击场景，明确良性伪装、隐藏恶意行为、注入指令、激活条件及预期影响。
    4.  **技能变异**：LLM修改技能文件，将恶意代码注入辅助脚本，同时在SKILL.md中嵌入伪装的指令，使得恶意行为仅在代理遵循跨模态工作流时被触发。
-   **迭代规避精炼**：变异后的技能由多个扫描器（规则型、商业型、LLM型）检测。若被检出，则根据反馈重新生成变异，以消除可检测特征（如禁用`curl`、敏感路径等公开规则库中的模式），同时保留底层语义差异。最终产出高隐蔽性的变异技能。
-   **攻击类别**：涵盖四大威胁组共13类，例如数据窃取、持久化控制、供应链投毒、广告注入等。

### 2.2 四阶段推理轨迹蒸馏（本地扫描器训练）

将前沿教师模型（GPT-5.4）的跨模态安全分析能力蒸馏到小型的开放权重模型（如Qwen2.5-Coder-7B-Instruct）中，实现本地扫描。

-   **教师端四阶段分析链**（原则先行）：
    -   **阶段1：目的锚定**：从SKILL.md中提取技能的最小声明目的（动词-宾语对），并枚举必须的操作范围（文件、网络、命令等），作为基准允许列表。
    -   **阶段2：超范围检测**：扫描所有文件，检测任何超出阶段1允许范围的操作，或允许但危险的使用模式（如不安全反序列化）。
    -   **阶段3：安全原则推理**：将阶段2的证据映射到9项安全原则（如目的-动作对齐、信任边界完整性、知情透明度等），按原则对证据分组。
    -   **阶段4：攻击类别标注**：将原则违规与13种攻击类别关联，每个类别标签必须基于阶段3的现有证据，防止无根据的分类。
-   **学生模型微调**：
    -   使用教师模型为训练集技能生成上述四阶段推理轨迹。
    -   通过LoRA（低秩适应）将轨迹蒸馏到Qwen2.5-Coder-7B-Instruct模型中，使其学会执行相同的链式分析。
    -   **元数据确定性精炼**：若教师模型在阶段4遗漏了该技能的真实攻击类别，则利用变异日志中的元数据合成一个证据子弹，并插入阶段3和阶段4的训练轨迹中，确保训练信号覆盖所有目标类别，且无需额外LLM调用。
-   **推理时预填充**：在推理时强制模型从阶段4的标题开始解码，引导其直接进入攻击类别标注阶段，提升检测效果（效果因模型而异）。

## 6. 主要结论与发现

1.  **现有防御严重失效**：规则型、商业型及提示注入检测器对跨模态攻击的检测率低于17%，因为它们无法理解自然语言指令和代码的交互语义。
2.  **专有大模型的局限性**：GPT-5.4虽能检测86.8%，但成本高昂且涉及第三方内容暴露，非理想部署选项。
3.  **四阶段蒸馏显著提升小模型能力**：通过推理轨迹蒸馏，Qwen2.5-Coder-7B-Instruct从17.1%提升至88.2%，超过GPT-5.4-mini（79.0%），并略微超越教师模型GPT-5.4（86.8%）。
4.  **框架泛化性好**：蒸馏方法对Llama、Mistral等不同基座模型均有效，可普遍提升检测率。
5.  **本地扫描器实现最佳性价比**：微调后的7B模型提供前沿级检测能力，且每次检出成本仅为GPT-5.4的约1/7，并避免了数据泄露风险。
6.  **每个分析阶段均有贡献**：消融实验证明，从目的锚定到攻击类别标注的四个阶段逐步且单调地提升了检测性能。
7.  **真实场景可迁移**：扫描器在真实技能市场数据上表现良好，总准确率82.5%，误报率仅8.0%，且对高严重性威胁检测更灵敏。

## 8. 不足与局限

-   **攻击类别覆盖可能不完全**：变异仅基于13类已知攻击，可能无法覆盖未来新出现的攻击模式；隐蔽性得分阈值（80分）的选择有一定主观性。
-   **评判依赖外部LLM**：LLM扫描器的检测率主要由GPT-5.4等评判模型决定，虽然有双评判验证，但评判模型的固有偏差和出错率仍可能影响精度计算。
-   **训练数据规模与分布**：蒸馏训练仅基于一个教师模型（GPT-5.4）和68个社区技能，样本量相对有限，在极端多样化或未来新型技能上的泛化能力仍需验证。
-   **误报率仍需关注**：虽然真实场景评估中误报率仅8.0%，但在安全敏感场景下，每100个干净技能产生8次误报，可能需要人工复核。精确率（60.9%）也表明存在改进空间。
-   **计算资源门槛**：尽管推理成本低，但微调阶段仍需A100 80GB GPU，对某些个人或小型团队可能有一定门槛。
-   **预填充效果依赖模型架构**：推理时的阶段4预填充对Qwen有效，但对Llama、Mistral等模型反而降低性能，导致该技巧不通用，需针对基座模型调整。
