## 摘要
OpenClaw 已迅速确立其作为领先开源自主智能体运行时的地位，提供了包括工具集成、本地文件访问和 Shell 命令执行在内的强大功能。然而，这些广泛的操作权限引入了严重的安全漏洞，将模型错误转化为切实的系统级威胁，例如敏感数据泄露、权限提升以及恶意第三方技能的执行。OpenClaw 生态系统现有的安全措施仍高度碎片化，仅处理智能体生命周期的孤立阶段，未能提供整体保护。为了填补这一空白，我们提出了 ClawKeeper，这是一个实时安全框架，在三个互补的架构层级上集成了多维保护机制：(1) 基于技能的保护在指令级别运行，将结构化安全策略直接注入智能体上下文，以强制执行特定环境的约束和跨平台边界；(2) 基于插件的保护作为内部运行时执行器，在整个执行流水线中提供配置加固、主动威胁检测和持续行为监控；(3) 基于观察者（Watcher）的保护引入了一种新型的、解耦的系统级安全中间件，持续验证智能体状态的演化。它能够在不与智能体内部逻辑耦合的情况下实现实时执行干预，支持停止高风险动作或强制人工确认等操作。我们认为，这种 Watcher 范式具有成为保护下一代自主智能体系统基础构建模块的巨大潜力。广泛的定性和定量评估证明了 ClawKeeper 在多种威胁场景下的有效性和鲁棒性。我们开源了代码。

## Abstract
OpenClaw has rapidly established itself as a leading open-source autonomous agent runtime, offering powerful capabilities including tool integration, local file access, and shell command execution. However, these broad operational privileges introduce critical security vulnerabilities, transforming model errors into tangible system-level threats such as sensitive data leakage, privilege escalation, and malicious third-party skill execution. Existing security measures for the OpenClaw ecosystem remain highly fragmented, addressing only isolated stages of the agent lifecycle rather than providing holistic protection. To bridge this gap, we present ClawKeeper, a real-time security framework that integrates multi-dimensional protection mechanisms across three complementary architectural layers. (1) \textbf{Skill-based protection} operates at the instruction level, injecting structured security policies directly into the agent context to enforce environment-specific constraints and cross-platform boundaries. (2) \textbf{Plugin-based protection} serves as an internal runtime enforcer, providing configuration hardening, proactive threat detection, and continuous behavioral monitoring throughout the execution pipeline. (3) \textbf{Watcher-based protection} introduces a novel, decoupled system-level security middleware that continuously verifies agent state evolution. It enables real-time execution intervention without coupling to the agent's internal logic, supporting operations such as halting high-risk actions or enforcing human confirmation. We argue that this Watcher paradigm holds strong potential to serve as a foundational building block for securing next-generation autonomous agent systems. Extensive qualitative and quantitative evaluations demonstrate the effectiveness and robustness of ClawKeeper across diverse threat scenarios. We release our code.

#### 1. 核心问题与整体含义（研究动机和背景）
随着自主智能体（如 OpenClaw）能力的增强，它们已能执行 Shell 命令、访问本地文件并集成第三方工具。这种高权限模式使得模型层面的错误（如幻觉或被诱导）直接转化为系统级的安全威胁，包括敏感数据泄露、权限提升和恶意代码执行。
目前 OpenClaw 生态的防御手段存在四大局限：
*   **防御碎片化**：仅针对特定阶段或威胁，缺乏统一视图。
*   **安全与效用的权衡**：防御逻辑嵌入在任务智能体内部，导致性能与安全的冲突。
*   **反应式防御**：多为事后日志分析，缺乏实时拦截能力。
*   **静态机制**：无法随智能体能力的进化而自我更新。
**ClawKeeper** 的提出旨在通过一个三层架构（技能、插件、观察者）提供全生命周期的实时主动防护。

#### 2. 方法论：核心思想与关键技术
ClawKeeper 采用了三层互补的防御架构：
*   **基于技能的保护（Skill-based）**：在指令层运行。通过向智能体注入结构化的 Markdown 安全策略文件，定义环境约束（如 Windows/Linux 差异）和软件操作规范（如飞书/Telegram 交互红线）。它还包含定时扫描脚本和交互总结组件。
*   **基于插件的保护（Plugin-based）**：在运行时层运行。作为硬编码的执行器，提供配置加固（如限制网关仅绑定 localhost）、关键文件（AGENTS.md 等）的哈希校验防篡改、以及持续的行为监控与日志记录。
*   **基于观察者的保护（Watcher-based）**：这是核心创新点。引入一个**解耦的外部监控智能体（Watcher）**。
    *   **独立性**：Watcher 是一个独立的 OpenClaw 实例，专门负责安全审计，不参与任务解决。
    *   **实时干预**：通过 WebSocket 接收任务智能体的实时状态，一旦发现高风险动作（如危险命令、超限工具调用），立即触发“人工确认（HITL）”或直接阻断。
    *   **自进化**：Watcher 作为一个智能体，能通过学习新的威胁案例不断更新其内部的安全知识库和判断阈值。

#### 6. 主要结论与发现
*   **性能领先**：ClawKeeper 在所有 7 类任务中均显著优于基准方法，防御成功率达到 **85%-90%**，比最优基准高出 15-45 个百分点。
*   **Watcher 的优越性**：解耦的 Watcher 架构解决了“任务-安全耦合”问题，且具备抗攻击性（攻击者无法通过任务指令卸载外部 Watcher）。
*   **自进化特性**：Watcher 的防御成功率随处理案例增加从 90% 提升至 95%，展现了动态适应新威胁的能力。
*   **多层协同**：单一层级的防御均有局限，三层结合能实现从静态配置到动态执行的闭环保护。

#### 7. 优点
*   **架构解耦**：Watcher 模式实现了“监管独立”，避免了安全逻辑干扰任务性能。
*   **全生命周期覆盖**：从安装时的配置检查到运行时的行为拦截，再到事后的审计报告。
*   **通用性强**：Watcher 范式不局限于 OpenClaw，可推广至任何具有通信接口的智能体系统。
*   **易于部署**：提供了本地和云端两种部署模式，兼顾隐私与扩展性。

#### 8. 不足与局限
*   **运行成本**：Watcher 模式本质上是运行两个智能体，会增加推理成本（Token 消耗）和系统资源占用。
*   **隐私风险**：在云端部署模式下，任务智能体的所有会话数据需传输至 Watcher，存在潜在的数据泄露风险。
*   **延迟问题**：实时监控和人工确认环节可能会引入一定的操作延迟，影响用户体验。
*   **模型依赖**：防御效果仍部分取决于底层 LLM（如 GLM-5）的理解能力，若模型发生严重幻觉，防御层可能失效。
