## 摘要
AI 智能体越来越多地通过模型上下文协议（MCP）调用外部工具（文件系统、网络、API）。这些工具调用相当于智能体的系统调用（syscalls）——即对共享状态产生副作用的特权操作——然而，目前的安全执行完全位于用户空间，仅需 10 行脚本即可将其绕过。本文提出了受控 MCP（Governed MCP），这是一种构建在基于 Logit 的安全原语（ProbeLogits，见配套论文：arXiv:2604.11943）之上的内核驻留工具治理网关。该网关通过一个六层流水线拦截每一次 MCP 工具调用：模式验证、信任层级检查、速率限制、对抗性预过滤、ProbeLogits 门控（核心语义检查）以及宪法策略匹配，并配有 Blake3 哈希审计链。我在 Anima OS 中实现了受控 MCP，这是一个使用约 86,000 行 Rust 代码编写的裸机 x86_64 操作系统。五个非推理层为每次调用增加了 65.3 微秒的开销；在 7B Q4_0 模型上，ProbeLogits 增加了 65 毫秒（每个 token 类别的语义决策）的延迟。在包含 101 个提示词的 MCP 领域基准测试中进行的四种配置消融实验表明，移除 ProbeLogits 层会导致 F1 分数从 0.773 骤降至 0.327（Delta F1 = -0.446），证明仅靠手动规则防火墙是不够的。运行时中所有 15 个 WASM 到系统的宿主函数均通过该网关路由（实现了对 WASM ABI 表面的完全调解；该声明的范围和注意事项见第 4.6 节）；对于内核驻留门控而言，能够击败现有护栏库的 10 行代码用户空间绕过在结构上是不可能的。

## Abstract
AI agents increasingly call external tools (file system, network, APIs) through the Model Context Protocol (MCP). These tool calls are the agent's syscalls -- privileged operations with side effects on shared state -- yet today's safety enforcement lives entirely in userspace, where a 10-line script can bypass it. I propose Governed MCP, a kernel-resident tool governance gateway built on a logit-based safety primitive (ProbeLogits, companion paper: arXiv:2604.11943). The gateway interposes on every MCP tool call in a 6-layer pipeline: schema validation, trust tier check, rate limit, adversarial pre-filter, ProbeLogits gate (the load-bearing semantic check), and constitutional policy match, with a Blake3-hashed audit chain.   I implement Governed MCP in Anima OS, a bare-metal x86_64 OS in approximately 86,000 lines of Rust. The five non-inference layers add 65.3 microseconds of overhead per call; ProbeLogits adds 65 ms (per-token-class semantic decision) on 7B Q4_0. A 4-config ablation on a 101-prompt MCP-domain benchmark shows that removing the ProbeLogits layer collapses F1 from 0.773 to 0.327 (Delta F1 = -0.446) -- hand-rule firewalling alone is insufficient. All 15 WASM-to-system host functions in the runtime route through the gateway (complete mediation of the WASM ABI surface; the scope and caveats of this claim are stated in Section 4.6); a 10-LoC userspace bypass that defeats existing guardrail libraries is structurally impossible against the kernel-resident gate.

### 2. 方法论：核心思想与技术细节
*   **核心思想**：在自研的 **Anima OS**（基于 Rust 的裸机操作系统）内核中实现一个 6 层治理网关，强制拦截所有从 WASM 沙箱发出的工具调用。
*   **六层流水线（Pipeline）**：
    1.  **模式验证（Schema Validation）**：检查 JSON-RPC 格式和参数合法性。
    2.  **信任层级检查（Trust Tier）**：根据智能体身份（系统级、原生 AI 等）匹配工具白名单。
    3.  **速率限制（Rate Limit）**：防止拒绝服务攻击或病态循环。
    4.  **对抗性预过滤（Adversarial Pre-filter）**：使用正则 DFA 快速拦截已知的提示词注入模式。
    5.  **ProbeLogits 门控（核心语义检查）**：利用内核推理引擎，读取模型在“安全/危险”词元位置的 Logit 概率差，进行语义意图分类。
    6.  **宪法策略匹配（Constitutional Policy）**：应用 12 条预设的原则性规则。
*   **关键技术**：
    *   **WASM 隔离**：智能体运行在 WASM 沙箱中，唯一的外部出口是内核暴露的宿主函数，这些函数在结构上强制路由至治理网关。
    *   **Fail-Closed 机制**：如果推理引擎故障，网关默认拒绝所有工具调用。
    *   **审计链**：使用 Blake3 哈希链记录所有决策，确保审计日志不可篡改。

### 6. 主要结论与发现
*   **语义检查不可或缺**：移除 ProbeLogits 层后，F1 分数下降了 0.446。这证明仅靠手动规则（防火墙式过滤）无法应对复杂的语义攻击。

*   **结构化防御**：通过内核驻留，成功消除了现有安全库易被“猴子补丁”绕过的结构性缺陷。

### 7. 优点
*   **架构创新**：首次将 AI 语义安全检查下沉到操作系统内核边界，实现了真正的“完全调解（Complete Mediation）”。
*   **高性能语义原语**：相比于调用外部模型（秒级延迟），基于 Logit 的 ProbeLogits 显著降低了安全检查的开销。
*   **安全性高**：利用 Rust 语言和 WASM 沙箱提供了较强的内存安全和隔离保障。

### 8. 不足与局限
*   **推理延迟瓶颈**：虽然比用户态方案快，但 65 毫秒的延迟在多租户高并发场景下可能成为瓶颈（目前受限于 CPU/内存带宽）。
*   **数据集偏差**：Custom-101 是作者自标注的，可能存在主观偏差，缺乏第三方红队测试。
*   **硬件支持有限**：目前不支持 GPU/NPU 加速，这限制了其在更大模型或更高吞吐量场景下的应用。
*   **模型依赖性**：系统的安全性高度依赖于底层加载的 LLM 的质量，若模型本身被投毒，治理网关将失效。
