## 摘要
将大语言模型（LLMs）部署为自主浏览器智能体，会以间接提示注入（IPI）的形式暴露巨大的攻击面。基于云的防御可以提供强大的语义分析，但会引入延迟并引发隐私担忧。我们提出了“认知防火墙”（Cognitive Firewall），这是一种三阶段的分离计算架构，将安全检查分布在客户端和云端。该系统由本地视觉哨兵（Sentinel）、基于云的深度规划器（Deep Planner）以及在执行时强制执行策略的确定性守卫（Guard）组成。在 1,000 个对抗样本中，仅靠边缘端的防御无法检测到 86.9% 的语义攻击。相比之下，完整的混合架构将总攻击成功率（ASR）降低到 1% 以下（静态评估下为 0.88%，自适应评估下为 0.67%），同时保持对副作用操作的确定性约束。通过在本地过滤表现层攻击，该系统避免了不必要的云端推理，并比纯云端基准实现了约 17,000 倍的延迟优势。这些结果表明，执行边界处的确定性强制执行可以补充概率性语言模型，且分离计算为保护交互式 LLM 智能体提供了实用的基础。

## Abstract
Deploying large language models (LLMs) as autonomous browser agents exposes a significant attack surface in the form of Indirect Prompt Injection (IPI). Cloud-based defenses can provide strong semantic analysis, but they introduce latency and raise privacy concerns. We present the Cognitive Firewall, a three-stage split-compute architecture that distributes security checks across the client and the cloud. The system consists of a local visual Sentinel, a cloud-based Deep Planner, and a deterministic Guard that enforces execution-time policies. Across 1,000 adversarial samples, edge-only defenses fail to detect 86.9% of semantic attacks. In contrast, the full hybrid architecture reduces the overall attack success rate (ASR) to below 1% (0.88% under static evaluation and 0.67% under adaptive evaluation), while maintaining deterministic constraints on side-effecting actions. By filtering presentation-layer attacks locally, the system avoids unnecessary cloud inference and achieves an approximately 17,000x latency advantage over cloud-only baselines. These results indicate that deterministic enforcement at the execution boundary can complement probabilistic language models, and that split-compute provides a practical foundation for securing interactive LLM agents.

### 论文总结：认知防火墙（The Cognitive Firewall）

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）被广泛部署为自主浏览器智能体（如自动订票、管理工单），它们面临着严重的**间接提示注入（IPI）**威胁。由于智能体在同一个上下文窗口中处理用户指令（控制平面）和外部网页内容（数据平面），攻击者可以通过在网页中嵌入不可见的恶意指令（如零透明度文本）来操纵智能体，导致敏感信息泄露或未经授权的操作。
现有的防御手段存在矛盾：**云端防御**语义分析强但延迟高（>500ms）且有隐私风险；**边缘端防御**延迟低但难以识别复杂的认知操纵。本文旨在通过一种“端云协同”的架构，平衡安全深度与响应速度。

#### 2. 方法论：核心思想与关键技术
论文提出了**认知防火墙（Cognitive Firewall）**，采用“防御漏斗”模型，将安全检查分为三个阶段：
*   **第一层：边缘哨兵（Edge Sentinel - 视觉感知层）**
    *   **核心思想**：利用浏览器内置的轻量级模型（如 Gemini Nano）和 DOM/CSS 启发式算法，确保智能体只处理用户“看得见”的内容。
    *   **技术细节**：通过遍历 DOM 树并检查 `opacity`、`font-size`、`z-index` 等属性，过滤掉隐藏文本。若检测到视觉异常，直接在本地拦截，不触发云端推理。
*   **第二层：云端深度规划器（Deep Planner - 语义推理层）**
    *   **核心思想**：将清洗后的文本发送至云端大模型（如 Llama 3 或 GPT-4），进行高层意图分析。
    *   **技术细节**：使用专门的“安全分析师”系统提示词，识别角色扮演、指令覆盖或虚假紧急情况等语义攻击。
*   **第三层：起源守卫（Origin Guard - 执行对齐层）**
    *   **核心思想**：在执行边界进行确定性拦截。
    *   **技术细节**：作为同步 JavaScript 拦截器，强制执行白名单策略和谓词策略（如：若用户目标是“只读”，则拦截所有 POST/DELETE 请求），确保即使模型被诱导，恶意动作也无法生效。

#### 7. 优点
*   **分层防御思想**：将低成本、高频率的检查放在边缘，高成本、深度的检查放在云端，极大地优化了性能。

*   **延迟累积**：虽然边缘层极快，但若进入完整流程，总延迟仍接近 1 秒，对实时性要求极高的场景仍有挑战。
