## 摘要
在 LLM/VLM 智能体中，提示词隐私风险会传播到单次模型调用之外，因为原始用户内容可能会流入检索查询、内存写入、工具调用和日志中。现有的去标识化流水线解决了文档边界问题，但未能应对这种跨阶段传播。我们提出了 BodhiPromptShield，这是一个策略感知框架，它检测敏感片段，通过类型化占位符、语义抽象或安全符号映射对其进行路由，并将恢复操作延迟至授权边界。相比于企业级脱敏，该框架增加了显式的传播感知调解，并将恢复时机作为安全变量。在受控提示词隐私基准（CPPB）的受控评估下，检索、内存和工具阶段的分阶段传播率从 10.7% 降至 7.1%；PER 达到 9.3%，AC 为 0.94，TSR 为 0.92，优于通用的去标识化方法。这些是基于 CPPB 的受控系统结果，而非正式的隐私保证或公共基准迁移声明。项目仓库见 https://github.com/mabo1215/BodhiPromptShield.git。

## Abstract
In LLM/VLM agents, prompt privacy risk propagates beyond a single model call because raw user content can flow into retrieval queries, memory writes, tool calls, and logs. Existing de-identification pipelines address document boundaries but not this cross-stage propagation. We propose BodhiPromptShield, a policy-aware framework that detects sensitive spans, routes them via typed placeholders, semantic abstraction, or secure symbolic mapping, and delays restoration to authorized boundaries. Relative to enterprise redaction, this adds explicit propagation-aware mediation and restoration timing as a security variable. Under controlled evaluation on the Controlled Prompt-Privacy Benchmark (CPPB), stage-wise propagation suppresses from 10.7\% to 7.1\% across retrieval, memory, and tool stages; PER reaches 9.3\% with 0.94 AC and 0.92 TSR, outperforming generic de-identification. These are controlled systems results on CPPB rather than formal privacy guarantees or public-benchmark transfer claims.   The project repository is available at https://github.com/mabo1215/BodhiPromptShield.git.

#### 1. 核心问题与研究动机
*   **核心问题**：在大型语言模型（LLM）和视觉语言模型（VLM）驱动的智能体（Agent）系统中，隐私风险不仅存在于单次模型调用中，还会通过检索（RAG）、内存写入、工具调用和系统日志进行**跨阶段传播**。
*   **研究动机**：传统的脱敏技术（如企业级遮盖）通常只关注文档边界的单次处理，忽略了隐私片段在智能体工作流中的流转。如果处理不当，敏感信息（如发票ID、地址）可能在最终执行前就已泄露到中间环节。本文旨在研究如何通过推理前的中介处理，在抑制隐私传播的同时，保留足够的语义供下游任务执行。

#### 2. 方法论
BodhiPromptShield 提出了一个四阶段的推理前中介框架：
*   **敏感片段提取**：结合正则表达式、命名实体识别（NER）、上下文隐私判断以及 OCR（针对多模态输入），识别提示词中的敏感跨度。
*   **策略感知的提示词脱敏**：根据隐私类别和下游需求，通过策略选择器（$\Pi_\pi$）将敏感片段路由至三种模式：
    1.  **类型化占位符替换**：如将姓名替换为 `<PERSON_1>`。
    2.  **语义抽象**：将具体地址替换为“奥克兰的某处住宅地址”，保留推理所需的语义。
    3.  **安全符号映射**：使用不透明的令牌（Token）替换敏感值，并将原始映射存储在受控的本地映射表中。
*   **下游智能体推理**：智能体在脱敏后的提示词上运行，减少了中间环节（检索、内存）的原始隐私暴露。
*   **受控恢复**：仅在授权的执行边界（如调用特定工具时），根据映射表恢复原始值。
*   **核心思想**：将“恢复时机”视为一个安全变量，通过延迟恢复来切断隐私在智能体内部状态中的传播路径。

#### 6. 主要结论与发现
*   **抑制传播有效**：BodhiPromptShield 将跨阶段传播率从 10.7% 抑制到 7.1%，显著优于正则脱敏（约 60%）。

*   **恢复时机至关重要**：实验证明，延迟到工具边界再恢复原始值，可以在不损失任务成功率的前提下，大幅降低中间环节的泄露风险。
*   **对抗性发现**：虽然 Unicode 硬化能降低部分风险，但复杂的语义绕过和上下文推理攻击仍是潜在威胁。

#### 7. 优点
*   **传播感知设计**：首次明确将提示词隐私视为一个“传播控制问题”，而非简单的静态遮盖问题。
*   **策略灵活性**：提供了“宽松、平衡、严格”三种策略配置文件，允许根据业务场景调整隐私与效用的权重。
*   **多模态支持**：将 OCR 提取的文本纳入统一的脱敏逻辑，适应现代 VLM 智能体的需求。
*   **实验严谨**：引入了 SPE 指标来量化隐私在智能体内部的流转，更符合实际部署情况。

#### 8. 不足与局限
*   **上下文依赖性**：隐私片段的识别高度依赖上下文，在复杂或低质量 OCR 场景下可能存在误判或漏判。
*   **缺乏形式化保证**：该框架属于启发式防御，不提供类似差分隐私（DP）或同态加密的严格数学隐私证明。
*   **推理攻击风险**：即使遮盖了直接标识符，攻击者仍可能通过剩余的语义上下文推断出敏感属性（如通过病历描述推断出罕见病）。
*   **应用限制**：对于必须在检索阶段使用原始精确值的极少数任务，该框架可能会导致检索质量下降。
