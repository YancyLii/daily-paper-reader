## 摘要
在许多实际的大语言模型（LLM）部署中，通常使用单一的护栏（guardrail）来同时进行提示词和响应审核。提示词审核针对的是完全观测到的文本，而流式响应审核则要求在部分生成的文本上做出安全决策。现有的基于文本的流式护栏通常将这一输出端问题建模为边界检测，即训练模型识别出响应已经变得不安全的最早前缀。在这项工作中，我们提出了 StreamGuard，这是一种统一的、与模型无关的流式护栏，它将审核重新定义为一个预测问题：给定一个部分前缀，模型预测未来可能续写内容的预期有害性。我们使用蒙特卡洛展开（Monte Carlo rollouts）来监督这一预测，从而在不需要精确的词元级（token-level）边界标注的情况下实现早期干预。在标准安全基准测试中，StreamGuard 在输入审核和流式输出审核方面均表现出色。在 8B 参数规模下，相较于 Qwen3Guard-Stream-8B-strict，StreamGuard 将综合输入审核 F1 分数从 86.7 提升至 88.2，将综合流式输出审核 F1 分数从 80.4 提升至 81.9。在 QWENGUARDTEST response_loc 流式基准测试中，StreamGuard 达到了 97.5 的 F1 分数、95.1 的召回率和 92.6% 的准时干预率，而 Qwen3Guard-Stream-8B-strict 的对应指标分别为 95.9 F1、92.1 召回率和 89.9%，同时漏检率从 7.9% 降低至 4.9%。我们进一步证明，基于预测的监督可以有效地跨分词器和模型家族进行迁移：通过迁移目标，Gemma3-StreamGuard-1B 达到了 81.3 的响应审核 F1 分数、98.2 的流式 F1 分数和 3.5% 的漏检率。这些结果表明，无需精确的边界标签即可获得强大的端到端流式审核能力，且预测未来风险是实现低延迟安全干预的一种有效监督策略。

## Abstract
In many practical LLM deployments, a single guardrail is used for both prompt and response moderation. Prompt moderation operates on fully observed text, whereas streaming response moderation requires safety decisions to be made over partial generations. Existing text-based streaming guardrails commonly frame this output-side problem as boundary detection, training models to identify the earliest prefix at which a response has already become unsafe. In this work, we introduce StreamGuard, a unified model-agnostic streaming guardrail that instead formulates moderation as a forecasting problem: given a partial prefix, the model predicts the expected harmfulness of likely future continuations. We supervise this prediction using Monte Carlo rollouts, which enables early intervention without requiring exact token-level boundary annotations.   Across standard safety benchmarks, StreamGuard performs strongly both for input moderation and for streaming output moderation. At the 8B scale, StreamGuard improves aggregated input-moderation F1 from 86.7 to 88.2 and aggregated streaming output-moderation F1 from 80.4 to 81.9 relative to Qwen3Guard-Stream-8B-strict. On the QWENGUARDTEST response_loc streaming benchmark, StreamGuard reaches 97.5 F1, 95.1 recall, and 92.6% on-time intervention, compared to 95.9 F1, 92.1 recall, and 89.9% for Qwen3Guard-Stream-8B-stric, while reducing the miss rate from 7.9% to 4.9%. We further show that forecasting-based supervision transfers effectively across tokenizers and model families: with transferred targets, Gemma3-StreamGuard-1B reaches 81.3 response-moderation F1, 98.2 streaming F1, and a 3.5% miss rate. These results show that strong end-to-end streaming moderation can be obtained without exact boundary labels, and that forecasting future risk is an effective supervision strategy for low-latency safety intervention.

### 1. 核心问题与整体含义
*   **研究动机**：在 LLM 的实际部署中，流式输出（Streaming）已成为主流，但现有的安全护栏（如 LlamaGuard）大多针对完整文本设计，在流式场景下会面临“高延迟”或“安全窗口暴露”的矛盾。
*   **核心挑战**：现有的流式审核方法（如 Qwen3Guard-Stream）通常将其视为“边界检测”问题，即识别文本何时已经变得不安全。这种方法依赖昂贵的词元级（Token-level）标注，且往往在有害内容已经出现后才做出反应。
*   **整体含义**：论文提出将流式审核从“被动反应”转变为“主动预测”，通过预测当前前缀未来可能生成的有害性期望值，实现更早、更准的干预。

### 2. 方法论
*   **核心思想**：引入**基于价值的安全预测（Value-Based Safety Forecasting）**。模型不再判断当前前缀是否违规，而是估计在给定当前前缀的情况下，未来可能生成的续写内容的预期有害性。
*   **关键技术细节**：
    *   **前缀条件安全值**：定义一个函数 $V^*(x, y_{\le t})$，表示在提示词 $x$ 和当前前缀 $y_{\le t}$ 下，未来续写内容被安全判别器判定为有害的概率期望。
    *   **蒙特卡洛采样（Monte Carlo Rollouts）**：为了近似上述期望值，在训练阶段，针对每个前缀，使用多个生成器（如 Llama 3、Qwen 2.5 等）采样 $M$ 条可能的未来路径，并使用离线安全判别器打分，取平均值作为该前缀的监督目标（Soft Target）。
    *   **训练目标**：使用二元交叉熵损失函数，训练一个轻量级的分类头（Linear Head），使其在每个 Token 位置输出未来风险预测值。
*   **优势**：这种方法不需要精确的违规边界标注，且由于监督信号是在文本空间定义的，因此具有极强的跨模型、跨分词器迁移能力。

### 3. 实验设计
*   **数据集**：
    *   **输入审核**：ToxicChat, OpenAI Moderation, Aegis 2.0, SimpleSafetyTests, HarmBench, WildGuard Test。

    *   **流式基准**：QWEN GUARD TEST (response_loc)，用于评估干预的及时性。
*   **对比方法**：LlamaGuard 3/4, WildGuard, PolyGuard, 以及最强的流式基准 Qwen3Guard-Stream。
*   **评估指标**：F1 分数、召回率、准时干预率（OnTime）、漏检率（Miss）和误报率（FPR）。

### 5. 实验数量与充分性
*   **实验规模**：论文在多个参数规模（1B 到 8B）和多个模型家族（Llama, Gemma, Qwen）上进行了验证。
*   **消融实验**：非常充分。作者研究了不同的聚合规则（均值、最大值、中位数）、采样生成器的组合方式、以及监督信号的密度对性能的影响。
*   **客观性**：所有实验均运行了 3 个随机种子并报告了均值和标准差，对比基准涵盖了目前市面上最先进的开源护栏模型，实验设计公平且客观。

### 6. 主要结论与发现
*   **性能领先**：在 8B 规模下，StreamGuard 的输入审核 F1（88.2）和输出审核 F1（81.9）均优于 Qwen3Guard-Stream。
*   **干预更及时**：在流式测试中，StreamGuard 实现了 92.6% 的准时干预率，显著高于基准模型的 89.9%，同时将漏检率从 7.9% 降至 4.9%。
*   **迁移性强**：证明了在 Llama 架构上生成的预测目标可以完美迁移到 Gemma 和 Qwen 模型上，且性能依然强劲。
*   **低延迟**：决策速度极快，能够与主流 LLM 的流式输出节奏完美匹配。

### 7. 优点
*   **前瞻性**：通过预测未来风险而非检测既定事实，实现了真正的“早期干预”。
*   **模型无关性**：不需要访问生成器的内部激活状态（White-box），仅靠文本前缀即可工作，适配性广。
*   **标注效率**：利用蒙特卡洛采样自动生成监督信号，规避了昂贵的人工词元级标注。

### 8. 不足与局限
*   **训练成本**：虽然推理很快，但训练阶段需要进行大量的蒙特卡洛采样（Rollouts），这需要消耗显著的计算资源来构建数据集。
*   **策略偏差**：性能受限于采样时使用的离线安全判别器（Oracle），如果判别器本身存在偏见或错误，会直接传递给 StreamGuard。
*   **长文本挑战**：对于极长文本的流式审核，采样覆盖的未来路径可能不足以代表所有潜在风险。
