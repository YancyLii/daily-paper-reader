## 摘要
代码大语言模型日益从代码库、文档、问题讨论串和编码代理环境中检索外部代码上下文，形成了一个间接提示注入攻击面，攻击者将指令隐藏在注释、字符串、标识符或伪装代码中。我们提出CodeSentinel，一个三层的推理时净化器。它利用Tree-sitter提取面向模型的高风险CST节点，然后结合语法引导预过滤、CST引导的动态Min-K%评分和节点扰动分析，以检测对抗性和自然语义触发器。检测到的节点在到达下游代码LLM之前被移除或中和。在六种最新的攻击家族中，CodeSentinel取得了平均节点级F1分数0.80，优于CodeGarrison、DePA和KillBadCode。

## Abstract
Code large language models increasingly retrieve external code context from repositories, documentation, issue threads, and coding-agent environments, creating an indirect prompt-injection surface where attackers hide instructions in comments, strings, identifiers, or decoy code. We propose CodeSentinel, a three-layer inference-time sanitizer. It uses Tree-sitter to extract high-risk model-facing CST nodes, then combines syntax-guided pre-filtering, CST-guided Dynamic Min-K\% scoring, and node perturbation analysis to detect adversarial and natural-looking semantic triggers. Detected nodes are removed or neutralized before reaching the downstream Code LLM. Across six recent attack families, \CodeSentinel achieves 0.80 average node-level F1, outperforming CodeGarrison, DePA, and KillBadCode.

## 1. 论文的核心问题与整体含义

- **核心问题**：代码大模型（Code LLMs）在从仓库、文档、问题讨论串等外部来源检索代码上下文时，极易遭受**间接提示注入攻击**（indirect prompt injection）。攻击者可将恶意指令嵌入注释、字符串字面量、标识符或伪装（decoy）代码块中，这些内容对模型可见但通常对运行无影响，容易被忽视。
- **整体含义**：现有防御手段（如关键词过滤、文件级困惑度检测、基于训练的方法）难以有效应对代码上下文中复杂多变、自然语义化的注入负载。论文提出一个推理时（inference-time）的三层净化器 **CodeSentinel**，在代码进入下游模型之前，从语法和语义层面检测并清除注入内容，填补了该空白。

## 2. 方法论

### 核心思想
- 将代码视为**结构化输入**，而非纯文本。用Tree-sitter解析出具体语法树（CST），提取面向模型的高风险节点（注释、字符串、标识符、低可达性但高显著的“伪装”节点）。
- 对高风险候选节点依次应用三个互补的检测层，被前层标记的节点不再进入后层（早退机制），最后将所有标记节点进行净化（sanitization）后再送入下游模型。

### 三层架构及关键公式/流程

#### Layer 1：语法引导预过滤
- **目标**：极低开销地捕获显式或结构可疑的负载。
- 对每个节点计算三项风险得分的最大值：
  \[
  s_1(n) = \max(R_{\text{regex}}(n), R_{\text{lex}}(n), R_{\text{graph}}(n;X))
  \]
  - \(R_{\text{regex}}\)：基于正则匹配“忽略此前指令”、角色控制指令、命令模板等模式。
  - \(R_{\text{lex}}\)：检测不可见Unicode、异常标点密度、重复控制token、过长连续字符串等。
  - \(R_{\text{graph}}\)：结合模型显著度与代码可达性（Call-graph、控制流、数据流），捕捉“显著但不连接”的伪装节点：  
    \[
    R_{\text{graph}}(n;X) = \text{Sal}(n;X) \cdot (1 - \text{Reach}(n))
    \]

#### Layer 2：CST引导的动态 Min-K%
- **目标**：检测ShadowCode、INSEC等通过非功能区域注入的统计异常负载。
- 以节点为单位（而非整个文件）计算每个 token 的负对数似然（NLL）：
  \[
  r_i(n) = -\log q_{\phi}(z_i \mid X_{<\text{pos}(z_i)})
  \]
- 根据节点类型、长度和熵动态调整Min-K%选取的比例：
  \[
  k_{\text{dyn}}(n) = \text{clip}\left( \frac{k_{\text{base}}(t_n) H(n)}{\log \max(N_{\text{tok}}(n),2)}, k_{\min}, k_{\max} \right)
  \]
  - 对节点内NLL最高的前 \(M(n)\) 个 token 求均值 \(\text{MeanTopK}(n)\)。
  - 同时计算滑动窗口最大损失 \(\text{MaxWinLoss}(n)\) 和相邻 token 间最大跳跃 \(\text{MaxSpike}(n)\)。
- 使用同一CST类型的干净验证节点进行Z-score校准：
  \[
  Z_h(n) = \frac{h(n) - \mu_{h,t_n}}{\sigma_{h,t_n} + \epsilon}
  \]
  最终得分：
  \[
  s_2(n) = \max(Z_{\text{MeanTopK}}(n), \alpha Z_{\text{MaxWinLoss}}(n) + (1-\alpha) Z_{\text{MaxSpike}}(n))
  \]

#### Layer 3：CST引导的节点扰动分析
- **目标**：检测XOXO、ITGen等看起来语法正确、统计自然的语义触发器。
- 首先用惊奇度（surprise）筛选节点，计算节点token在有无上下文时的NLL差异的Top-Mean绝对差，直指**上下文敏感**而非单纯稀有。
- 对通过筛选的节点（最多B个），进行语法保留的净化（comment移除/替换为中性标记、字符串替换为安全字面量、标识符重命名、伪装块替换为no-op），构建扰动后输入 \(X_{(n)}\)。
- 使用代理模型在任务决策位置（如下一个token或yes/no）的logits，经温度T转换为分布：
  \[
  \pi_T(X) = \text{softmax}(g_{\phi}(X)/T)
  \]
- 计算原始与扰动后分布的Jensen-Shannon散度：
  \[
  s_3(n) = \text{JSD}(\pi_T(X), \pi_T(X_{(n)}))
  \]
- 若散度大于阈值，判定该节点为语义触发器。

#### 净化与总体流程
- 最终检测集合 \(\hat{S} = \hat{S}_1 \cup \hat{S}_2 \cup \hat{S}_3\)，统一进行净化。
- 默认**仅修改送入模型的副本**（prompt-only sanitization），不修改仓库源码，以保证语法有效性和大部分可执行结构。必要时可改用编译保持模式（基于语言特定重写，且回退语法检查）。

## 6. 主要结论与发现

- CodeSentinel在六类攻击上取得了**平均节点级F1 0.80**，显著优于CG (0.70)、DePA (0.51)、KBC (0.30)。
- 三层结构**均不可少**：层叠加时F1持续提升，而Layer 3单独虽强但高延迟，早退设计平衡了效率与精度。
- 检测性能**不依赖于特定代理模型**，跨模型和跨分词器稳定性好，适用于黑盒受害者模型（ASR显著下降）。
- 净化过程基本保持代码的可编译性和功能正确性，HumanEval/MBPP编译率不变，MBPP的Pass@1仅下降约6%。

## 7. 优点

- **结构化语法感知**：基于CST节点的细粒度分析，远优于行级或文件级方法。
- **三层互补机制**：覆盖显式规则异常、统计扰动、语义行为偏离，且通过早退控制开销。
- **推理时可部署**：无需受害者模型训练或权重，适用于黑盒Code LLM。
- **实验扎实**：包含多种攻击、多个基线和大量鲁棒性测试（自适应攻击等），且评估了代码效用保持。
- **净化策略安全**：仅修改模型输入副本，避免破坏源代码库，且支持语法回退检查。

## 8. 不足与局限

- **未涉及多文件/仓库级推理**：当前聚焦单文件上下文的净化，对跨文件分布式注入或多轮交互代理未做深入验证。
- **标识符重命名风险**：清洗标识符可能影响表面代码相似度（RepoBench精确匹配下降），虽编译率不变，但语义等价性未完全保证。
