## 摘要
大语言模型 (LLM) 在安全关键型工程领域产生了可控性差距：即使是极低比例的未检测约束违规，也会导致系统无法部署。当前的编排范式受困于谄媚性合规、上下文注意力衰减 [Liu et al., 2024] 以及自我修正过程中的随机振荡 [Huang et al., 2024]。我们提出了收敛式 AI 智能体框架 (CAAF)，通过三大支柱将智能体工作流从开环生成转变为闭环故障安全确定性 (Fail-Safe Determinism)：(1) 带有物理上下文防火墙的递归原子分解；(2) 以约束为资产 (Harness as an Asset)，将领域不变性形式化为由确定性统一断言接口 (UAI) 强制执行的机器可读注册表；(3) 带有状态锁定的结构化语义梯度，以实现单调收敛。在两个领域——SAE L3 级自动驾驶 (AD)（n=30，7 种工况）和制药连续流反应器设计（n=20，4 种工况，包括 Mono+UAI 消融实验）——的实证评估表明，基于 GPT-4o-mini 的 CAAF 实现了 100% 的悖论检测，而单体 GPT-4o 的检测率为 0%（即使在 temperature=0 时）。制药基准测试包含 7 个具有非线性阿伦尼乌斯 (Arrhenius) 相互作用的同步约束以及一个三向最小不可满足子集，在结构上比双约束的自动驾驶悖论更具挑战性。替代性的多智能体架构（辩论、顺序检查）在 80 次试验中同样表现为 0%，证实了 CAAF 的可靠性源于其确定性 UAI，而非多智能体编排本身。Mono+UAI 消融实验 (95%) 进一步证明了 UAI 是其核心贡献。CAAF 的可靠性对提示词暗示具有不变性；所有组件均使用单一通用模型，支持完全离线部署。

## Abstract
Large Language Models (LLMs) produce a controllability gap in safety-critical engineering: even low rates of undetected constraint violations render a system undeployable. Current orchestration paradigms suffer from sycophantic compliance, context attention decay [Liu et al., 2024], and stochastic oscillation during self-correction [Huang et al., 2024].   We introduce the Convergent AI Agent Framework (CAAF), which transitions agentic workflows from open-loop generation to closed-loop Fail-Safe Determinism via three pillars: (1) Recursive Atomic Decomposition with physical context firewalls; (2) Harness as an Asset, formalizing domain invariants into machine-readable registries enforced by a deterministic Unified Assertion Interface (UAI); and (3) Structured Semantic Gradients with State Locking for monotonic convergence.   Empirical evaluation across two domains -- SAE Level 3 (L3) autonomous driving (AD) (n=30, 7 conditions) and pharmaceutical continuous flow reactor design (n=20, 4 conditions including a Mono+UAI ablation) -- shows that CAAF-all-GPT-4o-mini achieves 100% paradox detection while monolithic GPT-4o achieves 0% (even at temperature=0). The pharmaceutical benchmark features 7 simultaneous constraints with nonlinear Arrhenius interactions and a 3-way minimal unsatisfiable subset, representing a structurally harder challenge than the 2-constraint AD paradox. Alternative multi-agent architectures (debate, sequential checking) also achieve 0% across 80 trials, confirming that CAAF's reliability derives from its deterministic UAI, not from multi-agent orchestration per se. A Mono+UAI ablation (95%) isolates UAI as the core contribution. CAAF's reliability is invariant to prompt hints; all components use a single commodity model, enabling fully offline deployment.

    1.  **谄媚性合规（Sycophantic Compliance）**：模型为了完成任务而忽视物理不可能性，产生看似合理实则违背物理定律的“合规幻觉”。
    2.  **上下文腐烂（Context Rot）**：在长文本中，模型会逐渐忽略早期定义的关键安全约束。
    3.  **随机振荡（Stochastic Oscillation）**：在自我修正循环中，修复一个约束往往会导致另一个已满足的约束再次失效，无法收敛。
*   **核心目标**：通过引入控制论思想，将 LLM 从“开环生成”转变为“闭环确定性控制”，确保系统要么输出物理验证合格的方案，要么明确报告约束冲突（悖论）。

### 2. 方法论：核心思想与关键技术
CAAF 框架由三大支柱组成：
*   **支柱一：递归原子分解（RAD）与物理上下文防火墙**
    *   将复杂任务分解为拓扑图（DAG）。
    *   **核心技术**：通过独立的 API 线程实现“上下文防火墙”，每个执行器（Executor）只能看到与其原子任务相关的上下文，防止安全约束与成本等业务目标发生交叉污染。
*   **支柱二：约束即资产（Harness as an Asset, HaaA）**
    *   将领域知识（如物理定律、行业标准）形式化为机器可读的 YAML 注册表。
    *   **统一断言接口（UAI）**：使用确定性的 Python `eval()` 或外部仿真工具进行验证。UAI 是不可违抗的“物理红线”，LLM 无法通过修辞手段对其进行辩解。
*   **支柱三：结构化语义梯度（SSG）与状态锁定（State Locking）**
    *   **语义梯度**：当 UAI 报错时，不仅返回失败，还返回精确的数学边界（如“速度必须 ≤ 55km/h”），引导模型向可行域搜索。
    *   **状态锁定**：一旦某个维度的约束通过验证，该参数在后续迭代中被标记为“只读”，确保系统单调收敛，彻底消除随机振荡。

1.  **架构优于模型**：使用廉价的 GPT-4o-mini 配合 CAAF 架构，在可靠性上完胜单体 GPT-4o。

4.  **多智能体辩论无效**：在物理约束场景下，没有外部 ground truth 的辩论往往会演变成“虚假共识”，无法解决可靠性问题。

### 7. 优点（亮点）
*   **闭环控制思想**：将传统的软件工程断言与 LLM 的启发式搜索完美结合。
*   **故障安全（Fail-Safe）**：系统在无法满足约束时会主动“报错”并提供决策菜单，而非默默失败。
*   **资产化思维**：将行业知识沉淀为可复用的 YAML 约束资产，不随模型更换而流失。
*   **部署友好**：支持完全离线部署，满足高安全性行业的合规需求。

### 8. 不足与局限
*   **约束完备性依赖**：系统的确定性上限取决于 Harness（约束集）的完备性。如果人类漏写了某个物理定律，系统仍可能产生错误。
*   **延迟开销**：由于涉及多次迭代和拓扑节点执行，其推理延迟高于单体模型，不适用于毫秒级的实时控制（更适合工程设计和需求分析）。
*   **领域覆盖**：目前仅在物理定律明确的两个领域进行了验证，对于法律、金融等模糊约束领域的泛化能力尚待进一步研究。
*   **编排复杂性**：对于极其复杂的系统（如数千个约束），Orchestrator 可能会面临上下文饱和，需要更高级的分层分区策略。
