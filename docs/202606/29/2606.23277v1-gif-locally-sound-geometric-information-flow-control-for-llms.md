## 摘要
大语言模型在智能体系统中日益成为敏感数据、不可信输入和特权操作之间的交互中介，带来了安全与隐私风险。这些风险包括通过提示注入操纵下游工具使用，以及通过模型输出泄露机密信息。近期基于信息流控制（IFC）的防御方法展现了潜力，但缺乏用于推理模型自身信息流的严格语义基础。由于自回归大语言模型中任何输入 token 都可能影响任何输出 token，现有方法面临严重的污点爆炸问题。我们提出几何信息流（GIF），一种追踪从输入 token 到输出的信息流的语义框架。GIF 利用大语言模型的雅可比矩阵和局部输出几何，为扰动输入片段与模型输出之间的香农互信息提供上界，从而得到一种可通过自动微分和低秩近似在大模型上计算的可扩展度量。与基于注意力或相关性的归因启发式方法不同，GIF 满足局部几何可靠性，并且我们提供了完整的 Lean 4 机械化证明，表明在局部正则性假设下，它对给定提示引发的真实信息流进行了上界估计。我们在多个提示注入和隐私泄露基准上评估了 GIF 在完整性和机密性任务上的表现。即使没有下游降密器，GIF 也能实现近乎完美的召回率，优于基于注意力的基准。结合轻量级基于大语言模型的降密器，它在 F1 分数上达到或超过了 LLM-as-judge 基准（如 GPT-5.5 xhigh reasoning），同时 token 成本最多降低 81 倍。使用小型代理模型检测到的 GIF 流可以迁移到更大的先进模型和其他模型家族，即使代理模型小至 200 倍，这表明可以在没有梯度访问的情况下进行黑盒部署。

## Abstract
Large language models increasingly mediate interactions between sensitive data, untrusted inputs, and privileged actions in agentic systems, creating security and privacy risks. These range from prompt injections that manipulate downstream tool use to leakage of confidential information through model outputs. Recent Information Flow Control (IFC)-based defenses show promise but lack a principled semantic foundation for reasoning about information flow through the model itself. Since any input token may influence any output token in an autoregressive LLM, existing approaches suffer from severe taint explosion.   We present Geometric Information Flow (GIF), a semantic framework for tracking information flow from input tokens to outputs. GIF uses the LLM Jacobian and local output geometry to upper-bound the Shannon mutual information between perturbed input spans and model outputs, yielding a scalable measure computable on large models via automatic differentiation and low-rank approximation. Unlike attention-based or correlational attribution heuristics, GIF satisfies local geometric soundness, and we provide a fully mechanized Lean 4 proof that it upper-bounds the true information flow induced by a given prompt under local regularity assumptions.   We evaluate GIF on integrity and confidentiality tasks across multiple prompt-injection and privacy-leakage benchmarks. GIF achieves near-perfect recall even without a downstream declassifier, outperforming attention-based baselines. Combined with lightweight LLM-based declassifiers, it matches or exceeds the F1 of direct LLM-as-judge baselines such as GPT-5.5 xhigh reasoning while using up to 81x lower token cost. GIF flows detected with small surrogate models transfer to larger state-of-the-art models and other model families, even when the surrogate is up to 200x smaller, suggesting black-box deployment without gradient access.

## 1. 核心问题与整体含义

大型语言模型（LLMs）正日益嵌入到智能体（agentic）系统中，在这些系统中，敏感数据、不可信输入和特权操作通过模型调用交织在一起，带来了两大安全风险：

- **完整性（integrity）威胁**：攻击者通过提示注入，使不可信内容成为指令来源，操纵下游工具选择、代码修改或记忆更新等特权行为。
- **机密性（confidentiality）威胁**：模型在处理私有上下文时，可能将机密信息泄露到生成输出、工具参数或记忆更新中。

传统的基于信息流控制（IFC）的方法试图通过安全标签传播来约束这些依赖，但它们假设模型内部有透明的执行语义，而自回归LLM中任何输入token都可能影响任何输出token，导致标签污染（taint explosion）严重，要么极度保守，要么计算上不可行。现有的启发式防御（如注意力分数、LLM-as-judge监控）又缺乏语义基础，容易产生盲点。因此，论文的核心问题是：**如何以一种具有原则性语义基础、可扩展且保守的方式，定量跟踪LLM内部从输入片段到后续输出的信息流**，从而支持更精确的安全策略执行。

## 2. 方法论

论文提出**几何信息流（GIF）**框架，其核心思想是将LLM对输入片段（token span）的局部敏感性量化为一种信息论信道容量，并以此作为信息流的上界。关键技术点如下：

- **局部信道建模**：选定一个基提示（base prompt）x，一个输入片段T和输出位置t\*，在嵌入空间中对T施加小的随机扰动 U_T ~ N(0, τ I)，观察模型下一个token分布的变化。定义真实信息流为扰动与输出token间的**香农互信息** I(U_T; Y)。

- **局部高斯代理信道**：由于真实互信息难以计算，GIF利用模型在基提示处的**雅可比矩阵** J_T(x) 以及输出分布在该点的**Fisher信息阵** F_sm(p(x))，构造一个线性高斯代理信道：
  Y_T = A_T(x) U_T + N,  其中 A_T = F_sm(p)^{1/2} W J_T。
  该代理信道在局部二阶意义上忠实于真实模型（其每点KL散度与真实模型匹配至o(∥u∥²)）。

- **操作化度量与局部可靠性**：代理信道的**信道容量** Cap_T(τ;x) = ½ log det(I + τ M_T)（其中 M_T = J_T^T W^T F_sm W J_T 称为Fisher拉回矩阵）被采纳为GIF分数。论文在Lean 4中机械化证明了：在局部二次控制假设（Assumption 3.1）下，Cap_T(τ;x) 是真实互信息 I(U_T;Y) 的上界（至 o(τ) 误差），即该度量是**局部可靠的（locally sound）**——永远不会渐近低估真实信息流。

- **可扩展近似**：直接计算M_T需要完整的雅可比矩阵，代价巨大。GIF利用**影响度（Influence）** Inf_T(x) = tr(M_T) 是信道容量的一阶代理，并采用 **Hutch++随机迹估计**，通过少量矩阵-向量乘积 oracle 调用（无需显式构建M_T）高效估计Inf_T。每个 probe 只需一次前向/反向传播，大大降低成本。

- **策略执行层面**：在实际系统中，GIF与安全标签和降密器（declassifier）结合。当生成一个受保护输出token时，GIF对上下文中的每个token计算影响度，得到排序。然后将少数最高影响度片段与任务描述一同交给轻量级LLM降密器做出ALLOW/DENY裁决，从而以极低成本执行安全策略。

## 3. 实验设计

实验旨在从多个维度验证GIF的有效性、高效性和可迁移性。

- **数据集与场景**：
  - **AgentDojo**（完整性）：提示注入攻击的安全测试环境。
  - **MSB (MCP Security Bench)**（完整性）：针对MCP协议工具调用的提示注入基准。
  - **AgentDAM**（机密性）：在WebArena派生的浏览器任务中评估隐私泄露，覆盖GitLab、Reddit、购物等环境。
  - 所有任务均通过基准提供的攻击成功检查器或LLM-judge获取轨迹级标签；策略相关token通过字符串匹配与人工辅助标注。

- **模型**：使用6个开源推理模型作为骨干生成代理轨迹和分析对象：Qwen 3 (8B, 14B, 32B), Gemma 4 31B, GPT-OSS (20B, 120B)。此外，GPT-5.5 xhigh 用作对比的LLM-as-judge基线。

- **对比方法**：
  - **RTBAS**：基于注意力分数的启发式归因。
  - **GIF⁻**：基于梯度（单logit）的归因基线，仅考虑生成token的logit，而非完整的下一token分布几何。
  - **GPT-5.5 xhigh**：作为纯推理型LLM判断器，对完整轨迹或截断证据进行ALLOW/DENY裁决。
  - **全轨迹LLM judge**：使用与降密器相同的LLM但处理完整轨迹。

- **典型实验设置**：
  - **RQ1**: GIF作为独立策略信号，通过固定阈值分类器（PSP@k）评估其对真实策略相关token的排序能力（AUROC/AUPRC）。
  - **RQ2**: 将GIF与LLM降密器结合，使用top-k高影响证据压缩轨迹，比较准确率与token消耗。
  - **RQ3**: 消融研究，包括Hutch++近似准确性、代理模型迁移（用小模型分析大模型轨迹）、归因截止值k的影响。
  - **RQ4**: 定性案例分析，展示GIF在部分成功攻击和零流出情形下的流模式。

## 6. 主要结论与发现

- **GIF作为策略信号极强**：即使没有降密器，GIF也能以近乎完美的召回率识别出策略相关token，显著优于注意力基准，并优于单logit梯度法。与GPT-5.5 judge相比，AUROC平均提高11.2%，同时成本降低约9.3倍。
- **压缩证据有效性**：GIF与LLM降密器结合时，仅用top-3高影响片段即可达到甚至超过全轨迹LLM法官的性能（相对提升最高22%），而token消耗减少63-73%，使得弱小模型（如Qwen 3 8B）也能达到强大模型（如GPT-5.5）的策略判断水平。
- **近似与迁移能力强**：Hutch++低秩估计很快收敛（秩8时Spearman相关系数>0.98），表明低秩近似已足够区分关键token。更关键的是，用极小的代理模型（如Qwen 3 0.6B）计算的GIF流可以几乎无损地迁移到200倍大的目标模型和跨模型家族，意味着可以在没有梯度访问的黑盒场景下部署GIF。
- **几何语义的实践意义**：GIF将雅可比敏感性转化为一种具有局部上界保证的定量信息流度量，为LLM IFC提供了一种原则性基础，避免了启发式方法的不可靠性。

## 7. 优点

- **理论扎实，形式化保证**：用信息论和微分几何严格定义了信息流，并在Lean 4中进行了机械化证明，确保了度量的保守性，这在此类工作中十分罕见。
- **可扩展、低成本**：通过低秩迹估计避免了显式构建大规模雅可比矩阵，使得计算具有实用性；结合代理模型可实现黑盒迁移。
- **性能显著且一致**：在多个基准、多个模型、多种任务上，GIF均明显优于注意力和简单梯度基线，压缩证据后能以极小成本匹配强大判断器。
- **系统设计完整**：从理论到实现，包括安全标签、流跟踪、降密器集成，形成了一套可以直接嵌入代理工作流的IFC方案。
- **跨模型迁移的发现**：小模型分析大模型流具有高度保真性，这对实际部署（如API访问受限模型）非常有益。

## 8. 不足与局限

- **白盒依赖假设**：GIF的计算需要访问模型内部梯度，对于完全闭源的API服务，目前只能依赖代理模型，文中虽证明可迁移，但代理模型的可用性和通用性还需进一步验证。
- **局部线性假设**：整个分析基于嵌入侵扰的局部二次控制假设，当输入扰动幅度大或模型非线性强时，度量的可靠性可能下降。文中的理论工作仅确保小扰动下的上界，实际安全策略仍需谨慎。
- **对抗适应性未知**：攻击者可能构造对抗性提示，使扰动方向处在模型“不敏感”区域内，从而规避GIF检测。论文未测试有针对性的自适应攻击。
- **实验环境与真实落差**：基准环境（AgentDojo等）是精心设计的测试，虽然代表性较强，但实际生产环境中的代理交互可能更复杂，安全策略的多样性也超出这些基准。
- **降密器依赖LLM**：降密器本身仍是基于LLM，可能受到欺骗或偏见，并未形成完整的形式化降密策略，只是一个半自动化解决方案。
- **计算仍有开销**：尽管使用了近似，每次生成安全敏感token时仍需进行多步梯度计算，在高吞吐量系统中可能成为瓶颈。
